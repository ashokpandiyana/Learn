# Chapter 8: Positioning & Layout

## 8.1 Display Property

Control how elements are displayed in the document flow.

### Block & Inline

```html
<!-- Block: Full width, new line -->
<div class="block">Takes full width, stacks vertically</div>

<!-- Inline: Content width, flows with text -->
<span class="inline">Flows with text</span>

<!-- Inline-block: Inline flow but respects width/height -->
<span class="inline-block w-32 h-8">Has dimensions but inline</span>
```

### Flex & Grid

```html
<!-- Flex containers -->
<div class="flex">Block-level flex container</div>
<span class="inline-flex">Inline flex container</span>

<!-- Grid containers -->
<div class="grid">Block-level grid container</div>
<span class="inline-grid">Inline grid container</span>
```

### Table Display

```html
<div class="table w-full">
  <div class="table-header-group">
    <div class="table-row">
      <div class="table-cell">Header 1</div>
      <div class="table-cell">Header 2</div>
    </div>
  </div>
  <div class="table-row-group">
    <div class="table-row">
      <div class="table-cell">Cell 1</div>
      <div class="table-cell">Cell 2</div>
    </div>
  </div>
</div>

<!-- Other table utilities -->
<div class="table-caption">Caption</div>
<div class="table-column">Column</div>
<div class="table-column-group">Column Group</div>
<div class="table-footer-group">Footer</div>
```

### Special Display Values

```html
<!-- Flow-root: Creates new Block Formatting Context -->
<div class="flow-root">
  <!-- Contains floats, prevents margin collapse -->
</div>

<!-- Contents: Element disappears, children remain -->
<div class="contents">
  <!-- This div is removed from layout, children promoted -->
  <span>Child 1</span>
  <span>Child 2</span>
</div>

<!-- Hidden: Completely removed from DOM flow -->
<div class="hidden">Not rendered at all</div>

<!-- List item: For custom lists -->
<div class="list-item">Acts like <li></div>
```

## 8.2 Position

Control positioning method for elements.

```html
<!-- Static: Normal document flow (default) -->
<div class="static">
  Normal flow, top/right/bottom/left have no effect
</div>

<!-- Relative: Positioned relative to normal position -->
<div class="relative">
  Offset from where it would normally be
  <div class="absolute">Child can be positioned relative to this</div>
</div>

<!-- Absolute: Removed from flow, positioned relative to nearest positioned ancestor -->
<div class="absolute top-0 left-0">
  Positioned relative to nearest relative/absolute/fixed parent
</div>

<!-- Fixed: Positioned relative to viewport -->
<div class="fixed top-0 left-0 right-0">
  Stays in place when scrolling (like sticky headers)
</div>

<!-- Sticky: Hybrid of relative and fixed -->
<div class="sticky top-0">
  Normal flow until scroll threshold, then sticks
</div>
```

## 8.3 Inset (Top, Right, Bottom, Left)

Position offset values for positioned elements.

### All Sides

```html
<!-- Inset: All four sides at once -->
<div class="absolute inset-0">
  top: 0; right: 0; bottom: 0; left: 0;
  (Fills the positioned parent)
</div>

<div class="absolute inset-4">
  16px from all edges
</div>

<div class="absolute inset-auto">
  All sides auto
</div>
```

### Axis-Based

```html
<!-- Horizontal (left + right) -->
<div class="absolute inset-x-0">left: 0; right: 0;</div>
<div class="absolute inset-x-4">left: 16px; right: 16px;</div>

<!-- Vertical (top + bottom) -->
<div class="absolute inset-y-0">top: 0; bottom: 0;</div>
<div class="absolute inset-y-4">top: 16px; bottom: 16px;</div>
```

### Individual Sides

```html
<div class="absolute top-0">top: 0</div>
<div class="absolute right-0">right: 0</div>
<div class="absolute bottom-0">bottom: 0</div>
<div class="absolute left-0">left: 0</div>

<!-- With spacing scale -->
<div class="absolute top-4">top: 16px</div>
<div class="absolute right-8">right: 32px</div>
<div class="absolute bottom-2">bottom: 8px</div>
<div class="absolute left-1/2">left: 50%</div>

<!-- Negative values -->
<div class="absolute -top-4">top: -16px (outside parent)</div>
<div class="absolute -right-2">right: -8px</div>
```

### Logical Properties (RTL Support)

```html
<!-- Start/End instead of Left/Right -->
<div class="absolute start-0">Inline start (left in LTR, right in RTL)</div>
<div class="absolute end-0">Inline end (right in LTR, left in RTL)</div>

<div class="absolute inset-s-0">Inline start: 0</div>
<div class="absolute inset-e-0">Inline end: 0</div>
```

### Common Positioning Patterns

```html
<!-- Full overlay -->
<div class="absolute inset-0 bg-black/50">
  Covers entire parent
</div>

<!-- Centered with transform -->
<div class="absolute top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2">
  Perfectly centered
</div>

<!-- Bottom right corner -->
<div class="absolute bottom-4 right-4">
  Bottom right with padding
</div>

<!-- Full width at top -->
<div class="absolute top-0 inset-x-0 h-16">
  Full width header
</div>

<!-- Sidebar -->
<div class="absolute top-0 bottom-0 left-0 w-64">
  Left sidebar
</div>
```

## 8.4 Z-Index

Control stacking order of positioned elements.

```html
<!-- Predefined values -->
<div class="z-0">z-index: 0</div>
<div class="z-10">z-index: 10</div>
<div class="z-20">z-index: 20</div>
<div class="z-30">z-index: 30</div>
<div class="z-40">z-index: 40</div>
<div class="z-50">z-index: 50</div>

<!-- Auto -->
<div class="z-auto">z-index: auto</div>

<!-- Negative values -->
<div class="-z-10">z-index: -10 (behind)</div>

<!-- Arbitrary values -->
<div class="z-[100]">z-index: 100</div>
<div class="z-[9999]">z-index: 9999</div>
```

## 8.5 Float & Clear

Legacy layout technique (rarely needed with Flexbox/Grid).

```html
<!-- Float -->
<img class="float-left mr-4" src="..." alt="...">
<p>Text wraps around the floated image...</p>

<img class="float-right ml-4" src="..." alt="...">
<p>Text wraps around to the left...</p>

<div class="float-none">No floating</div>

<!-- Logical floats -->
<img class="float-start">Float to inline-start</img>
<img class="float-end">Float to inline-end</img>

<!-- Clear -->
<div class="clear-left">Clears left floats</div>
<div class="clear-right">Clears right floats</div>
<div class="clear-both">Clears all floats</div>
<div class="clear-none">No clearing</div>
```

## 8.6 Isolation

Control stacking context creation.

```html
<!-- Create new stacking context -->
<div class="isolate">
  <div class="relative z-10">Inside isolated context</div>
</div>

<!-- Auto (default) -->
<div class="isolation-auto">
  Normal stacking behavior
</div>
```

### When to Use Isolate

```html
<!-- Problem: z-index fighting between components -->
<div class="relative z-10">Component A</div>
<div class="relative z-20">Component B overlaps A</div>

<!-- Solution: Isolate each component -->
<div class="isolate">
  <div class="relative z-10">Component A</div>
  <!-- Internal z-index doesn't leak out -->
</div>
<div class="isolate">
  <div class="relative z-20">Component B</div>
  <!-- Can't overlap A anymore -->
</div>
```

## 8.7 Object Fit

Control how replaced elements (img, video) fit their container.

```html
<!-- Contain: Scale to fit, maintain aspect ratio, may have gaps -->
<img class="w-full h-48 object-contain" src="..." alt="...">

<!-- Cover: Scale to cover, maintain aspect ratio, may crop -->
<img class="w-full h-48 object-cover" src="..." alt="...">

<!-- Fill: Stretch to fill, ignore aspect ratio -->
<img class="w-full h-48 object-fill" src="..." alt="...">

<!-- None: Original size, may overflow -->
<img class="w-full h-48 object-none" src="..." alt="...">

<!-- Scale-down: Smaller of none or contain -->
<img class="w-full h-48 object-scale-down" src="..." alt="...">
```

## ğŸ’¡ Interview Quick Tips

1. **"Difference between relative and absolute?"**
   - `relative`: Offset from original position, keeps space
   - `absolute`: Removed from flow, positioned to ancestor

2. **"How does sticky work?"**
   - Normal flow until scroll threshold
   - Then behaves like fixed within container

3. **"What creates a stacking context?"**
   - Positioned element with z-index
   - `isolate`, opacity < 1, transforms, filters

4. **"object-fit vs background-size?"**
   - `object-fit`: For `<img>`, `<video>` elements
   - `background-size`: For CSS background images

5. **"flow-root use cases?"**
   - Containing floats
   - Preventing margin collapse
