## 1. Center Anything

Centering content is one of the most common tasks in web development. There are multiple approaches, each with its own use case.

### 1.1 The Three Main Approaches

| Method | Best For | Pros | Cons |
|--------|----------|------|------|
| **Flexbox** | Most cases | Simple, flexible | Requires parent container |
| **Grid** | Complex layouts | Powerful, modern | Overkill for simple centering |
| **Absolute + Transform** | Overlays, modals | Works with unknown sizes | Requires relative parent |

### 1.2 Flexbox Centering

#### Why Flexbox?

Flexbox is the most common and flexible approach for centering. It works for:
- Unknown content sizes
- Multiple items
- Both horizontal and vertical centering
- Responsive layouts

#### Basic Centering:

```html
<!-- Center anything horizontally and vertically -->
<div class="flex items-center justify-center h-screen">
  <div class="p-8 bg-white rounded-lg shadow-lg">
    <h2 class="text-2xl font-bold">Centered Content</h2>
    <p>This is perfectly centered</p>
  </div>
</div>
```

**How it works:**
- `flex` - Enables flexbox
- `items-center` - Vertical centering (align-items: center)
- `justify-center` - Horizontal centering (justify-content: center)
- `h-screen` - Full viewport height

#### Centering Multiple Items:

```html
<!-- Center multiple items with spacing -->
<div class="flex flex-col items-center justify-center gap-4 h-screen">
  <button class="px-6 py-3 bg-blue-600 text-white rounded-lg">
    Button 1
  </button>
  <button class="px-6 py-3 bg-blue-600 text-white rounded-lg">
    Button 2
  </button>
  <button class="px-6 py-3 bg-blue-600 text-white rounded-lg">
    Button 3
  </button>
</div>
```

**Key changes:**
- `flex-col` - Stack items vertically
- `gap-4` - Space between items

#### Centering with Max Width:

```html
<!-- Center content with maximum width -->
<div class="flex items-center justify-center min-h-screen px-4">
  <div class="w-full max-w-md p-8 bg-white rounded-lg shadow-lg">
    <h2 class="text-2xl font-bold mb-4">Sign In</h2>
    <form class="space-y-4">
      <input 
        type="email" 
        placeholder="Email"
        class="w-full px-4 py-2 border rounded-lg"
      />
      <input 
        type="password" 
        placeholder="Password"
        class="w-full px-4 py-2 border rounded-lg"
      />
      <button class="w-full py-2 bg-blue-600 text-white rounded-lg">
        Sign In
      </button>
    </form>
  </div>
</div>
```

**Why this approach:**
- `min-h-screen` - At least full viewport height
- `px-4` - Padding on sides for mobile
- `w-full max-w-md` - Responsive width with maximum

#### Horizontal Centering Only:

```html
<!-- Center horizontally, top-aligned -->
<div class="flex justify-center pt-20">
  <div class="p-8 bg-white rounded-lg shadow-lg">
    Content at top, centered horizontally
  </div>
</div>
```

#### Vertical Centering Only:

```html
<!-- Center vertically, left-aligned -->
<div class="flex items-center h-screen">
  <div class="p-8 bg-white rounded-lg shadow-lg">
    Content on left, centered vertically
  </div>
</div>
```

### 1.3 Grid Centering

#### Why Grid?

Grid is excellent for:
- Complex layouts
- Two-dimensional centering
- When you're already using grid

#### Basic Grid Centering:

```html
<!-- Grid approach - most concise -->
<div class="grid place-items-center h-screen">
  <div class="p-8 bg-white rounded-lg shadow-lg">
    <h2 class="text-2xl font-bold">Grid Centered</h2>
    <p>Centered with CSS Grid</p>
  </div>
</div>
```

**How it works:**
- `grid` - Enables CSS Grid
- `place-items-center` - Shorthand for `align-items: center` + `justify-items: center`

#### Grid with Multiple Items:

```html
<!-- Center multiple items in a grid -->
<div class="grid place-items-center gap-4 h-screen">
  <div class="p-4 bg-blue-100 rounded-lg">Item 1</div>
  <div class="p-4 bg-blue-100 rounded-lg">Item 2</div>
  <div class="p-4 bg-blue-100 rounded-lg">Item 3</div>
</div>
```

#### place-content vs place-items:

```html
<!-- place-items - centers each item in its grid cell -->
<div class="grid grid-cols-3 place-items-center gap-4 h-screen">
  <div class="p-4 bg-blue-100">1</div>
  <div class="p-4 bg-blue-100">2</div>
  <div class="p-4 bg-blue-100">3</div>
</div>

<!-- place-content - centers the entire grid -->
<div class="grid grid-cols-3 place-content-center gap-4 h-screen">
  <div class="p-4 bg-blue-100">1</div>
  <div class="p-4 bg-blue-100">2</div>
  <div class="p-4 bg-blue-100">3</div>
</div>
```

### 1.4 Absolute Positioning + Transform

#### Why Absolute + Transform?

Best for:
- Modals and overlays
- Tooltips
- Elements that need to break out of normal flow
- When you don't want to affect parent layout

#### Basic Absolute Centering:

```html
<!-- Parent must have position: relative -->
<div class="relative h-screen bg-gray-100">
  <div class="absolute top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2
              p-8 bg-white rounded-lg shadow-lg">
    <h2 class="text-2xl font-bold">Absolutely Centered</h2>
    <p>Using transform technique</p>
  </div>
</div>
```

**How it works:**
- `relative` on parent - Creates positioning context
- `absolute` on child - Takes element out of flow
- `top-1/2 left-1/2` - Positions top-left corner at center
- `-translate-x-1/2 -translate-y-1/2` - Shifts element back by half its size

#### Modal Overlay Example:

```html
<!-- Full-screen modal overlay -->
<div class="fixed inset-0 bg-black/50 flex items-center justify-center z-50">
  <div class="relative bg-white rounded-lg shadow-2xl max-w-md w-full mx-4">
    <!-- Close button -->
    <button class="absolute -top-2 -right-2 w-8 h-8 bg-red-500 text-white rounded-full
                   hover:bg-red-600 focus:outline-none focus:ring-2 focus:ring-red-300">
      √ó
    </button>
    
    <!-- Modal content -->
    <div class="p-6">
      <h2 class="text-2xl font-bold mb-4">Modal Title</h2>
      <p class="text-gray-600 mb-6">Modal content goes here...</p>
      <button class="w-full py-2 bg-blue-600 text-white rounded-lg">
        Confirm
      </button>
    </div>
  </div>
</div>
```

#### Tooltip Example:

```html
<!-- Tooltip positioned above button -->
<div class="relative inline-block">
  <button class="px-4 py-2 bg-blue-600 text-white rounded-lg">
    Hover me
  </button>
  
  <!-- Tooltip -->
  <div class="absolute bottom-full left-1/2 -translate-x-1/2 mb-2
              px-3 py-2 bg-gray-900 text-white text-sm rounded-lg
              opacity-0 hover:opacity-100 transition-opacity
              whitespace-nowrap">
    This is a tooltip
    <!-- Arrow -->
    <div class="absolute top-full left-1/2 -translate-x-1/2
                border-4 border-transparent border-t-gray-900"></div>
  </div>
</div>
```

## 2. Responsive Card Grid

### 2.1 Understanding the Pattern

Card grids are one of the most common UI patterns. They need to:
- Display multiple cards in a grid
- Adapt to different screen sizes
- Maintain consistent spacing
- Handle varying content heights

### 2.2 Basic Responsive Grid

```html
<!-- Basic responsive card grid -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6">
  <!-- Card 1 -->
  <div class="bg-white rounded-lg shadow-md overflow-hidden">
    <img 
      src="image1.jpg" 
      alt="Product 1"
      class="w-full h-48 object-cover"
    />
    <div class="p-6">
      <h3 class="text-xl font-bold text-gray-900 mb-2">Product Name</h3>
      <p class="text-gray-600 mb-4">Product description goes here...</p>
      <button class="w-full py-2 bg-blue-600 text-white rounded-lg">
        Add to Cart
      </button>
    </div>
  </div>

  <!-- Repeat cards... -->
</div>
```

**Breakpoint behavior:**
- Mobile (`< 640px`): 1 column
- Tablet (`640px+`): 2 columns
- Desktop (`1024px+`): 3 columns
- Large (`1280px+`): 4 columns

### 2.3 Grid Gap Variations

```html
<!-- Small gap -->
<div class="grid grid-cols-1 md:grid-cols-3 gap-4">
  <!-- Cards -->
</div>

<!-- Medium gap (default) -->
<div class="grid grid-cols-1 md:grid-cols-3 gap-6">
  <!-- Cards -->
</div>

<!-- Large gap -->
<div class="grid grid-cols-1 md:grid-cols-3 gap-8">
  <!-- Cards -->
</div>

<!-- Responsive gap -->
<div class="grid grid-cols-1 md:grid-cols-3 gap-4 md:gap-6 lg:gap-8">
  <!-- Cards -->
</div>
```

### 2.4 Auto-Fit vs Fixed Columns

#### Auto-Fit (Fluid Columns):

```html
<!-- Cards automatically fit based on minimum width -->
<div class="grid grid-cols-[repeat(auto-fit,minmax(280px,1fr))] gap-6">
  <!-- Cards will automatically adjust -->
</div>
```

**Pros:**
- Truly fluid
- No breakpoints needed
- Adapts to any screen size

**Cons:**
- Less control
- Can create awkward layouts
- Requires arbitrary values

#### Fixed Columns (More Control):

```html
<!-- Explicit breakpoints for better control -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6">
  <!-- More predictable behavior -->
</div>
```

## 3. Sticky Header

### 3.1 What is a Sticky Header?

A sticky header stays at the top of the viewport as you scroll. It provides:
- Persistent navigation
- Better UX
- Easy access to important actions
- Professional feel

### 3.2 Basic Sticky Header

```html
<!-- Simple sticky header -->
<header class="sticky top-0 z-50 bg-white shadow-md">
  <div class="container mx-auto px-4">
    <nav class="flex items-center justify-between h-16">
      <!-- Logo -->
      <div class="font-bold text-xl">Logo</div>
      
      <!-- Navigation -->
      <ul class="flex gap-6">
        <li><a href="/" class="hover:text-blue-600">Home</a></li>
        <li><a href="/about" class="hover:text-blue-600">About</a></li>
        <li><a href="/contact" class="hover:text-blue-600">Contact</a></li>
```

## 5. Aspect Ratio

### 5.1 What is Aspect Ratio?

Aspect ratio maintains a consistent width-to-height ratio regardless of screen size. Common use cases:
- Video players (16:9)
- Images
- Cards
- Thumbnails

### 5.2 Built-in Aspect Ratios

```html
<!-- 1:1 Square -->
<div class="aspect-square bg-gray-200">
  <img src="image.jpg" alt="..." class="w-full h-full object-cover" />
</div>

<!-- 16:9 Video -->
<div class="aspect-video bg-gray-200">
  <iframe class="w-full h-full" src="https://youtube.com/embed/..."></iframe>
</div>

<!-- Auto (natural aspect ratio) -->
<div class="aspect-auto">
  <img src="image.jpg" alt="..." />
</div>
```

### 5.3 Custom Aspect Ratios

```html
<!-- 4:3 aspect ratio -->
<div class="aspect-[4/3] bg-gray-200">
  <img src="image.jpg" alt="..." class="w-full h-full object-cover" />
</div>

<!-- 21:9 ultrawide -->
<div class="aspect-[21/9] bg-gray-200">
  Content
</div>

<!-- 3:2 photography -->
<div class="aspect-[3/2] bg-gray-200">
  <img src="photo.jpg" alt="..." class="w-full h-full object-cover" />
</div>
```

## 6. Line Clamp (Truncation)

### 6.1 What is Line Clamp?

Line clamp truncates text after a specific number of lines, adding an ellipsis (...). Perfect for:
- Card descriptions
- Article previews
- Product names
- Comments

### 6.2 Basic Line Clamp

```html
<!-- Truncate after 1 line -->
<p class="line-clamp-1">
  This is a very long text that will be truncated after one line with an ellipsis...
</p>

<!-- Truncate after 2 lines -->
<p class="line-clamp-2">
  This is a longer text that will be truncated after two lines. 
  It allows for a bit more content to show before adding the ellipsis...
</p>

<!-- Truncate after 3 lines -->
<p class="line-clamp-3">
  This is an even longer text that will be truncated after three lines. 
  This is useful for card descriptions or article previews where you want 
  to show just enough to give the reader an idea of the content...
</p>

<!-- Truncate after 4, 5, or 6 lines -->
<p class="line-clamp-4">Long text...</p>
<p class="line-clamp-5">Long text...</p>
<p class="line-clamp-6">Long text...</p>

<!-- No truncation (show all) -->
<p class="line-clamp-none">
  All text will be shown regardless of length...
</p>
```

## 7. Gradient Text

### 7.1 Creating Gradient Text

Gradient text uses a background gradient with `bg-clip-text` to apply the gradient to text only.

```html
<!-- Basic gradient text -->
<h1 class="text-6xl font-bold bg-gradient-to-r from-purple-500 to-pink-500 
           bg-clip-text text-transparent">
  Gradient Text
</h1>
```

**How it works:**
1. `bg-gradient-to-r` - Creates gradient background
2. `from-purple-500 to-pink-500` - Gradient colors
3. `bg-clip-text` - Clips background to text shape
4. `text-transparent` - Makes text transparent so gradient shows

### 7.2 Gradient Directions

```html
<!-- Left to Right -->
<h1 class="text-5xl font-bold bg-gradient-to-r from-blue-500 to-purple-500

  Diagonal Gradient
</h1>
```

## 8. Glass Effect (Glassmorphism)

### 8.1 What is Glassmorphism?

Glassmorphism creates a frosted glass effect with:
- Semi-transparent background
- Backdrop blur
- Subtle borders
- Shadows


- `bg-white/10` - 10% opaque white background

## 10. Custom Scrollbar

### 10.1 Note About Scrollbars

Custom scrollbars require a plugin or custom CSS. Tailwind doesn't include scrollbar utilities by default.

### 10.2 Using tailwind-scrollbar Plugin

```bash
npm install -D tailwind-scrollbar
```

```javascript
// tailwind.config.js
module.exports = {
  plugins: [
    require('tailwind-scrollbar'),
  ],
}
```

```html
<!-- Custom scrollbar -->
<div class="h-64 overflow-auto scrollbar-thin scrollbar-thumb-gray-400 scrollbar-track-gray-100">
  <p>Long content that scrolls...</p>
</div>
```

### 10.3 Custom CSS Approach

```html
<div class="custom-scrollbar h-64 overflow-auto">
  <p>Content here...</p>
</div>

<style>
.custom-scrollbar::-webkit-scrollbar {
  width: 8px;
}

.custom-scrollbar::-webkit-scrollbar-track {
  background: #f1f1f1;
  border-radius: 10px;
}

.custom-scrollbar::-webkit-scrollbar-thumb {
  background: #888;
  border-radius: 10px;
}

.custom-scrollbar::-webkit-scrollbar-thumb:hover {
  background: #555;
}
</style>
```
