## Introduction to Typography Plugin

### What is @tailwindcss/typography?

The Typography plugin (`@tailwindcss/typography`) adds a `prose` class that provides beautiful typographic defaults for HTML you don't control, such as:

- üìù **Markdown content** rendered to HTML
- üì∞ **CMS content** (WordPress, Contentful, Sanity, etc.)
- üìñ **Documentation** sites
- ‚úâÔ∏è **Rich text editors** (TinyMCE, Quill, etc.)
- üìÑ **User-generated content**

### The Problem It Solves

**Without the plugin:**
```html
<!-- Unstyled HTML from markdown/CMS -->
<div>
  <h1>Article Title</h1>
  <p>This paragraph has no styling...</p>
  <ul>
    <li>Unstyled list</li>
    <li>No spacing</li>
  </ul>
</div>

<!-- You'd need to style every element manually -->
<style>
  h1 { font-size: 2.5rem; margin-bottom: 1rem; }
  p { margin-bottom: 1.5rem; line-height: 1.75; }
  ul { list-style: disc; padding-left: 1.5rem; }
  /* ... many more rules ... */
</style>
```

**With the plugin:**
```html
<!-- Just add prose class -->
<article class="prose">
  <h1>Article Title</h1>
  <p>This paragraph automatically has beautiful typography!</p>
  <ul>
    <li>Styled list</li>
    <li>Perfect spacing</li>
  </ul>
</article>

<!-- Everything is styled automatically ‚ú® -->
```

### What Gets Styled Automatically

When you apply the `prose` class, these elements are automatically styled:

| Element | What's Styled |
|---------|--------------|
| **Headings** | h1-h6 with proper sizing, weight, spacing |
| **Paragraphs** | Line height, spacing, color |
| **Links** | Color, hover states, underlines |
| **Lists** | ul, ol, li with proper bullets/numbers and spacing |
| **Blockquotes** | Borders, padding, italic, color |
| **Code** | Inline code styling with background |
| **Pre/Code blocks** | Syntax styling, padding, background |
| **Tables** | Borders, padding, striped rows |
| **Images** | Margins, responsive sizing |
| **HR** | Subtle horizontal rules |
| **Strong/Em** | Bold and italic |
| **Figures** | Captions and spacing |

## Installation & Basic Usage

### Installation

```bash
# npm
npm install -D @tailwindcss/typography

# yarn
yarn add -D @tailwindcss/typography

# pnpm
pnpm add -D @tailwindcss/typography
```

### Configuration

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    // ... your theme config
  },
  plugins: [
    require('@tailwindcss/typography'),
  ],
};
```

### Basic Usage

```html
<!-- Wrap your content in prose -->
<article class="prose">
  <h1>Getting Started with Typography</h1>
  <p>
    The Typography plugin automatically styles all your content.
    You don't need to add any classes to individual elements.
  </p>
  <h2>Features</h2>
  <ul>
    <li>Beautiful typography out of the box</li>
    <li>Works with any HTML source</li>
    <li>Highly customizable</li>
  </ul>
</article>
```

### Typical Use Cases

**1. Blog Post from Markdown:**
```jsx
import { marked } from 'marked';

function BlogPost({ markdown }) {
  const html = marked(markdown);
  
  return (
    <article 
      className="prose lg:prose-xl"
      dangerouslySetInnerHTML={{ __html: html }}
    />
  );
}
```

**2. CMS Content:**
```jsx
function CMSContent({ content }) {
  return (
    <div className="max-w-4xl mx-auto px-4 py-8">
      <article className="prose prose-slate">
        <div dangerouslySetInnerHTML={{ __html: content }} />
      </article>
    </div>
  );
}
```

**3. Documentation:**
```jsx
function DocPage({ doc }) {
  return (
    <div className="prose prose-lg max-w-none">
      <h1>{doc.title}</h1>
      <div dangerouslySetInnerHTML={{ __html: doc.body }} />
    </div>
  );
}
```

## Size Modifiers

The plugin includes five size modifiers that scale all typography proportionally:

### Available Sizes

```html
<!-- Extra Small: Compact content, sidebars -->
<article class="prose prose-sm">
  <h1>Small Typography</h1>
  <p>All elements scale down proportionally.</p>
</article>

<!-- Default (Base): Standard reading size -->
<article class="prose">
  <h1>Default Typography</h1>
  <p>Perfect for most content.</p>
</article>

<!-- Large: Feature articles, blog posts -->
<article class="prose prose-lg">
  <h1>Large Typography</h1>
  <p>More readable for long-form content.</p>
</article>

<!-- Extra Large: Landing pages, important content -->
<article class="prose prose-xl">
  <h1>Extra Large Typography</h1>
  <p>Makes a statement.</p>
</article>

<!-- 2X Large: Hero sections, emphasis -->
<article class="prose prose-2xl">
  <h1>2X Large Typography</h1>
  <p>Maximum impact and readability.</p>
</article>
```

### Size Comparison Table

| Class | Base Font | Line Height | H1 Size | H2 Size | Max Width | Best For |
|-------|-----------|-------------|---------|---------|-----------|----------|
| `prose-sm` | 14px (0.875rem) | 1.714 | 30px | 20px | 576px (36rem) | Sidebars, compact spaces |
| `prose` | 16px (1rem) | 1.75 | 36px | 30px | 65ch | Default, most content |
| `prose-lg` | 18px (1.125rem) | 1.778 | 42px | 36px | 732px (45.75rem) | Feature articles |
| `prose-xl` | 20px (1.25rem) | 1.8 | 48px | 42px | 816px (51rem) | Landing pages |
| `prose-2xl` | 24px (1.5rem) | 1.667 | 60px | 48px | 960px (60rem) | Hero sections |

### Responsive Sizing

Combine size modifiers with breakpoints:

```html
<!-- Small on mobile, large on tablet, xl on desktop -->
<article class="prose prose-sm md:prose-lg lg:prose-xl">
  <h1>Responsive Typography</h1>
  <p>
    This content scales beautifully across all devices. 
    On mobile, it's compact and readable. On tablets, it gets larger. 
    On desktop, it's spacious and impressive.
  </p>
</article>

<!-- Default on mobile, large on desktop -->
<article class="prose lg:prose-lg">
  <h1>Simple Responsive</h1>
  <p>Scales up on large screens only.</p>
</article>
```

## Summary

### Key Takeaways

1. **Installation**: `npm install -D @tailwindcss/typography`
2. **Basic Usage**: Add `prose` class to content containers
3. **Sizes**: Use `prose-sm`, `prose`, `prose-lg`, `prose-xl`, `prose-2xl`
4. **Themes**: Apply color themes like `prose-slate`, `prose-stone`
5. **Dark Mode**: Use `dark:prose-invert` for automatic dark mode
6. **Element Modifiers**: Target specific elements like `prose-h1:`, `prose-a:`
7. **Customization**: Full control via `tailwind.config.js`

### When to Use Typography Plugin

‚úÖ **Perfect for:**
- Blog posts from markdown
- CMS-generated content
- Documentation sites
- User-generated content
- Rich text editor output

‚ùå **Not needed for:**
- Component-based layouts (use regular Tailwind)
- Forms and inputs (too opinionated)
- Marketing pages (need more control)
- When you need pixel-perfect designs

### Pro Tips

1. **Combine with max-width**: `prose max-w-none` for full-width
2. **Responsive sizing**: `prose md:prose-lg lg:prose-xl`
3. **Custom max-width**: Override with `prose prose-lg max-w-4xl`
4. **Not all elements**: Use `not-prose` to exclude elements
5. **Lead paragraphs**: No special class needed, just style with utilities
