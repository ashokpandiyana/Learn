```html
<!-- Tailwind's default breakpoints -->
sm:  640px   <!-- Small devices (landscape phones) -->
md:  768px   <!-- Medium devices (tablets) -->
lg:  1024px  <!-- Large devices (laptops) -->
xl:  1280px  <!-- Extra large (desktops) -->
2xl: 1536px  <!-- 2X large (large desktops) -->
```

```css

/* Base (no prefix) - applies to all */
.text-sm { font-size: 0.875rem; }

/* sm: - 640px and up */
@media (min-width: 640px) {
  .sm\:text-base { font-size: 1rem; }
}

/* md: - 768px and up */
@media (min-width: 768px) {
  .md\:text-lg { font-size: 1.125rem; }
}

/* lg: - 1024px and up */
@media (min-width: 1024px) {
  .lg\:text-xl { font-size: 1.25rem; }
}
```

## 14.3 Using Responsive Prefixes

Apply any utility at specific breakpoints.

### Basic Usage

```html
<!-- Font size changes at each breakpoint -->
<p class="text-sm md:text-base lg:text-lg xl:text-xl">
  Responsive text
</p>

<!-- Padding increases on larger screens -->
<div class="p-4 md:p-6 lg:p-8 xl:p-12">
  Responsive padding
</div>

<!-- Grid columns increase -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4">
  Responsive grid
</div>

<!-- Flex direction changes -->
<div class="flex flex-col md:flex-row">
  Stack on mobile, row on tablet+
</div>

<!-- Show/hide elements -->
<div class="hidden md:block">
  Hidden on mobile, visible on tablet+
</div>
<div class="block md:hidden">
  Visible on mobile, hidden on tablet+
</div>
```

### Common Responsive Patterns

```html
<!-- Responsive container with max-width -->
<div class="w-full max-w-sm md:max-w-md lg:max-w-lg xl:max-w-xl mx-auto">
  Content
</div>

<!-- Responsive spacing -->
<section class="py-8 md:py-12 lg:py-16 xl:py-24">
  <div class="px-4 md:px-6 lg:px-8">
    Content with responsive spacing
  </div>
</section>

<!-- Responsive typography -->
<h1 class="text-2xl sm:text-3xl md:text-4xl lg:text-5xl xl:text-6xl font-bold">
  Responsive Heading
</h1>

<!-- Responsive images -->
<img class="w-full md:w-1/2 lg:w-1/3" src="...">
```

## 14.4 Max-Width Breakpoints

Target screens SMALLER than a breakpoint.

```html
<!-- max-* variants (opposite of regular breakpoints) -->
<div class="max-sm:text-xs">Below 640px only</div>
<div class="max-md:hidden">Below 768px: hidden</div>
<div class="max-lg:flex-col">Below 1024px: column</div>
<div class="max-xl:p-4">Below 1280px: p-4</div>
<div class="max-2xl:grid-cols-3">Below 1536px: 3 cols</div>
```

## 14.5 Breakpoint Ranges

Target specific screen ranges.

```html
<!-- Only between md and lg (768px - 1023px) -->
<div class="md:max-lg:bg-blue-500">
  Blue only on tablets
</div>

<!-- Only on small screens (640px - 767px) -->
<div class="sm:max-md:text-center">
  Centered only on small screens
</div>

<!-- Complex ranges -->
<div class="sm:max-xl:grid-cols-2 xl:grid-cols-4">
  2 columns from sm to xl, then 4 columns
</div>
```

### Range Examples

```html
<!-- Hide sidebar only on tablets -->
<aside class="md:max-lg:hidden">
  Sidebar (hidden 768-1023px)
</aside>

<!-- Different layouts per range -->
<div class="
  grid-cols-1           
  sm:max-md:grid-cols-2  
  md:max-lg:grid-cols-3  
  lg:grid-cols-4         
">
  Progressive grid
</div>
```

### Advanced Breakpoint Syntax

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    screens: {
      // Min-width (default behavior)
      'sm': '640px',
      // or
      'sm': { 'min': '640px' },
      
      // Max-width only
      'mobile': { 'max': '639px' },
      
      // Fixed range
      'tablet': { 'min': '640px', 'max': '1023px' },
      
      // Raw media query
      'tall': { 'raw': '(min-height: 800px)' },
      'portrait': { 'raw': '(orientation: portrait)' },
      'retina': { 'raw': '(-webkit-min-device-pixel-ratio: 2)' },
      'print': { 'raw': 'print' },
      'dark': { 'raw': '(prefers-color-scheme: dark)' },
      
      // Multiple conditions
      'md-tall': { 'raw': '(min-width: 768px) and (min-height: 800px)' },
    },
  },
}
```

### Using Custom Breakpoints

```html
<!-- With custom xs breakpoint -->
<div class="text-xs xs:text-sm sm:text-base">
  Extra small breakpoint
</div>

<!-- With raw media query breakpoint -->
<div class="py-4 tall:py-8">
  More padding on tall screens
</div>

<!-- Print styles -->
<div class="text-black print:text-gray-900 print:bg-white">
  Print-optimized
</div>
```

## 14.7 Container Queries

Respond to container size instead of viewport (v3.2+).

### Setup Container

```html
<!-- Mark element as container -->
<div class="@container">
  <!-- Children can use container queries -->
  <div class="@sm:flex @lg:grid @lg:grid-cols-2">
    Responds to container width
  </div>
</div>

<!-- Named container -->
<div class="@container/main">
  <div class="@sm/main:text-lg">
    Responds to named container
  </div>
</div>
```

### Container Breakpoints

```html
<!-- Container query breakpoints -->
@xs:   160px
@sm:   320px
@md:   384px
@lg:   512px
@xl:   672px
@2xl:  896px
@3xl:  1024px
@4xl:  1152px
@5xl:  1280px
@6xl:  1408px
@7xl:  1536px

<!-- Usage -->
<div class="@container">
  <div class="block @md:flex @xl:grid @xl:grid-cols-3">
    Container-responsive layout
  </div>
</div>

<!-- Arbitrary container sizes -->
<div class="@[400px]:flex">
  Flex when container is 400px+
</div>
```

### Container Query Example

```html
<!-- Card that adapts to its container -->
<div class="@container">
  <article class="@sm:flex @sm:items-center gap-4">
    <img class="w-full @sm:w-32 @sm:h-32 rounded-lg object-cover" src="...">
    <div class="mt-4 @sm:mt-0">
      <h2 class="text-lg @lg:text-xl font-bold">Card Title</h2>
      <p class="text-gray-600 @md:text-lg">Description text...</p>
    </div>
  </article>
</div>
```

## ğŸ’¡ Interview Quick Tips

1. **"What does mobile-first mean in Tailwind?"**
   - Unprefixed = mobile (base)
   - Prefixed = that breakpoint and UP

2. **"How do breakpoints work?"**
   - `md:` = `@media (min-width: 768px)`
   - Applies from that width upward

3. **"Container queries vs media queries?"**
   - Media: Responds to viewport
   - Container: Responds to parent element

4. **"How to hide on mobile only?"**
   - `hidden sm:block` (hidden base, show on sm+)
   - OR `max-sm:hidden` (hidden below sm)

5. **"Custom breakpoints?"**
   - Configure in `theme.screens`
   - Can use min, max, or raw media queries
