# Chapter 2: Configuration Deep Dive

## 2.1 The tailwind.config.js File

The configuration file is the heart of Tailwind. It controls everything from colors to breakpoints to plugins.

### Generate Default Config

```bash
# Basic config (minimal)
npx tailwindcss init

# Full config with all defaults exposed
npx tailwindcss init --full

# With PostCSS config
npx tailwindcss init -p
```

### âš ï¸ CRITICAL: Dynamic Class Names

**The #1 Tailwind mistake**: Using dynamic class names that can't be detected.

```javascript
// â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•
// âŒ WRONG - Tailwind can't detect these classes
// â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

// String interpolation
const color = 'blue'
<div className={`bg-${color}-500`} />
// Tailwind sees: bg-${color}-500 (not a valid class!)

// Template literals with variables
<div className={`text-${size}`} />

// Dynamic class construction
const classes = ['bg', 'blue', '500'].join('-')

// â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•
// âœ… CORRECT - Full class names are detectable
// â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

// Object mapping
const colorClasses = {
  blue: 'bg-blue-500',
  red: 'bg-red-500',
  green: 'bg-green-500',
}
<div className={colorClasses[color]} />

// Conditional with complete classes
<div className={isActive ? 'bg-blue-500' : 'bg-gray-500'} />

// Array of complete classes
const classes = [
  'p-4',
  isLarge && 'text-lg',
  isActive && 'bg-blue-500',
].filter(Boolean).join(' ')
```

## ğŸ’¡ Interview Quick Tips

1. **"What happens if content paths are wrong?"**
   - Utilities won't be generated
   - Classes appear to not work
   - Production CSS will be nearly empty

2. **"theme vs theme.extend?"**
   - `theme`: Replaces ALL defaults
   - `theme.extend`: Adds to defaults (safer)

3. **"How do you handle dynamic classes?"**
   - Use complete class names in code
   - Use safelist for truly dynamic needs
   - Object/array mapping pattern

4. **"When would you disable preflight?"**
   - Legacy CSS integration
   - Email templates
   - Third-party widgets
