## 20.3 Avoiding Dynamic Class Name Issues

### The Problem

```javascript
// âŒ WRONG - Tailwind can't detect these classes!

// String interpolation
const color = 'blue';
<div className={`bg-${color}-500`} />
// Tailwind sees: bg-${color}-500 (not valid)

// Dynamic construction
const intensity = 500;
<div className={`text-red-${intensity}`} />

// Array joining
const classes = ['bg', props.color, '500'].join('-');
```

### The Solution

```javascript
// âœ… CORRECT - Full class names are detectable

// Object mapping
const colorClasses = {
  blue: 'bg-blue-500',
  red: 'bg-red-500',
  green: 'bg-green-500',
};
<div className={colorClasses[color]} />

// Conditional complete classes
<div className={isActive ? 'bg-blue-500' : 'bg-gray-500'} />

// clsx/classnames with complete classes
import clsx from 'clsx';
<div className={clsx(
  'p-4 rounded',
  isError && 'bg-red-500',
  isSuccess && 'bg-green-500',
)} />

// Object of all possible classes
const bgColors = {
  primary: 'bg-blue-500 hover:bg-blue-600',
  secondary: 'bg-gray-500 hover:bg-gray-600',
  danger: 'bg-red-500 hover:bg-red-600',
};
<button className={bgColors[variant]} />
```

## 20.5 Blocklist (Exclude Classes)

Remove unwanted utilities.

```javascript
// tailwind.config.js
module.exports = {
  blocklist: [
    'container',        // Disable container utility
    'blur',            // Disable blur if not needed
    'grayscale',       // Disable grayscale
  ],
};
```

## 20.6 Disabling Core Plugins

Remove entire categories of utilities you don't need.

```javascript
// tailwind.config.js
module.exports = {
  corePlugins: {
    // Disable unused utilities
    float: false,           // If not using floats
    clear: false,           // If not using clear
    objectFit: false,       // If not styling images
    objectPosition: false,
    
    // Disable if not using these features
    container: false,       // If using custom container
    animation: false,       // If using CSS animations directly
    
    // Be careful with these
    preflight: false,       // Only disable if you have another reset
  },
};

// Or use array to enable ONLY specific plugins
module.exports = {
  corePlugins: [
    'margin',
    'padding',
    'backgroundColor',
    'textColor',
    'fontSize',
    'fontWeight',
    'borderRadius',
    // Only these plugins will be available
  ],
};
```

## 20.7 CSS Optimization

### PostCSS Configuration

```javascript
// postcss.config.js
module.exports = {
  plugins: {
    'tailwindcss': {},
    'autoprefixer': {},
    // Production optimizations
    ...(process.env.NODE_ENV === 'production' ? {
      'cssnano': {
        preset: ['default', {
          discardComments: { removeAll: true },
          normalizeWhitespace: true,
        }],
      },
    } : {}),
  },
};
```

### Using LightningCSS (Faster Alternative)

```javascript
// vite.config.js
import { defineConfig } from 'vite';

export default defineConfig({
  css: {
    transformer: 'lightningcss',
    lightningcss: {
      drafts: {
        customMedia: true,
      },
    },
  },
  build: {
    cssMinify: 'lightningcss',
  },
});
```

### Next.js Configuration

```javascript
// next.config.js
module.exports = {
  experimental: {
    optimizeCss: true, // Enable CSS optimization
  },
};
```

## 20.8 Build Optimization Tips

### 1. Limit Content Paths

```javascript
// âŒ Slow - scans too much
content: ['./**/*.{js,jsx,ts,tsx}']

// âœ… Fast - specific directories
content: [
  './src/components/**/*.{js,jsx,ts,tsx}',
  './src/pages/**/*.{js,jsx,ts,tsx}',
]
```

### 2. Use Specific Extensions

```javascript
// âŒ Slower - checks all files
content: ['./src/**/*']

// âœ… Faster - specific extensions
content: ['./src/**/*.{js,jsx,ts,tsx}']
```

### 3. Exclude Build Outputs

```javascript
// Ignore output directories
content: [
  './src/**/*.{js,jsx}',
  '!./src/**/*.test.{js,jsx}',  // Exclude tests
  '!./src/**/*.spec.{js,jsx}',
  '!./src/**/*.stories.{js,jsx}', // Exclude Storybook
]
```

### 4. Parallel Builds

Most build tools parallelize automatically. Ensure you're using latest versions.

## 20.10 Caching Strategies

### Long-Term Caching

```javascript
// vite.config.js
export default {
  build: {
    rollupOptions: {
      output: {
        // Hash-based filenames for caching
        assetFileNames: 'assets/[name]-[hash][extname]',
      },
    },
  },
};
```

### CDN Configuration

```nginx
# nginx.conf
location ~* \.(css|js)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

### Critical CSS

```javascript
// Extract critical CSS for above-the-fold content
// Use tools like:
// - critters (for Vite/Webpack)
// - critical (standalone)

// vite.config.js
import critters from 'critters';

export default {
  plugins: [
    critters({
      preload: 'swap',
    }),
  ],
};
```

## ğŸ’¡ Interview Quick Tips

1. **"How does Tailwind handle unused CSS?"**
   - JIT mode scans content files
   - Only generates CSS for used classes
   - No purging step needed (it's built-in)

2. **"Why can't you use dynamic class names?"**
   - Tailwind scans files as text, not runtime
   - Can't detect `bg-${color}-500`
   - Use object mapping or safelist instead

3. **"What's a typical production CSS size?"**
   - 10-50KB uncompressed
   - 5-15KB gzipped
   - If larger, check configuration

4. **"How to optimize build performance?"**
   - Specific content paths
   - Specific file extensions
   - Disable unused core plugins

5. **"What is safelist for?"**
   - Force-include classes Tailwind can't detect
   - CMS content, user-configurable themes
   - Use sparingly - increases bundle size
