# Chapter 1: React Basics - In-Depth Explanation

## 1.1 Introduction to React

### What is React?

React is a **JavaScript library** (not a framework) for building user interfaces, particularly single-page applications (SPAs). Created by Facebook in 2013, it's now maintained by Meta and a large open-source community.

**Key Characteristics:**
- **Declarative**: You describe what the UI should look like, React handles the "how"
- **Component-Based**: Build encapsulated components that manage their own state
- **Learn Once, Write Anywhere**: Can render on server, mobile (React Native), VR, etc.

### Why Use React?

**1. Declarative Programming**
```javascript
// Traditional JavaScript (Imperative)
const button = document.createElement('button');
button.textContent = 'Click me';
button.onclick = () => alert('Clicked!');
document.body.appendChild(button);

// React (Declarative)
function App() {
  return <button onClick={() => alert('Clicked!')}>Click me</button>;
}
```

**2. Component Reusability**
```javascript
// Create once, use everywhere
function Button({ text, onClick }) {
  return <button onClick={onClick}>{text}</button>;
}

// Use multiple times
<Button text="Submit" onClick={handleSubmit} />
<Button text="Cancel" onClick={handleCancel} />
<Button text="Delete" onClick={handleDelete} />
```

**3. Efficient Updates with Virtual DOM**

### Virtual DOM Concept

The Virtual DOM is a lightweight copy of the actual DOM kept in memory.

**How it Works:**

```
1. Initial Render:
   React Component ‚Üí Virtual DOM ‚Üí Real DOM

2. State Changes:
   Update State ‚Üí New Virtual DOM Created
   
3. Reconciliation (Diffing):
   Old Virtual DOM ‚ö° New Virtual DOM ‚Üí Calculate Changes
   
4. Efficient Update:
   Only Changed Elements ‚Üí Real DOM Updated
```

**Visual Representation:**
```
Real DOM (Expensive):
‚îî‚îÄ‚îÄ <div id="root">
    ‚îú‚îÄ‚îÄ <h1>Hello</h1>        ‚Üê Change this
    ‚îú‚îÄ‚îÄ <p>World</p>
    ‚îî‚îÄ‚îÄ <button>Click</button>

Virtual DOM (Fast):
‚îî‚îÄ‚îÄ {type: 'div', props: {...}}
    ‚îú‚îÄ‚îÄ {type: 'h1', children: 'Hello'}  ‚Üê Detect change
    ‚îú‚îÄ‚îÄ {type: 'p', children: 'World'}
    ‚îî‚îÄ‚îÄ {type: 'button', children: 'Click'}

Result: Only <h1> is updated in Real DOM, not entire tree!
```

**Example:**
```javascript
function Counter() {
  const [count, setCount] = useState(0);
  
  // Only the text content of <h1> updates, not the entire component
  return (
    <div>
      <h1>{count}</h1>  {/* Only this updates */}
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <p>This paragraph doesn't re-render in the DOM</p>
    </div>
  );
}
```

### React vs Other Frameworks

| Feature | React | Angular | Vue |
|---------|-------|---------|-----|
| Type | Library | Framework | Framework |
| Learning Curve | Moderate | Steep | Easy |
| Size | ~42KB | ~500KB | ~33KB |
| Architecture | Components | MVC | Components |
| Language | JavaScript/JSX | TypeScript | JavaScript |
| Data Binding | One-way | Two-way | Two-way |

**Important Note:** React is a **library**, not a framework. This means:

- More flexibility but requires more decisions
- Smaller core, but ecosystem is vast

## 1.3 JSX (JavaScript XML)

### What is JSX?

JSX is a syntax extension for JavaScript that looks like HTML but isn't.

```javascript
// This is JSX
const element = <h1>Hello, World!</h1>;

// JSX is NOT a string
const wrong = "<h1>Hello, World!</h1>";  // This is just a string!

// JSX is NOT HTML
// It compiles to JavaScript function calls
```

### JSX Under the Hood

**What Actually Happens:**

```javascript
// You write:
const element = <h1 className="greeting">Hello, World!</h1>;

// Babel transforms it to:
const element = React.createElement(
  'h1',
  { className: 'greeting' },
  'Hello, World!'
);

// Which creates an object (Virtual DOM node):
const element = {
  type: 'h1',
  props: {
    className: 'greeting',
    children: 'Hello, World!'
  }
};
```

**Important:** JSX is syntactic sugar for `React.createElement()`

### JSX Syntax Rules

#### 1. Single Root Element

```javascript
// ‚ùå Wrong: Multiple root elements
function App() {
  return (
    <h1>Hello</h1>
    <p>World</p>
  );
}

// ‚úÖ Correct: Wrapped in single element
function App() {
  return (
    <div>
      <h1>Hello</h1>
      <p>World</p>
    </div>
  );
}

// ‚úÖ Better: Use Fragment
function App() {
  return (
    <>
      <h1>Hello</h1>
      <p>World</p>
    </>
  );
}
```

#### 2. JavaScript Expressions in JSX

Use curly braces `{}` to embed JavaScript:

```javascript
function Greeting() {
  const name = 'John';
  const age = 25;
  const isLoggedIn = true;
  
  return (
    <div>
      {/* Variables */}
      <h1>Hello, {name}!</h1>
      
      {/* Expressions */}
      <p>You are {age} years old</p>
      <p>Next year you'll be {age + 1}</p>
      
      {/* Function calls */}
      <p>{name.toUpperCase()}</p>
      
      {/* Ternary operators */}
      <p>{isLoggedIn ? 'Welcome back!' : 'Please log in'}</p>
      
      {/* Logical AND */}
      {isLoggedIn && <button>Logout</button>}
    </div>
  );
}
```

**What You CAN'T Put in {}:**

```javascript
function Invalid() {
  return (
    <div>
      {/* ‚ùå Can't use if statements */}
      {if (true) { return <p>Hi</p> }}
      
      {/* ‚ùå Can't use for loops */}
      {for (let i = 0; i < 5; i++) { <p>{i}</p> }}
      
      {/* ‚ùå Can't use objects directly */}
      {{ name: 'John', age: 25 }}
      
      {/* ‚úÖ Use ternary instead of if */}
      {true ? <p>Hi</p> : null}
      
      {/* ‚úÖ Use map instead of for */}
      {[1, 2, 3].map(num => <p key={num}>{num}</p>)}
      
      {/* ‚úÖ Access object properties */}
      {person.name}
    </div>
  );
}
```

#### 3. JSX vs HTML Differences

**Attribute Names (camelCase):**

```javascript
// HTML vs JSX
<div class="container">           // HTML
<div className="container">       // JSX

<label for="name">                // HTML
<label htmlFor="name">            // JSX

<button onclick="handleClick()">  // HTML
<button onClick={handleClick}>    // JSX

<div tabindex="0">                // HTML
<div tabIndex="0">                // JSX
```

**Style Attribute:**

```javascript
// HTML - string
<div style="color: red; font-size: 16px;">

// JSX - object with camelCase properties
<div style={{ color: 'red', fontSize: '16px' }}>
<div style={{ color: 'red', fontSize: 16 }}>  // Numbers are pixels

// Better: Extract to variable
const styles = {
  color: 'red',
  fontSize: 16,
  backgroundColor: 'lightblue'  // Note: camelCase!
};
<div style={styles}>
```

**Self-Closing Tags:**

```javascript
// HTML - optional closing
<img src="photo.jpg">
<input type="text">
<br>

// JSX - MUST self-close
<img src="photo.jpg" />
<input type="text" />
<br />
```

**Reserved Words:**

```javascript
// ‚ùå Can't use 'class' (JavaScript reserved word)
<div class="container">

// ‚úÖ Use 'className'
<div className="container">

// ‚ùå Can't use 'for'
<label for="name">

// ‚úÖ Use 'htmlFor'
<label htmlFor="name">
```

#### 4. Comments in JSX

```javascript
function App() {
  return (
    <div>
      {/* This is a comment in JSX */}
      
      {/* 
        Multi-line
        comment
      */}
      
      <h1>Hello</h1>
      
      {/* Can't use // or regular comments inside JSX */}
      
      {
        // This works because it's inside {}
        // but it's not recommended
      }
    </div>
  );
}
```

## 1.4 Components

### What are Components?

Components are **independent, reusable pieces of UI**. Think of them as custom HTML elements.

```javascript
// Before components - repetitive HTML
<div>
  <button class="btn primary">Submit</button>
  <button class="btn secondary">Cancel</button>
  <button class="btn danger">Delete</button>
</div>

// With components - reusable and maintainable
<div>
  <Button type="primary">Submit</Button>
  <Button type="secondary">Cancel</Button>
  <Button type="danger">Delete</Button>
</div>
```

### Functional Components

Modern React uses functional components (hooks-based).

**Basic Syntax:**

```javascript
// Method 1: Function declaration
function Welcome() {
  return <h1>Hello, World!</h1>;
}

// Method 2: Arrow function
const Welcome = () => {
  return <h1>Hello, World!</h1>;
};

// Method 3: Arrow function with implicit return
const Welcome = () => <h1>Hello, World!</h1>;

// Method 4: Multi-line JSX with implicit return
const Welcome = () => (
  <div>
    <h1>Hello, World!</h1>
    <p>Welcome to React!</p>
  </div>
);
```

**Component Rules:**

1. **Name must start with capital letter** (React convention)
```javascript
// ‚úÖ Correct
function Button() { }
const Card = () => { };

// ‚ùå Wrong - treated as HTML element
function button() { }  // React looks for <button> HTML tag
const card = () => { }; // React looks for <card> HTML tag
```

2. **Must return JSX or null**
```javascript
// ‚úÖ Correct
function Component1() {
  return <div>Content</div>;
}

function Component2() {
  return null;  // Valid: renders nothing
}

// ‚ùå Wrong
function Component3() {
  // No return statement
  <div>Content</div>;
}

function Component4() {
  return undefined;  // ‚ùå Invalid
}
```

3. **Can contain logic before return**
```javascript
function Greeting() {
  // Variables
  const name = 'John';
  
  // Logic
  const hour = new Date().getHours();
  const greeting = hour < 12 ? 'Good morning' : 'Good evening';
  
  // Helper functions
  const formatName = (name) => name.toUpperCase();
  
  // Return JSX
  return (
    <div>
      <h1>{greeting}, {formatName(name)}!</h1>
    </div>
  );
}
```

### Class Components (Legacy but Important)

Class components are older but still appear in interviews and legacy code.

**Basic Syntax:**

```javascript
import React, { Component } from 'react';

class Welcome extends Component {
  render() {
    return <h1>Hello, World!</h1>;
  }
}

// With state and methods
class Counter extends Component {
  // Constructor for state
  constructor(props) {
    super(props);  // MUST call super(props)
    this.state = {
      count: 0
    };
    
    // Bind methods (or use arrow functions)
    this.increment = this.increment.bind(this);
  }
  
  // Method
  increment() {
    this.setState({ count: this.state.count + 1 });
  }
  
  // Lifecycle method
  componentDidMount() {
    console.log('Component mounted');
  }
  
  // render() is required
  render() {
    return (
      <div>
        <h1>Count: {this.state.count}</h1>
        <button onClick={this.increment}>Increment</button>
      </div>
    );
  }
}
```

**Modern Alternative (Functional with Hooks):**

```javascript
import { useState, useEffect } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  
  const increment = () => {
    setCount(count + 1);
  };
  
  useEffect(() => {
    console.log('Component mounted');
  }, []);
  
  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={increment}>Increment</button>
    </div>
  );
}
```

**Highlight:** Modern React heavily favors functional components. Learn class components for interviews, but use functional components in practice.

### Component Composition

Building complex UIs by combining simple components.

**Example: Building a Blog Post**

```javascript
// Small, focused components
function Avatar({ src, alt }) {
  return <img src={src} alt={alt} className="avatar" />;
}

function Author({ name, avatar }) {
  return (
    <div className="author">
      <Avatar src={avatar} alt={name} />
      <span>{name}</span>
    </div>
  );
}

function PostMeta({ author, date, readTime }) {
  return (
    <div className="post-meta">
      <Author name={author.name} avatar={author.avatar} />
      <span>{date}</span>
      <span>{readTime} min read</span>
    </div>
  );
}

function PostContent({ title, excerpt }) {
  return (
    <div className="post-content">
      <h2>{title}</h2>
      <p>{excerpt}</p>
    </div>
  );
}

// Compose into larger component
function BlogPost({ post }) {
  return (
    <article className="blog-post">
      <PostMeta 
        author={post.author}
        date={post.date}
        readTime={post.readTime}
      />
      <PostContent 
        title={post.title}
        excerpt={post.excerpt}
      />
    </article>
  );
}

// Usage
function App() {
  const post = {
    author: {
      name: 'Jane Doe',
      avatar: 'https://via.placeholder.com/50'
    },
    date: 'Nov 26, 2025',
    readTime: 5,
    title: 'Learning React',
    excerpt: 'React is a powerful library for building UIs...'
  };
  
  return <BlogPost post={post} />;
}
```

**Benefits of Composition:**
- **Reusability**: Use `Avatar` in comments, profile pages, etc.
- **Maintainability**: Each component has one job
- **Testability**: Test small components independently
- **Readability**: Clear component hierarchy

    <button 
      className={`btn ${BUTTON_SIZES[size]}`}
      onClick={onClick}
      onMouseEnter={() => setIsHovered(true)}
      onMouseLeave={() => setIsHovered(false)}
    >
      {children}
    </button>
  );
}

export default Button;

// ProductCard.jsx
import { useState } from 'react';
import './ProductCard.css';

function ProductCard({ product }) {
  const [quantity, setQuantity] = useState(1);
  const [isInCart, setIsInCart] = useState(false);
  
  // Helper functions
  const formatPrice = (price) => {
    return `$${price.toFixed(2)}`;
  };
  
  const calculateTotal = () => {
    return product.price * quantity;
  };
  
  // Event handlers
  const handleAddToCart = () => {
    console.log(`Added ${quantity} ${product.name} to cart`);
    setIsInCart(true);
  };
  
  const handleQuantityChange = (change) => {
    const newQuantity = quantity + change;
    if (newQuantity >= 1 && newQuantity <= product.stock) {
      setQuantity(newQuantity);
    }
  };
  
  // Conditional rendering
  const renderStock = () => {
    if (product.stock === 0) {
      return <span className="out-of-stock">Out of Stock</span>;
    } else if (product.stock < 10) {
      return <span className="low-stock">Only {product.stock} left!</span>;
    }
    return <span className="in-stock">In Stock</span>;
  };
  
  // Main render
  return (
    <div className="product-card">
      {/* Image */}
      <img 
        src={product.image} 
        alt={product.name}
        className="product-image"
      />
      
      {/* Product info */}
      <div className="product-info">
        <h3>{product.name}</h3>
        <p className="description">{product.description}</p>
        
        {/* Stock status */}
        {renderStock()}
        
        {/* Price */}
        <div className="price-section">
          <span className="price">{formatPrice(product.price)}</span>
          {quantity > 1 && (
            <span className="total">
              Total: {formatPrice(calculateTotal())}
            </span>
          )}
        </div>
        
        {/* Quantity controls */}
        <div className="quantity-controls">
          <button 
            onClick={() => handleQuantityChange(-1)}
            disabled={quantity === 1}
          >
            -
          </button>
          <span>{quantity}</span>
          <button 
            onClick={() => handleQuantityChange(1)}
            disabled={quantity === product.stock}
          >
            +
          </button>
        </div>
        
        {/* Add to cart button */}
        <button 
          className={`add-to-cart ${isInCart ? 'added' : ''}`}
          onClick={handleAddToCart}
          disabled={product.stock === 0}
        >
          {isInCart ? '‚úì Added to Cart' : 'Add to Cart'}
        </button>
      </div>
    </div>
  );
}

export default ProductCard;
