üéØ Why Monitoring Matters

GraphQL applications have unique monitoring challenges compared to REST. Each query can have vastly different performance characteristics, making per-operation metrics essential.

Key Metrics to Track

  • Query duration: How long each operation takes
  • Error rates: Both GraphQL and network errors
  • Cache performance: Hit/miss ratio
  • Query complexity: Identify expensive queries
  • Field usage: Which fields are actually used
  • Client distribution: Which apps/versions make requests

links/performanceLink.ts
import { ApolloLink, Operation } from '@apollo/client'; // Define the shape of our metrics interface QueryMetrics { operationName: string; operationType: string; duration: number; timestamp: string; variables: Record<string, any>; cacheHit: boolean; status: 'success' | 'error'; errorMessage?: string; } // Send metrics to your analytics service async function sendMetrics(metrics: QueryMetrics) { // Option 1: Send to custom analytics endpoint await fetch('/api/analytics/graphql', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(metrics), }); // Option 2: Send to third-party service analytics.track('graphql_operation', metrics); // Option 3: Log to console in development if (process.env.NODE_ENV === 'development') { console.log(`[GraphQL] ${metrics.operationName}: ${metrics.duration}ms`); } } // Extract operation type from query function getOperationType(operation: Operation): string { const definition = operation.query.definitions[0]; if (definition.kind === 'OperationDefinition') { return definition.operation; } return 'unknown'; } export const performanceLink = new ApolloLink((operation, forward) => { // Record start time before request const startTime = performance.now(); // Add unique request ID for tracing operation.setContext(({ headers = {} }) => ({ headers: { ...headers, 'x-request-id': `${Date.now()}-${Math.random().toString(36).slice(2)}`, }, })); return forward(operation).map(response => { const duration = performance.now() - startTime; // Build metrics object const metrics: QueryMetrics = { operationName: operation.operationName || 'anonymous', operationType: getOperationType(operation), duration: Math.round(duration), timestamp: new Date().toISOString(), variables: operation.variables, cacheHit: response.data && !response.errors, status: response.errors ? 'error' : 'success', errorMessage: response.errors?.[0]?.message, }; // Send asynchronously (don't block response) sendMetrics(metrics).catch(console.error); // Log slow queries if (duration > 1000) { console.warn(`[Slow Query] ${metrics.operationName}: ${duration}ms`); } return response; }); });

Cache Hit Rate Tracking

import { ApolloLink } from '@apollo/client'; // Track cache performance let cacheHits = 0; let cacheMisses = 0; export const cacheTrackingLink = new ApolloLink((operation, forward) => { return forward(operation).map(response => { // Check if response came from cache const context = operation.getContext(); if (context.response?.fromCache) { cacheHits++; } else { cacheMisses++; } // Calculate and log hit rate periodically const total = cacheHits + cacheMisses; if (total % 100 === 0) { const hitRate = ((cacheHits / total) * 100).toFixed(1); console.log(`[Cache] Hit rate: ${hitRate}% (${cacheHits}/${total})`); } return response; }); }); // Export function to get current stats export function getCacheStats() { const total = cacheHits + cacheMisses; return { hits: cacheHits, misses: cacheMisses, total, hitRate: total > 0 ? (cacheHits / total) * 100 : 0, }; }

üí° Performance Optimization Tips

  • Set thresholds: Alert when queries exceed 1-2 seconds
  • Track p95/p99: Average hides outliers; use percentiles
  • Segment by operation: Different queries have different norms
  • Sample in production: Don't track every request at scale

üö® 16.3 Error Tracking Integration

Integrate with error tracking services like Sentry, Datadog, or LogRocket to capture and analyze GraphQL errors in production.

Sentry Integration

links/sentryLink.ts
import { onError } from '@apollo/client/link/error'; import * as Sentry from '@sentry/react'; export const sentryLink = onError(({ graphQLErrors, networkError, operation }) => { // Extract operation details for context const operationContext = { operationName: operation.operationName, variables: operation.variables, // Sanitize sensitive data before sending query: operation.query.loc?.source.body.slice(0, 500), }; // Handle GraphQL Errors if (graphQLErrors) { graphQLErrors.forEach((error) => { // Set error severity based on code const severity = error.extensions?.code === 'INTERNAL_SERVER_ERROR' ? 'error' : 'warning'; Sentry.withScope((scope) => { // Add context scope.setTag('graphql.operation', operation.operationName); scope.setTag('graphql.errorCode', error.extensions?.code); scope.setLevel(severity); // Add breadcrumb for debugging scope.addBreadcrumb({ category: 'graphql', message: `GraphQL Error: ${error.message}`, level: severity, data: operationContext, }); // Capture the error Sentry.captureException( new Error(`GraphQL Error: ${error.message}`), { extra: { ...operationContext, path: error.path, locations: error.locations, }, fingerprint: [ 'graphql', operation.operationName, error.extensions?.code || 'UNKNOWN', ], } ); }); }); } // Handle Network Errors if (networkError) { Sentry.withScope((scope) => { scope.setTag('graphql.operation', operation.operationName); scope.setTag('error.type', 'network'); scope.setLevel('error'); Sentry.captureException(networkError, { extra: { ...operationContext, statusCode: (networkError as any).statusCode, }, fingerprint: [ 'graphql', 'network_error', operation.operationName, ], }); }); } });

Complete Client Setup with All Links

lib/apollo-client.ts
import { ApolloClient, InMemoryCache, HttpLink, from } from '@apollo/client'; import { performanceLink } from './links/performanceLink'; import { sentryLink } from './links/sentryLink'; import { authLink } from './links/authLink'; import { retryLink } from './links/retryLink'; const httpLink = new HttpLink({ uri: process.env.NEXT_PUBLIC_GRAPHQL_URL, }); // Link order matters! // 1. Error tracking (captures all errors) // 2. Performance tracking (measures total time) // 3. Auth (adds headers) // 4. Retry (retries failed requests) // 5. HTTP (makes the request) export const client = new ApolloClient({ link: from([ sentryLink, performanceLink, authLink, retryLink, httpLink, ]), cache: new InMemoryCache(), name: 'my-app', version: process.env.NEXT_PUBLIC_APP_VERSION, });

‚ö†Ô∏è Production Monitoring Best Practices

  • Sample strategically: Track 100% of errors, but sample successful requests
  • Sanitize data: Never send passwords, tokens, or PII to monitoring services
  • Set up alerts: Get notified when error rates or latency spike
  • Use fingerprinting: Group similar errors together
  • Track trends: Compare week-over-week performance
