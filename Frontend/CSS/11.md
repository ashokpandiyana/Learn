# Chapter 11: Modern CSS Features - In-Depth Explanation

## Introduction to Modern CSS

Modern CSS provides powerful features that make code more maintainable, flexible, and expressive. These features are essential for production-grade CSS.

**What Makes These "Modern":**
- Introduced in recent CSS specifications
- Solve real-world problems elegantly
- Reduce need for preprocessors
- Widely supported in modern browsers

---

## 11.1 Custom Properties (CSS Variables)

**CSS Custom Properties** let you store values and reuse them throughout your CSS. **Game-changing for theming and maintainability!**

### Declaration and Usage

```css
/* Declare with -- prefix */
:root {
    --primary-color: #667eea;
    --spacing-unit: 8px;
    --border-radius: 4px;
}

/* Use with var() function */
.button {
    background: var(--primary-color);
    padding: var(--spacing-unit);
    border-radius: var(--border-radius);
}
```

**Key Points:**
- **Declare**: Use `--` prefix (e.g., `--my-variable`)
- **Use**: Use `var()` function (e.g., `var(--my-variable)`)
- **Scope**: Variables cascade and inherit like normal CSS
- **Case-sensitive**: `--myVar` ≠ `--myvar`

---

### Scope and Cascade

Custom properties follow normal CSS cascade and inheritance rules.

```css
/* Global scope (available everywhere) */
:root {
    --primary: blue;
}

/* Component scope */
.card {
    --card-bg: white;
    --card-padding: 20px;
}

.card-content {
    background: var(--card-bg);    /* Inherits from .card */
    padding: var(--card-padding);
}

/* Local scope override */
.card.dark {
    --card-bg: #1a1a1a;  /* Override for dark cards */
}
```

**Visual Scope Example:**

```html
<div style="--color: red;">
    <p style="color: var(--color);">Red (from parent)</p>
    <div style="--color: blue;">
        <p style="color: var(--color);">Blue (local override)</p>
    </div>
</div>
```

**Cascade Behavior:**
```css
:root {
    --spacing: 16px;  /* Default */
}

.container {
    --spacing: 24px;  /* Override for .container and descendants */
}

.item {
    margin: var(--spacing);  /* Uses 24px inside .container, 16px elsewhere */
}
```

---

### Fallback Values

Provide fallback when variable is not defined.

```css
.element {
    /* Syntax: var(--variable-name, fallback-value) */
    color: var(--text-color, black);
    
    /* Multiple fallbacks */
    color: var(--primary-color, var(--secondary-color, blue));
    
    /* Fallback with calculation */
    padding: var(--spacing, 1rem);
}
```

**Examples:**
```css
/* Safe defaults */
.button {
    background: var(--button-bg, #007bff);
    color: var(--button-text, white);
}

/* Nested fallbacks */
.text {
    color: var(--text-primary, var(--text-default, #333));
}

/* Calculation fallback */
.box {
    width: var(--box-width, calc(100% - 40px));
}
```

### Dynamic Manipulation with JavaScript

**Powerful feature:** Change CSS variables at runtime with JavaScript!

```javascript
// Get custom property value
const root = document.documentElement;
const primaryColor = getComputedStyle(root)
    .getPropertyValue('--primary-color');

// Set custom property value
root.style.setProperty('--primary-color', '#ff0000');

// Set on specific element
element.style.setProperty('--local-var', '20px');

// Remove custom property
root.style.removeProperty('--primary-color');
```

**Real-World Example:**
```css
/* CSS */
:root {
    --theme-color: #667eea;
}

.button {
    background: var(--theme-color);
}
```

```javascript
// JavaScript - Theme switcher
function setTheme(color) {
    document.documentElement.style.setProperty('--theme-color', color);
}

// Usage
setTheme('#e74c3c');  // Changes all buttons to red!
```

## 11.2 CSS Functions

### calc() - Mathematical Calculations

Perform calculations with different units.

```css
/* Basic arithmetic */
.element {
    width: calc(100% - 50px);
    height: calc(100vh - 80px);
    margin: calc(1rem + 10px);
    padding: calc(2em * 1.5);
}

/* Operators: + - * / */
.box {
    width: calc(100% / 3);           /* Division */
    padding: calc(10px + 2rem);      /* Addition */
    margin: calc(50px - 10px);       /* Subtraction */
    font-size: calc(16px * 1.5);     /* Multiplication */
}
```

**Important Rules:**
- `+` and `-` must have **spaces** on both sides: `calc(100% - 20px)` ✅, `calc(100%-20px)` ❌
- `*` and `/` don't require spaces (but recommended)
- Can't divide by zero
- Can mix units freely

**Common Patterns:**

```css
/* Full height minus header */
.main-content {
    height: calc(100vh - 80px);
}

/* Responsive grid with gap simulation */
.column {
    width: calc((100% - 60px) / 4);  /* 4 columns with 20px gaps */
}

/* Centered with offset */
.element {
    width: calc(100% - 40px);
    margin: 0 auto;
}

/* Font size with viewport scaling */
.heading {
    font-size: calc(1rem + 2vw);
}

/* Dynamic spacing */
.section {
    padding: calc(2rem + 3vh);
}

/* Aspect ratio container */
.video-container {
    height: 0;
    padding-bottom: calc(9 / 16 * 100%);  /* 16:9 ratio */
}
```

**With CSS Variables:**
```css
:root {
    --base-spacing: 8px;
    --sidebar-width: 250px;
}

.element {
    margin: calc(var(--base-spacing) * 3);
    width: calc(100% - var(--sidebar-width) - 40px);
}
```

### min() - Minimum Value

Returns the **smallest value** from a list.

```css
/* Responsive width with maximum */
.element {
    width: min(100%, 800px);
    /* Never wider than 800px, scales down on small screens */
}

/* Multiple values */
.element {
    padding: min(5vw, 50px, 3rem);
    /* Smallest of: 5vw, 50px, or 3rem */
}
```

**Use Cases:**

```css
/* Responsive container */
.container {
    width: min(90%, 1200px);
    /* 90% width on small screens, max 1200px */
}

/* Responsive padding */
.section {
    padding: min(10vw, 100px);
    /* Grows with viewport but caps at 100px */
}

/* Font size with maximum */
.heading {
    font-size: min(10vw, 60px);
    /* Scales but never larger than 60px */
}
```

---

### max() - Maximum Value

Returns the **largest value** from a list.

```css
/* Minimum width that grows */
.element {
    width: max(50%, 300px);
    /* At least 300px, grows beyond 50% when container > 600px */
}

/* Multiple values */
.element {
    padding: max(2vw, 20px, 1rem);
    /* Largest of: 2vw, 20px, or 1rem */
}
```

**Use Cases:**

```css
/* Minimum size */
.button {
    min-width: max(150px, 20vw);
    /* At least 150px, but grows with viewport */
}

/* Touch target size (accessibility) */
.clickable {
    min-height: max(44px, 3rem);
    /* At least 44px for touch targets */
}

/* Responsive font with minimum */
.text {
    font-size: max(16px, 1rem);
    /* Never smaller than 16px */
}
```

---

### clamp() - Range with Min and Max

**Most powerful!** Constrains value between minimum and maximum.

```css
/* Syntax: clamp(min, preferred, max) */
.element {
    font-size: clamp(16px, 4vw, 32px);
    /* Min: 16px, Preferred: 4vw, Max: 32px */
}

/* Width example */
.container {
    width: clamp(300px, 50%, 1200px);
    /* Min 300px, preferred 50%, max 1200px */
}
```

**How It Works:**
1. If `preferred < min`: Uses min
2. If `min < preferred < max`: Uses preferred
3. If `preferred > max`: Uses max

**Visual Example:**

```
Container width: 400px
clamp(300px, 50%, 1200px)

Preferred: 50% of 400px = 200px
Min: 300px
200px < 300px → Uses 300px ✓

Container width: 1600px
Preferred: 50% of 1600px = 800px
Min: 300px, Max: 1200px
300px < 800px < 1200px → Uses 800px ✓

Container width: 3000px
Preferred: 50% of 3000px = 1500px
Max: 1200px
1500px > 1200px → Uses 1200px ✓
```

**Common Patterns:**

```css
/* 1. Fluid typography (best use case!) */
h1 {
    font-size: clamp(2rem, 5vw, 4rem);
    /* Scales between 2rem and 4rem based on viewport */
}

body {
    font-size: clamp(14px, 2.5vw, 18px);
}

/* 2. Responsive container */
.container {
    width: clamp(320px, 90%, 1200px);
    padding: clamp(1rem, 5vw, 3rem);
}

/* 3. Responsive spacing */
.section {
    margin-bottom: clamp(2rem, 5vh, 5rem);
    gap: clamp(1rem, 3vw, 2rem);
}

/* 4. Line length for readability */
.article {
    max-width: clamp(45ch, 65ch, 75ch);
}

/* 5. Component sizing */
.card {
    width: clamp(250px, 100%, 400px);
    padding: clamp(1rem, 2vw, 2rem);
}
```

**Why clamp() is Amazing:**
- **Responsive without media queries**
- Clean, declarative syntax
- Perfect for fluid typography
- Reduces code significantly

**Before clamp():**
```css
.heading {
    font-size: 24px;
}
@media (min-width: 768px) {
    .heading { font-size: 32px; }
}
@media (min-width: 1024px) {
    .heading { font-size: 40px; }
}
@media (min-width: 1440px) {
    .heading { font-size: 48px; }
}
```

**With clamp():**
```css
.heading {
    font-size: clamp(24px, 3.5vw, 48px);
}
/* One line! Fluid between 24px and 48px */
```

### Modern Math Functions

#### round(), mod(), rem()

**New in CSS (limited browser support, but coming!):**

```css
/* round() - Round to nearest interval */
.element {
    width: round(157px, 10px);  /* Rounds to 160px */
    margin: round(up, 23px, 5px); /* Rounds up to 25px */

.element {
    width: mod(18px, 5px);  /* 3px (18 ÷ 5 = 3 remainder 3) */

.element {
    width: rem(18px, 5px);  /* Same as mod for positive numbers */
}
```

## 11.3 Logical Properties

**Logical Properties** adapt to writing direction (LTR/RTL) automatically. **Essential for internationalization!**

### Concept: Logical vs Physical

**Physical Properties** (traditional):
- `left`, `right`, `top`, `bottom`
- `margin-left`, `padding-right`
- `border-left`, etc.

**Logical Properties** (modern):
- `inline-start`, `inline-end` (horizontal in LTR/RTL)
- `block-start`, `block-end` (vertical)

**Direction Mapping:**

```
LTR (English):
┌─────────────────────┐
│ inline-start: left  │
│ inline-end: right   │
│ block-start: top    │
│ block-end: bottom   │
└─────────────────────┘

RTL (Arabic/Hebrew):
┌─────────────────────┐
│ inline-start: right │ ← Flipped!
│ inline-end: left    │
│ block-start: top    │
│ block-end: bottom   │
└─────────────────────┘
```

---

### Logical Properties Examples

```css
/* Physical (old way) */
.element {
    margin-left: 20px;      /* Always left */
    padding-right: 10px;    /* Always right */
    border-left: 2px solid; /* Always left */
}

/* Logical (new way) */
.element {
    margin-inline-start: 20px;   /* Left in LTR, right in RTL */
    padding-inline-end: 10px;    /* Right in LTR, left in RTL */
    border-inline-start: 2px solid; /* Left in LTR, right in RTL */
}
```

**Complete Property Mapping:**

```css
/* Margin */
margin-left      → margin-inline-start
margin-right     → margin-inline-end
margin-top       → margin-block-start
margin-bottom    → margin-block-end

/* Shorthands */
margin-inline: 20px 40px;  /* start end */
margin-block: 10px 20px;   /* start end */

/* Padding */
padding-left     → padding-inline-start
padding-right    → padding-inline-end
padding-top      → padding-block-start
padding-bottom   → padding-block-end

padding-inline: 20px 40px;
padding-block: 10px 20px;

/* Border */
border-left      → border-inline-start
border-right     → border-inline-end
border-top       → border-block-start
border-bottom    → border-block-end

/* Positioning */
left             → inset-inline-start
right            → inset-inline-end
top              → inset-block-start
bottom           → inset-block-end

/* Dimensions */
width            → inline-size
height           → block-size
max-width        → max-inline-size
min-height       → min-block-size
```

**Practical Example:**

```css
/* Sidebar that adapts to direction */
.sidebar {
    /* Physical (doesn't adapt) */
    float: left;
    margin-right: 20px;
    border-right: 1px solid #ddd;
    
    /* Logical (adapts to RTL!) */
    float: inline-start;
    margin-inline-end: 20px;
    border-inline-end: 1px solid #ddd;
}
```

**When to Use:**
- Building multilingual sites
- Components that might be used in RTL contexts
- Future-proofing your CSS
- Modern, semantic CSS

### inset Shorthand

```css
/* Instead of top, right, bottom, left */
.element {
    position: absolute;
    top: 0;
    right: 0;
    bottom: 0;
    left: 0;
}

/* Use inset */
.element {
    position: absolute;
    inset: 0;  /* All sides */
}

/* Different values */
.element {
    inset: 10px 20px 30px 40px;  /* top right bottom left */
    inset: 10px 20px;             /* vertical horizontal */
}

/* Logical inset */
.element {
    inset-inline: 20px;   /* start and end */
    inset-block: 10px;    /* start and end */
}
```

## 11.4 Feature Queries (@supports)

Test if browser supports a CSS feature before using it. **Progressive enhancement!**

### Basic Syntax

```css
/* Check if browser supports feature */
@supports (display: grid) {
    .container {
        display: grid;
    }
}

/* Check if feature NOT supported */
@supports not (display: grid) {
    .container {
        display: flex;  /* Fallback */
    }
}
```

---

### Logical Operators

```css
/* AND: Both conditions must be true */
@supports (display: grid) and (gap: 20px) {
    .grid {
        display: grid;
        gap: 20px;
    }
}

/* OR: Either condition can be true */
@supports (display: grid) or (display: flex) {
    .container {
        display: grid;
    }
}

/* NOT: Feature not supported */
@supports not (display: grid) {
    .fallback {
        display: table;
    }
}

/* Complex combinations */
@supports (display: grid) and (not (gap: 20px)) {
    .grid {
        display: grid;
        margin: -10px;
    }
    .grid > * {
        margin: 10px;
    }
}
```

---

### Practical Examples

```css
/* Grid with flexbox fallback */
.container {
    display: flex;  /* Fallback */
}

@supports (display: grid) {
    .container {
        display: grid;
        grid-template-columns: repeat(3, 1fr);
        gap: 20px;
    }
}

/* Backdrop filter with fallback */
.glassmorphism {
    background: rgba(255, 255, 255, 0.7);
}

@supports (backdrop-filter: blur(10px)) {
    .glassmorphism {
        backdrop-filter: blur(10px);
        background: rgba(255, 255, 255, 0.5);
    }
}

/* Sticky positioning with fallback */
.header {
    position: relative;  /* Fallback */
}

@supports (position: sticky) {
    .header {
        position: sticky;
        top: 0;
    }
}

/* Custom properties check */
@supports (--custom: property) {
    /* Use CSS variables */
    :root {
        --primary: #667eea;
    }
    .element {
        color: var(--primary);
    }
}
```

---

### Feature Queries for Progressive Enhancement

```css
/* Base styles (work everywhere) */
.card {
    padding: 20px;
    background: white;
}

/* Enhanced with CSS Grid (if supported) */
@supports (display: grid) {
    .card {
        display: grid;
        grid-template-columns: 1fr 2fr;
        gap: 20px;
    }
}

/* Further enhanced with container queries (if supported) */
@supports (container-type: inline-size) {
    .card-container {
        container-type: inline-size;
    }
    
    @container (min-width: 400px) {
        .card {
            grid-template-columns: 1fr 3fr;
        }
    }
}
```

## 11.5 CSS Nesting (Modern!)

**Native CSS nesting** (like Sass) - reduces repetition and improves organization.

### Basic Nesting

```css
/* Traditional (flat) */
.card { }
.card .title { }
.card .content { }
.card:hover { }

/* Nested (modern) */
.card {
    /* Styles for .card */
    
    .title {
        /* Styles for .card .title */
    }
    
    .content {
        /* Styles for .card .content */
    }
    
    &:hover {
        /* Styles for .card:hover */
    }
}
```

---

### The & Symbol (Nesting Selector)

Represents the parent selector.

```css
.button {
    background: blue;
    
    /* & = .button */
    &:hover {
        background: darkblue;
    }
    
    &:active {
        background: navy;
    }
    
    &.primary {
        background: green;
    }
    
    &-icon {
        /* .button-icon (BEM-style) */
        margin-right: 8px;
    }
}
```

**Compiled Output:**
```css
.button { background: blue; }
.button:hover { background: darkblue; }
.button:active { background: navy; }
.button.primary { background: green; }
.button-icon { margin-right: 8px; }
```

---

### Complex Nesting

```css
.card {
    padding: 20px;
    
    .header {
        font-size: 24px;
        
        .title {
            font-weight: bold;
            /* Compiles to: .card .header .title */
        }
    }
    
    &:hover {
        box-shadow: 0 4px 8px rgba(0,0,0,0.1);
        
        .header {
            color: blue;
            /* Compiles to: .card:hover .header */
        }
    }
    
    @media (min-width: 768px) {
        padding: 40px;
        /* Compiles to: @media (min-width: 768px) { .card { ... } } */
    }
}
```

---

### Nesting Media Queries

```css
.element {
    font-size: 14px;
    
    @media (min-width: 768px) {
        font-size: 16px;
    }
    
    @media (min-width: 1024px) {
        font-size: 18px;
    }
}

/* Compiles to: */
.element { font-size: 14px; }
@media (min-width: 768px) {
    .element { font-size: 16px; }
}
@media (min-width: 1024px) {
    .element { font-size: 18px; }
}
```

---

### Best Practices for Nesting

```css
/* ✅ Good: 2-3 levels max */
.card {
    .header {
        .title { }
    }
}

/* ❌ Avoid: Too deep (hard to read, high specificity) */
.page {
    .container {
        .section {
            .card {
                .header {
                    .title { }  /* Too deep! */
                }
            }
        }
    }
}

/* ✅ Good: Clear hierarchy */
.nav {
    &-item { }
    &-link { }
    &:hover { }
}

/* ✅ Good: Media queries nested */
.component {
    display: block;
    
    @media (min-width: 768px) {
        display: flex;
    }
}
```

**Limitations:**
- Keep nesting shallow (2-3 levels max)
- Watch specificity (nesting increases it)
- Not all browsers support it yet (check compatibility)

## Interview Key Points

**Most Common Questions:**

**1. "What are CSS Custom Properties and how do they work?"**
```
Variables declared with -- prefix, used with var()
Follow CSS cascade and inheritance
Can be changed dynamically with JavaScript
Scoped to element and descendants
```

**2. "How to declare and use CSS variables?"**
```css
/* Declare */
:root { --color: blue; }
/* Use */
.element { color: var(--color); }
```

**3. "Explain clamp() function"**
```css
clamp(min, preferred, max)
/* Returns: min if preferred < min
           preferred if between min and max
           max if preferred > max */
font-size: clamp(16px, 4vw, 32px);
```

**4. "Difference between min(), max(), clamp()?"**
```
min(): Smallest value
max(): Largest value  
clamp(): Value constrained between min and max
```

**5. "What are logical properties and why use them?"**
```
Direction-agnostic properties (inline/block instead of left/right)
Automatically adapt to RTL languages
Better for internationalization
Example: margin-inline-start instead of margin-left
```

**6. "What's @supports used for?"**
```
Feature queries - test if browser supports CSS feature
Progressive enhancement pattern
Provide fallbacks for older browsers
```

**7. "Can you modify CSS variables with JavaScript?"**
```javascript
// Yes!
document.documentElement.style
    .setProperty('--color', 'red');
```

## Best Practices

```css
/* 1. Organize variables systematically */
:root {
    /* Colors */
    --color-primary: #667eea;
    --color-secondary: #764ba2;
    
    /* Spacing */
    --space-xs: 4px;
    --space-sm: 8px;
    --space-md: 16px;
    
    /* Typography */
    --font-size-sm: clamp(14px, 2vw, 16px);
    --font-size-md: clamp(16px, 2.5vw, 18px);
}

/* 2. Use clamp() for fluid scaling */
h1 {
    font-size: clamp(2rem, 5vw, 4rem);
}

/* 3. Prefer logical properties for i18n */
.element {
    margin-inline-start: 20px;  /* ✅ Adapts to RTL */
    /* margin-left: 20px; */    /* ❌ Fixed direction */
}

/* 4. Use @supports for progressive enhancement */
@supports (display: grid) {
    .container { display: grid; }
}

/* 5. Combine features */
:root {
    --sidebar-width: clamp(200px, 20vw, 300px);
}
.main {
    width: calc(100% - var(--sidebar-width));
}
```
