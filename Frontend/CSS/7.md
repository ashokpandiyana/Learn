# Chapter 7: Responsive Design - In-Depth Explanation

## Introduction to Responsive Design

**Responsive Design** makes websites adapt to different screen sizes, devices, and user preferences. It's not optional—it's essential for modern web development.

**Core Principles:**
1. **Fluid Grids**: Use relative units (%, fr, vw) instead of fixed pixels
2. **Flexible Images**: Images scale with their containers
3. **Media Queries**: Apply different styles based on device characteristics
4. **Mobile-First**: Design for mobile, then enhance for larger screens

---

## 7.1 Media Queries

Media queries allow you to apply CSS based on device characteristics like screen width, height, orientation, and more.

### Basic Syntax

```css
@media media-type and (condition) {
    /* CSS rules */
}
```

**Components:**
- `@media`: The at-rule that starts a media query
- `media-type`: Optional (screen, print, all)
- `condition`: Feature query (width, height, orientation, etc.)
- Logical operators: `and`, `or` (comma), `not`

---

### Media Types

```css
/* Screen devices (default) */
@media screen {
    body {
        font-size: 16px;
    }
}

/* Print styles */
@media print {
    body {
        font-size: 12pt;
        color: black;
    }
    .no-print {
        display: none;
    }
}

/* All media types */
@media all {
    /* Applies to everything */
}
```

**Common Types:**
- `screen`: Computer screens, tablets, phones
- `print`: Printed documents
- `all`: All devices (default if omitted)

**Deprecated types:** `speech`, `tv`, `projection` (rarely used)

---

### Media Features

#### Width-based Queries (Most Common!)

```css
/* Exact width (rarely used) */
@media (width: 768px) {
    /* Exactly 768px wide */
}

/* Minimum width */
@media (min-width: 768px) {
    /* 768px and wider */
    .container {
        max-width: 1200px;
    }
}

/* Maximum width */
@media (max-width: 767px) {
    /* 767px and narrower */
    .sidebar {
        display: none;
    }
}

/* Width range (modern syntax) */
@media (min-width: 768px) and (max-width: 1024px) {
    /* Between 768px and 1024px */
    .container {
        padding: 40px;
    }
}

/* Modern range syntax (Level 4) */
@media (768px <= width <= 1024px) {
    /* Same as above, cleaner syntax */
}
```

**Common Breakpoints:**
```css
/* Mobile-first approach (recommended) */

/* Base styles for mobile (< 640px) */
body {
    font-size: 14px;
}

/* Small tablets (≥ 640px) */
@media (min-width: 640px) {
    body {
        font-size: 15px;
    }
}

/* Tablets (≥ 768px) */
@media (min-width: 768px) {
    body {
        font-size: 16px;
    }
}

/* Desktop (≥ 1024px) */
@media (min-width: 1024px) {
    body {
        font-size: 17px;
    }
}

/* Large desktop (≥ 1280px) */
@media (min-width: 1280px) {
    body {
        font-size: 18px;
    }
}
```

**Standard Breakpoint Values:**
- **Mobile**: < 640px
- **Tablet**: 640px - 1024px
- **Desktop**: ≥ 1024px
- **Large Desktop**: ≥ 1280px or 1440px

---

#### Height-based Queries

```css
/* Minimum height */
@media (min-height: 600px) {
    .hero {
        height: 100vh;  /* Full height on tall screens */
    }
}

/* Maximum height (short screens) */
@media (max-height: 500px) {
    .header {
        height: 60px;  /* Shorter header on short screens */
    }
}
```

---

#### Orientation

```css
/* Portrait (taller than wide) */
@media (orientation: portrait) {
    .sidebar {
        width: 100%;
    }
}

/* Landscape (wider than tall) */
@media (orientation: landscape) {
    .sidebar {
        width: 300px;
    }
}
```

**Use Cases:**
- Tablets can switch between portrait and landscape
- Mobile games often require landscape
- Photo galleries optimize for orientation

---

#### Aspect Ratio

```css
/* Exact aspect ratio */
@media (aspect-ratio: 16/9) {
    /* 16:9 screens */
}

/* Minimum aspect ratio */
@media (min-aspect-ratio: 16/9) {
    /* Wider than 16:9 */
}

/* Maximum aspect ratio */
@media (max-aspect-ratio: 4/3) {
    /* Narrower than 4:3 */
}
```

---

#### Resolution (Pixel Density)

```css
/* Standard resolution */
@media (min-resolution: 192dpi) {
    /* High DPI displays (Retina, etc.) */
    .logo {
        background-image: url('logo@2x.png');
    }
}

/* Using dppx (dots per pixel) */
@media (min-resolution: 2dppx) {
    /* 2x displays (Retina) */
    .icon {
        background-image: url('icon@2x.png');
    }
}

/* Webkit syntax (legacy) */
@media (-webkit-min-device-pixel-ratio: 2) {
    /* Retina displays */
}
```

---

#### Hover Capability

```css
/* Devices that can hover (mouse, trackpad) */
@media (hover: hover) {
    .button:hover {
        background: blue;
    }
}

/* Devices without hover (touch screens) */
@media (hover: none) {
    .button {
        padding: 20px;  /* Larger touch targets */
    }
}
```

**Pointer Type:**
```css
/* Fine pointer (mouse) */
@media (pointer: fine) {
    .clickable {
        cursor: pointer;
    }
}

/* Coarse pointer (touch) */
@media (pointer: coarse) {
    .button {
        min-height: 44px;  /* Larger touch targets */
    }
}
```

---

#### Prefers Color Scheme (User Preference)

```css
/* User prefers light mode */
@media (prefers-color-scheme: light) {
    body {
        background: white;
        color: black;
    }
}

/* User prefers dark mode */
@media (prefers-color-scheme: dark) {
    body {
        background: #1a1a1a;
        color: #f0f0f0;
    }
}
```

**Complete Dark Mode Implementation:**
```css
/* Default (light mode) */
:root {
    --bg-color: #ffffff;
    --text-color: #000000;
    --border-color: #e0e0e0;
}

body {
    background: var(--bg-color);
    color: var(--text-color);
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
    :root {
        --bg-color: #1a1a1a;
        --text-color: #f0f0f0;
        --border-color: #333333;
    }
}
```

---

#### Prefers Reduced Motion (Accessibility)

```css
/* Default: animations enabled */
.element {
    transition: transform 0.3s ease;
}

/* User prefers reduced motion */
@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration: 0.01ms !important;
        transition-duration: 0.01ms !important;
    }
}

/* Better approach: conditional animations */
@media (prefers-reduced-motion: no-preference) {
    .element {
        animation: slide-in 0.5s ease;
    }
}
```

**Why This Matters:**
- Some users get motion sickness from animations
- Vestibular disorders affected by scrolling effects
- Required for WCAG AA accessibility compliance

---

#### Prefers Contrast

```css
/* Standard contrast */
@media (prefers-contrast: no-preference) {
    /* Default styles */
}

/* User wants high contrast */
@media (prefers-contrast: high) {
    body {
        background: black;
        color: white;
        border-color: white;
    }
    
    .button {
        border: 3px solid white;
    }
}

/* User wants low contrast */
@media (prefers-contrast: low) {
    body {
        color: #666;
    }
}
```

---

### Logical Operators

#### AND Operator

```css
/* Both conditions must be true */
@media (min-width: 768px) and (max-width: 1024px) {
    /* Between 768px and 1024px */
}

@media screen and (min-width: 768px) {
    /* Screen devices 768px and wider */
}

@media (min-width: 768px) and (orientation: landscape) {
    /* Wide screens in landscape */
}
```

#### OR Operator (Comma)

```css
/* Either condition can be true */
@media (max-width: 767px), (orientation: portrait) {
    /* Mobile OR portrait mode */
}

@media (min-width: 1024px), print {
    /* Desktop OR print */
}
```

#### NOT Operator

```css
/* Negates the condition */
@media not screen {
    /* Everything EXCEPT screen */
}

@media not (min-width: 768px) {
    /* NOT 768px or wider (same as max-width: 767px) */
}
```

### Mobile-First vs Desktop-First

#### Mobile-First (Recommended!)

Start with mobile styles, add complexity for larger screens.

```css
/* Base styles for mobile */
body {
    font-size: 14px;
}

.container {
    padding: 20px;
}

.sidebar {
    display: none;  /* Hidden on mobile */
}

/* Tablet and up */
@media (min-width: 768px) {
    body {
        font-size: 16px;
    }
    
    .container {
        padding: 40px;
    }
    
    .sidebar {
        display: block;  /* Show on larger screens */
    }
}

/* Desktop */
@media (min-width: 1024px) {
    .container {
        max-width: 1200px;
        margin: 0 auto;
    }
}
```

**Benefits of Mobile-First:**
- Forces you to prioritize content
- Smaller initial CSS (mobile loads faster)
- Progressive enhancement
- Easier to maintain
- Aligns with mobile-first indexing (SEO)

---

#### Desktop-First (Legacy Approach)

Start with desktop, remove features for mobile.

```css
/* Base styles for desktop */
body {
    font-size: 18px;
}

.container {
    max-width: 1200px;
    padding: 60px;
}

.sidebar {
    width: 300px;
    float: left;
}

/* Tablet */
@media (max-width: 1023px) {
    .container {
        padding: 40px;
    }
}

/* Mobile */
@media (max-width: 767px) {
    body {
        font-size: 14px;
    }
    
    .container {
        padding: 20px;
    }
    
    .sidebar {
        float: none;
        width: 100%;
    }
}
```

**Why Avoid Desktop-First:**
- Loads unnecessary CSS on mobile
- Harder to optimize for mobile
- Doesn't align with mobile-first development

### Common Responsive Patterns

#### Responsive Navigation

```css
/* Mobile: Hamburger menu */
.nav-toggle {
    display: block;
}

.nav-menu {
    display: none;
    position: fixed;
    top: 60px;
    left: 0;
    width: 100%;
    background: white;
}

.nav-menu.active {
    display: block;
}

/* Desktop: Horizontal menu */
@media (min-width: 768px) {
    .nav-toggle {
        display: none;
    }
    
    .nav-menu {
        display: flex;
        position: static;
        width: auto;
    }
}
```

#### Responsive Grid

```css
/* Mobile: 1 column */
.grid {
    display: grid;
    grid-template-columns: 1fr;
    gap: 20px;
}

/* Tablet: 2 columns */
@media (min-width: 640px) {
    .grid {
        grid-template-columns: repeat(2, 1fr);
    }
}

/* Desktop: 3 columns */
@media (min-width: 1024px) {
    .grid {
        grid-template-columns: repeat(3, 1fr);
    }
}

/* Alternative: No media queries! */
.grid-auto {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    gap: 20px;
}
```

#### Responsive Typography

```css
/* Fluid typography with clamp() */
body {
    font-size: clamp(14px, 2.5vw, 18px);
    /* Min: 14px, Preferred: 2.5vw, Max: 18px */
}

h1 {
    font-size: clamp(1.5rem, 5vw, 3rem);
}

/* Traditional approach */
body {
    font-size: 14px;
}

@media (min-width: 768px) {
    body {
        font-size: 16px;
    }
}

@media (min-width: 1024px) {
    body {
        font-size: 18px;
    }
}
```

## 7.2 Viewport and Responsive Units

### Viewport Meta Tag (Critical!)

**Required for responsive design!** Without it, mobile browsers zoom out to show desktop version.

```html
<!-- Essential viewport meta tag -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**What it does:**
- `width=device-width`: Set viewport width to device width
- `initial-scale=1.0`: Set initial zoom level to 100%

**Additional options:**
```html
<!-- Prevent zooming (use carefully - accessibility concern) -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">

<!-- Allow zooming (recommended) -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0">
```

**Best Practice:** Always allow zooming for accessibility!

---

### Viewport Units

Units relative to the viewport (browser window).

#### Basic Viewport Units

```css
/* vw: 1% of viewport width */
.element {
    width: 50vw;    /* 50% of viewport width */
    width: 100vw;   /* Full viewport width */
}

/* vh: 1% of viewport height */
.element {
    height: 50vh;   /* 50% of viewport height */
    height: 100vh;  /* Full viewport height */
}

/* vmin: 1% of viewport's smaller dimension */
.square {
    width: 50vmin;
    height: 50vmin;  /* Always square */
}

/* vmax: 1% of viewport's larger dimension */
.element {
    width: 50vmax;   /* 50% of larger dimension */
}
```

**Visual Example:**

Viewport: 1000px wide × 600px tall
```
vw:   10vw = 100px (10% of 1000px)
vh:   10vh = 60px (10% of 600px)
vmin: 10vmin = 60px (10% of 600px - smaller dimension)
vmax: 10vmax = 100px (10% of 1000px - larger dimension)
```

---

#### Dynamic Viewport Units (Modern!)

Solve the mobile browser address bar problem.

```css
/* Traditional vh problem on mobile */
.hero {
    height: 100vh;  /* Includes address bar space! */
}

/* Small viewport height (svh) - smallest possible */
.hero {
    height: 100svh;  /* Always accounts for UI */
}

/* Large viewport height (lvh) - largest possible */
.hero {
    height: 100lvh;  /* Maximum height when UI hidden */
}

/* Dynamic viewport height (dvh) - adjusts in real-time */
.hero {
    height: 100dvh;  /* Adapts to UI showing/hiding ✓ BEST */
}
```

**Explanation:**
- **svh**: Smallest viewport height (when all UI visible)
- **lvh**: Largest viewport height (when UI hidden)
- **dvh**: Dynamic (changes as UI shows/hides) **← Use this!**
- Same for width: `svw`, `lvw`, `dvw`
- Same for min: `svmin`, `lvmin`, `dvmin`
- Same for max: `svmax`, `lvmax`, `dvmax`

**Browser Support:** Chrome 108+, Safari 15.4+, Firefox 101+

---

### Common Viewport Unit Patterns

```css
/* Full-screen hero */
.hero {
    height: 100dvh;  /* Full viewport height */
    display: flex;
    align-items: center;
    justify-content: center;
}

/* Responsive font size */
h1 {
    font-size: 5vw;  /* Scales with viewport */
    font-size: clamp(2rem, 5vw, 4rem);  /* Better: with limits */
}

/* Full-width sections */
.full-width {
    width: 100vw;
    margin-left: calc(50% - 50vw);  /* Break out of container */
}

/* Responsive padding */
.section {
    padding: 5vh 5vw;  /* Scales with viewport */
}

/* Square elements */
.avatar {
    width: 20vmin;
    height: 20vmin;  /* Always square */
    border-radius: 50%;
}
```

---

### calc() with Viewport Units

```css
/* Subtract header height from viewport */
.main-content {
    height: calc(100vh - 80px);  /* Full height minus 80px header */
}

/* Responsive with minimum */
.container {
    width: calc(100vw - 40px);   /* Full width minus 40px padding */
    max-width: 1200px;
}

/* Fluid spacing */
.section {
    padding: calc(2rem + 2vw);   /* Grows with viewport */
}
```

## 7.3 Responsive Images

### Flexible Images (Basic)

```css
/* Make images responsive */
img {
    max-width: 100%;     /* Never wider than container */
    height: auto;        /* Maintain aspect ratio */
}

/* For better performance */
img {
    max-width: 100%;
    height: auto;
    display: block;      /* Remove bottom gap */
}
```

---

### srcset and sizes Attributes

Serve different images based on device capabilities.

```html
<!-- Different resolutions (1x, 2x, 3x) -->
<img 
    src="image.jpg" 
    srcset="image.jpg 1x, 
            image@2x.jpg 2x, 
            image@3x.jpg 3x"
    alt="Description">

<!-- Different widths -->
<img 
    src="image-800.jpg"
    srcset="image-400.jpg 400w,
            image-800.jpg 800w,
            image-1200.jpg 1200w"
    sizes="(max-width: 600px) 400px,
           (max-width: 1000px) 800px,
           1200px"
    alt="Description">
```

**How sizes works:**
1. Browser checks `sizes` conditions
2. Picks appropriate size
3. Selects best image from `srcset`

**Example Breakdown:**
```html
<img 
    srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1200w"
    sizes="(max-width: 600px) 100vw,
           (max-width: 1000px) 50vw,
           33vw">
```
- **< 600px wide**: Image is 100vw (full width) → use small.jpg
- **600-1000px**: Image is 50vw (half width) → use medium.jpg  
- **> 1000px**: Image is 33vw (one-third) → use large.jpg

---

### Picture Element

More control over image sources based on conditions.

```html
<!-- Art direction: Different crops for different sizes -->
<picture>
    <!-- Wide screens: horizontal crop -->
    <source media="(min-width: 1024px)" srcset="wide-crop.jpg">
    
    <!-- Medium screens: square crop -->
    <source media="(min-width: 640px)" srcset="square-crop.jpg">
    
    <!-- Mobile: vertical crop -->
    <img src="mobile-crop.jpg" alt="Description">
</picture>

<!-- Different formats with fallback -->
<picture>
    <!-- Modern browsers: WebP -->
    <source srcset="image.webp" type="image/webp">
    
    <!-- Older browsers: JPEG -->
    <img src="image.jpg" alt="Description">
</picture>

<!-- Combining conditions -->
<picture>
    <!-- Desktop: Large WebP -->
    <source 
        media="(min-width: 1024px)" 
        srcset="large.webp" 
        type="image/webp">
    
    <!-- Desktop: Large JPEG fallback -->
    <source 
        media="(min-width: 1024px)" 
        srcset="large.jpg">
    
    <!-- Mobile: Small WebP -->
    <source 
        srcset="small.webp" 
        type="image/webp">
    
    <!-- Mobile: Small JPEG fallback -->
    <img src="small.jpg" alt="Description">
</picture>
```

---

### object-fit and object-position

Control how images fill their containers.

```css
/* Cover: Fill container, may crop */
.image-cover {
    width: 300px;
    height: 200px;
    object-fit: cover;        /* Fill container */
    object-position: center;  /* Center image */
}

/* Contain: Show entire image, may have gaps */
.image-contain {
    width: 300px;
    height: 200px;
    object-fit: contain;
}

/* Other values */
.image {
    object-fit: fill;        /* Stretch to fill (default) */
    object-fit: none;        /* Original size */
    object-fit: scale-down;  /* Smaller of none or contain */
}

/* Custom positioning */
.image {
    object-fit: cover;
    object-position: top right;    /* Align to top-right */
    object-position: 20% 40%;      /* Custom percentage */
    object-position: 50px 100px;   /* Pixel offset */
}
```

**Visual Comparison:**

**cover:** (most common)
```
Container: 300×200px
Image: 400×300px

┌─────────────────────┐
│   [  Image Crop  ]  │  ← Image fills, crops top/bottom
└─────────────────────┘
```

**contain:**
```
┌─────────────────────┐
│                     │
│    [Full Image]     │  ← Entire image visible, gaps on sides
│                     │
└─────────────────────┘
```

---

### Lazy Loading

Defer loading images until needed (built-in browser feature).

```html
<!-- Native lazy loading -->
<img src="image.jpg" loading="lazy" alt="Description">

<!-- Eager loading (default) -->
<img src="hero.jpg" loading="eager" alt="Hero image">

<!-- Auto (browser decides) -->
<img src="image.jpg" loading="auto" alt="Description">
```

**Best Practices:**
- Use `loading="lazy"` for below-fold images
- Use `loading="eager"` for above-fold images
- Excellent for performance (reduces initial page load)

## 7.4 Container Queries (Modern Game-Changer!)

Query the **container size** instead of viewport size. **Component-based responsive design!**

### Basic Container Queries

```css
/* Define a container */
.card-container {
    container-type: inline-size;  /* Query container width */
    container-name: card;         /* Optional name */
}

/* Query the container */
@container (min-width: 400px) {
    .card-title {
        font-size: 2rem;
    }
}

/* Named container query */
@container card (min-width: 400px) {
    .card-content {
        display: grid;
        grid-template-columns: 1fr 1fr;
    }
}
```

**Container Types:**
```css
.container {
    container-type: size;        /* Query width and height */
    container-type: inline-size; /* Query width only (most common) */
    container-type: normal;      /* No containment (default) */
}
```

---

### Container Query Units

Special units that reference container size, not viewport.

```css
.container {
    container-type: inline-size;
}

.child {
    /* cqi: 1% of container inline size (width) */
    font-size: 5cqi;
    
    /* cqw: 1% of container width */
    padding: 2cqw;
    
    /* cqh: 1% of container height */
    height: 50cqh;
    
    /* cqmin: 1% of container's smaller dimension */
    width: 10cqmin;
    
    /* cqmax: 1% of container's larger dimension */
    margin: 2cqmax;
    
    /* cqb: 1% of container block size (height) */
    margin-block: 5cqb;
}
```

---

### Why Container Queries Are Amazing

**Problem with Media Queries:**
```css
/* Component looks different based on viewport,
   not where it's placed! */
@media (min-width: 768px) {
    .card {
        display: grid;
        grid-template-columns: 1fr 1fr;
    }
}
/* Card in sidebar still gets 2 columns on wide screens,
   even though sidebar is narrow! */
```

**Solution with Container Queries:**
```css
.card-container {
    container-type: inline-size;
}

/* Card responds to its container, not viewport! */
@container (min-width: 500px) {
    .card {
        display: grid;
        grid-template-columns: 1fr 1fr;
    }
}
/* Card in narrow sidebar stays single column,
   card in wide main area gets 2 columns! */
```

---

### Real-World Example

```css
/* Card component that adapts to its container */
.card-container {
    container-type: inline-size;
    container-name: card;
}

/* Small container: Stacked layout */
.card {
    display: flex;
    flex-direction: column;
}

/* Medium container: Side-by-side */
@container card (min-width: 400px) {
    .card {
        flex-direction: row;
    }
    
    .card-image {
        width: 40%;
    }
    
    .card-content {
        width: 60%;
    }
}

/* Large container: Grid layout */
@container card (min-width: 600px) {
    .card {
        display: grid;
        grid-template-columns: 1fr 2fr;
        gap: 20px;
    }
}
```

## Best Practices Summary

### 1. Mobile-First Always
```css
/* ✅ Start mobile, enhance for desktop */
.element { font-size: 14px; }
@media (min-width: 768px) { .element { font-size: 16px; } }

/* ❌ Avoid desktop-first */
.element { font-size: 18px; }
@media (max-width: 767px) { .element { font-size: 14px; } }
```

### 2. Use Modern Units
```css
/* ✅ Flexible, responsive */
.hero { height: 100dvh; }
.container { padding: clamp(1rem, 5vw, 3rem); }

/* ❌ Fixed, not responsive */
.hero { height: 600px; }
```

### 3. Responsive Images
```html
<!-- ✅ Optimized for device -->
<img src="image.jpg" srcset="..." sizes="..." loading="lazy">

<!-- ❌ Same image for all devices -->
<img src="huge-image.jpg">
```

### 4. Accessibility
```css
/* ✅ Respect user preferences */
@media (prefers-reduced-motion: reduce) {
    * { animation: none !important; }
}

@media (prefers-color-scheme: dark) {
    /* Dark mode styles */
}
```

### 5. Container Queries for Components
```css
/* ✅ Component responds to container */
@container (min-width: 400px) { ... }

/* ❌ Component responds to viewport (less flexible) */
@media (min-width: 400px) { ... }
```

## Interview Key Points

**Most Common Questions:**

**1. "Explain mobile-first approach"**
- Start with mobile styles as base
- Use min-width media queries to enhance for larger screens
- Benefits: Better performance, forces content prioritization

**2. "What's the viewport meta tag and why is it needed?"**
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
- Sets viewport width to device width
- Enables responsive design on mobile
- Without it, mobile browsers show zoomed-out desktop version

**3. "Difference between vh and dvh?"**
- `vh`: Viewport height (includes mobile address bar)
- `dvh`: Dynamic viewport height (adapts to UI changes) **← Better for mobile**

**4. "How to serve different images for different screen sizes?"**
- Use `srcset` with `sizes` for resolution/size variants
- Use `<picture>` for art direction (different crops)

**5. "What are container queries?"**
- Query component's container size instead of viewport
- Enables true component-based responsive design
- Component adapts to where it's placed, not screen size

**6. "Common breakpoints?"**
- Mobile: < 640px
- Tablet: 640-1024px
- Desktop: ≥ 1024px
- Large: ≥ 1280px
