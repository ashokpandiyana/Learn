# Chapter 10: Accessibility (A11y) - In-Depth Explanation

## Introduction: Why Accessibility Matters

**Accessibility (a11y)** means making your website usable by **everyone**, including people with disabilities.

**The Numbers:**
- 15% of the world's population has some form of disability
- 285+ million people are visually impaired
- 466+ million have hearing loss
- Many have motor disabilities, cognitive disabilities, or temporary impairments

**It's Not Just Moral - It's:**
- **Legal** - Required by law in many countries (ADA, Section 508, WCAG)
- **Business** - Larger audience = more customers
- **SEO** - Accessible sites rank better
- **UX** - Good accessibility = good user experience for everyone

**Temporary Disabilities:**
- Broken arm (motor impairment)
- Eye surgery (vision impairment)
- Ear infection (hearing impairment)
- Bright sunlight (situational vision impairment)

---

## 10.1 WCAG Guidelines

**WCAG (Web Content Accessibility Guidelines)** is the international standard for web accessibility.

**Current Version:** WCAG 2.1 (WCAG 2.2 and 3.0 in development)

**Conformance Levels:**
- **Level A** - Minimum (basic accessibility)
- **Level AA** - Mid-range (recommended, often legally required)
- **Level AAA** - Highest (enhanced accessibility)

**Most organizations target Level AA.**

---

### The Four Principles (POUR)

#### 1. Perceivable

**Content must be perceivable to users.**

**Requirements:**
- Text alternatives for images
- Captions for videos
- Audio descriptions
- Content can be presented differently (mobile, screen reader)
- Sufficient color contrast

**Examples:**

```html
<!-- ‚úÖ Perceivable: Image with alt text -->
<img src="chart.jpg" alt="Sales chart showing 25% growth in Q4 2024">

<!-- ‚úÖ Perceivable: Video with captions -->
<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="captions" src="captions.vtt" srclang="en" default>
</video>

<!-- ‚ùå Not perceivable: Image without alt text -->
<img src="chart.jpg">

<!-- ‚ùå Not perceivable: Poor contrast -->
<p style="color: #ccc; background: #fff;">Light gray text on white</p>
```

---

#### 2. Operable

**Users must be able to operate the interface.**

**Requirements:**
- Keyboard accessible
- Enough time to read/use content
- No seizure-inducing content (flashing)
- Navigable (clear navigation, skip links)
- Input modalities beyond keyboard

**Examples:**

```html
<!-- ‚úÖ Operable: Keyboard accessible -->
<button onclick="submit()">Submit</button>
<a href="/page">Link</a>

<!-- ‚ùå Not operable: div as button (not keyboard accessible) -->
<div onclick="submit()">Submit</div>

<!-- ‚úÖ Fix: Add role and tabindex -->
<div role="button" tabindex="0" onclick="submit()" onkeypress="handleKey(event)">
  Submit
</div>

<!-- ‚úÖ Operable: Skip link for navigation -->
<a href="#main-content" class="skip-link">Skip to main content</a>
```

---

#### 3. Understandable

**Content and operation must be understandable.**

**Requirements:**
- Readable text
- Predictable behavior
- Input assistance (labels, error messages)
- Clear language
- Consistent navigation

**Examples:**

```html
<!-- ‚úÖ Understandable: Clear labels -->
<label for="email">Email Address (required):</label>
<input type="email" id="email" required>
<small>We'll never share your email</small>

<!-- ‚ùå Not understandable: No label -->
<input type="email" placeholder="Email">

<!-- ‚úÖ Understandable: Clear error messages -->
<input type="email" id="email" aria-describedby="email-error">
<span id="email-error" role="alert">
  Please enter a valid email address (e.g., user@example.com)
</span>

<!-- ‚ùå Not understandable: Vague error -->
<span>Invalid input</span>
```

---

#### 4. Robust

**Content must work with current and future technologies.**

**Requirements:**
- Valid HTML (passes validation)
- Properly nested elements
- Unique IDs
- Compatible with assistive technologies
- Works in different browsers

**Examples:**

```html
<!-- ‚úÖ Robust: Valid HTML -->
<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
  </ul>
</nav>

<!-- ‚ùå Not robust: Invalid nesting -->
<ul>
  <div>
    <li>Item</li>
  </div>
</ul>

<!-- ‚úÖ Robust: Unique IDs -->
<label for="email1">Email 1:</label>
<input type="email" id="email1">

<label for="email2">Email 2:</label>
<input type="email" id="email2">

<!-- ‚ùå Not robust: Duplicate IDs -->
<input type="email" id="email">
<input type="email" id="email">  <!-- ERROR! -->
```

## 10.2 Semantic HTML for Accessibility

**Use semantic elements - they have built-in accessibility!**

### Landmark Regions

Screen readers can jump directly to landmarks.

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Semantic Landmarks</title>
</head>
<body>
  <!-- Banner landmark -->
  <header role="banner">
    <h1>Site Title</h1>
    
    <!-- Navigation landmark -->
    <nav role="navigation" aria-label="Main">
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
      </ul>
    </nav>
  </header>
  
  <!-- Main landmark -->
  <main role="main">
    <h1>Page Title</h1>
    
    <!-- Article -->
    <article>
      <h2>Article Title</h2>
      <p>Content...</p>
    </article>
    
    <!-- Complementary landmark -->
    <aside role="complementary">
      <h2>Related Content</h2>
    </aside>
  </main>
  
  <!-- Contentinfo landmark -->
  <footer role="contentinfo">
    <p>&copy; 2024 Company</p>
  </footer>
</body>
</html>
```

**Screen Reader Shortcuts:**

```
NVDA/JAWS:
- H: Next heading
- D: Next landmark
- K: Next link
- F: Next form field
- T: Next table
- L: Next list
```

---

### Heading Hierarchy

**Critical for screen reader navigation!**

```html
<!-- ‚úÖ CORRECT: Logical hierarchy -->
<h1>Main Page Title</h1>
  <h2>Section 1</h2>
    <h3>Subsection 1.1</h3>
    <h3>Subsection 1.2</h3>
  <h2>Section 2</h2>
    <h3>Subsection 2.1</h3>

<!-- ‚ùå WRONG: Skipped levels -->
<h1>Main Title</h1>
  <h3>Subsection</h3>  <!-- Skipped h2! -->
  
<!-- ‚ùå WRONG: Multiple h1 tags -->
<h1>Main Title</h1>
<h1>Another Main Title</h1>  <!-- Only one h1 per page! -->
```

**Screen Reader Navigation:**

```
Screen reader user presses "H" key:
1. Jumps to first heading
2. Presses "H" again ‚Üí next heading
3. Presses "2" ‚Üí jumps to next h2
4. Can list all headings to understand page structure
```

## 10.3 Keyboard Navigation

**All functionality must be keyboard accessible.**

**Standard Keyboard Controls:**
- **Tab** - Move to next focusable element
- **Shift + Tab** - Move to previous focusable element
- **Enter** - Activate link/button
- **Space** - Activate button/checkbox
- **Arrow keys** - Navigate within component (dropdowns, tabs, menus)
- **Escape** - Close modal/dialog

---

### Focus Management

**Elements that should be focusable:**
- Links (`<a href>`)
- Buttons (`<button>`)
- Form controls (`<input>`, `<select>`, `<textarea>`)
- Elements with `tabindex="0"`

```html
<!-- ‚úÖ Naturally focusable -->
<a href="/page">Link</a>
<button>Button</button>
<input type="text">

<!-- ‚úÖ Made focusable with tabindex -->
<div tabindex="0" role="button">Custom Button</div>

<!-- ‚ùå Not focusable (accessibility fail) -->
<div onclick="doSomething()">Click me</div>
```

---

### Focus Indicators

**Never remove focus outlines without providing alternative!**

```css
/* ‚ùå NEVER DO THIS (accessibility fail!) */
*:focus {
  outline: none;
}

/* ‚úÖ GOOD: Custom focus indicator */
button:focus {
  outline: 3px solid #0066cc;
  outline-offset: 2px;
}

/* ‚úÖ GOOD: Visible focus for all interactive elements */
a:focus,
button:focus,
input:focus,
select:focus,
textarea:focus {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}

/* ‚úÖ GOOD: Different style for mouse vs keyboard */
button:focus-visible {
  outline: 3px solid #0066cc;
  outline-offset: 2px;
}
```

```html
    }
```

```html
</head>
<body>
  <div class="dropdown">
    <button 
      class="dropdown-button"
      aria-haspopup="true"
      aria-expanded="false"
      aria-controls="dropdown-menu"
      id="dropdown-trigger"
    >
      Menu ‚ñº
    </button>
    
    <div 
      class="dropdown-menu"
      id="dropdown-menu"
      role="menu"
      aria-hidden="true"
    >
      <a href="#" role="menuitem">Option 1</a>
      <a href="#" role="menuitem">Option 2</a>
      <a href="#" role="menuitem">Option 3</a>
    </div>
  </div>

    const trigger = document.getElementById('dropdown-trigger');
    const menu = document.getElementById('dropdown-menu');
    const menuItems = menu.querySelectorAll('[role="menuitem"]');
    let currentIndex = -1;
    
    // Toggle menu
    trigger.addEventListener('click', toggleMenu);
    
    // Keyboard navigation
    trigger.addEventListener('keydown', (e) => {
      if (e.key === 'Enter' || e.key === ' ' || e.key === 'ArrowDown') {
        e.preventDefault();
        openMenu();
        focusMenuItem(0);
      }
    });
    
    menu.addEventListener('keydown', (e) => {
      switch(e.key) {
        case 'ArrowDown':
          e.preventDefault();
          focusMenuItem(currentIndex + 1);
          break;
        case 'ArrowUp':
          e.preventDefault();
          focusMenuItem(currentIndex - 1);
          break;
        case 'Escape':
          e.preventDefault();
          closeMenu();
          trigger.focus();
          break;
        case 'Home':
          e.preventDefault();
          focusMenuItem(0);
          break;
        case 'End':
          e.preventDefault();
          focusMenuItem(menuItems.length - 1);
          break;
      }
    });
    
    function toggleMenu() {
      if (menu.getAttribute('aria-hidden') === 'true') {
        openMenu();
      } else {
        closeMenu();
      }
    }
    
    function openMenu() {
      menu.setAttribute('aria-hidden', 'false');
      trigger.setAttribute('aria-expanded', 'true');
    }
    
    function closeMenu() {
      menu.setAttribute('aria-hidden', 'true');
      trigger.setAttribute('aria-expanded', 'false');
      currentIndex = -1;
    }
    
    function focusMenuItem(index) {
      // Wrap around
      if (index < 0) index = menuItems.length - 1;
      if (index >= menuItems.length) index = 0;
      
      currentIndex = index;
      menuItems[index].focus();
    }
    
    // Close on outside click
    document.addEventListener('click', (e) => {
      if (!e.target.closest('.dropdown')) {
        closeMenu();
      }
    });
  </script>
</body>
</html>
```

### Tab Trapping in Modals

**When modal is open, focus should stay inside.**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Focus Trap Modal</title>
  <style>
    .modal {
      position: fixed;
      top: 50%;
      left: 50%;
      transform: translate(-50%, -50%);
      background: white;
      padding: 30px;
      border-radius: 8px;
      box-shadow: 0 4px 20px rgba(0,0,0,0.3);
      z-index: 1000;
      display: none;
    }
    
    .modal.active {
      display: block;
    }
    
    .modal-backdrop {
      position: fixed;
      top: 0;
      left: 0;
      width: 100%;
      height: 100%;
      background: rgba(0,0,0,0.5);
      z-index: 999;
      display: none;
    }
    
    .modal-backdrop.active {
      display: block;
    }
    
    button {
      padding: 10px 20px;
      margin: 5px;
      border: none;
      border-radius: 4px;
      cursor: pointer;
    }
    
    .btn-primary {
      background: #0066cc;
      color: white;
    }
    
    .btn-secondary {
      background: #6c757d;
      color: white;
    }
  </style>
</head>
<body>
  <h1>Focus Trap Modal Demo</h1>
  <button onclick="openModal()" id="openBtn">Open Modal</button>
  
  <div class="modal-backdrop" id="backdrop"></div>
  
  <div 
    class="modal" 
    id="modal"
    role="dialog"
    aria-modal="true"
    aria-labelledby="modal-title"
  >
    <h2 id="modal-title">Modal Dialog</h2>
    <p>Try pressing Tab - focus stays inside the modal!</p>
    
    <input type="text" placeholder="First name">
    <input type="text" placeholder="Last name">
    
    <div>
      <button class="btn-primary" onclick="save()">Save</button>
      <button class="btn-secondary" onclick="closeModal()">Cancel</button>
    </div>
  </div>
  
  <script>
    const modal = document.getElementById('modal');
    const backdrop = document.getElementById('backdrop');
    const openBtn = document.getElementById('openBtn');
    let previousFocus = null;
    let focusableElements = [];
    let firstFocusable = null;
    let lastFocusable = null;
    
    function openModal() {
      // Save current focus
      previousFocus = document.activeElement;
      
      // Show modal
      modal.classList.add('active');
      backdrop.classList.add('active');
      
      // Get all focusable elements in modal
      focusableElements = modal.querySelectorAll(
        'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
      );
      
      firstFocusable = focusableElements[0];
      lastFocusable = focusableElements[focusableElements.length - 1];
      
      // Focus first element
      firstFocusable.focus();
      
      // Add keyboard trap
      document.addEventListener('keydown', trapFocus);
    }
    
    function closeModal() {
      modal.classList.remove('active');
      backdrop.classList.remove('active');
      
      // Remove keyboard trap
      document.removeEventListener('keydown', trapFocus);
      
      // Restore focus
      if (previousFocus) {
        previousFocus.focus();
      }
    }
    
    function trapFocus(e) {
      // Close on Escape
      if (e.key === 'Escape') {
        closeModal();
        return;
      }
      
      // Trap Tab key
      if (e.key === 'Tab') {
        if (e.shiftKey) { // Shift + Tab
          if (document.activeElement === firstFocusable) {
            e.preventDefault();
            lastFocusable.focus();
          }
        } else { // Tab
          if (document.activeElement === lastFocusable) {
            e.preventDefault();
            firstFocusable.focus();
          }
        }
      }
    }
    
    function save() {
      alert('Saved!');
      closeModal();
    }
    
    // Close on backdrop click
    backdrop.addEventListener('click', closeModal);
  </script>
</body>
</html>
```

## 10.4 Screen Reader Support

### Writing for Screen Readers

**Screen readers announce:**
1. Element type ("button", "link", "heading level 2")
2. Element label/text
3. Element state ("checked", "expanded", "selected")

```html
<!-- Screen reader announces: "Home, link" -->
<a href="/">Home</a>

<!-- Screen reader announces: "Submit, button" -->
<button>Submit</button>

<!-- Screen reader announces: "Heading level 1, Page Title" -->
<h1>Page Title</h1>

<!-- Screen reader announces: "Email, required, edit text" -->
<label for="email">Email:</label>
<input type="email" id="email" required>
```

### Live Regions

**Announce dynamic content changes.**

```html
<div aria-live="polite" role="status" id="status-message">
  <!-- Dynamic messages appear here -->
</div>

<script>
  function showStatus(message) {
    document.getElementById('status-message').textContent = message;
    // Screen reader announces this automatically
  }
  
  // Usage
  showStatus('Form saved successfully');
  showStatus('3 items added to cart');
</script>
```

**Politeness Levels:**

```html
<!-- polite: Waits for user to finish current task -->
<div aria-live="polite">
  Status updates, non-critical notifications
</div>

<!-- assertive: Interrupts immediately -->
<div aria-live="assertive" role="alert">
  Critical errors, urgent warnings
</div>

<!-- off: No announcements (default) -->
<div aria-live="off">
  Regular content
</div>
```

```html
<body>
  <h1>Accessible Form Validation</h1>
  
  <!-- Screen reader announcements -->
  <div aria-live="polite" role="status" class="sr-only" id="form-status"></div>
  
  <form id="myForm" novalidate>
    <div class="form-group">
      <label for="username">
        Username <span aria-label="required">*</span>
      </label>
      <input 
        type="text" 
        id="username"
        name="username"
        aria-required="true"
        aria-describedby="username-error"
        minlength="3"
        required
      >
      <div id="username-error" class="error-message" role="alert"></div>
    </div>
    
    <div class="form-group">
      <label for="email">
        Email Address <span aria-label="required">*</span>
      </label>
      <input 
        type="email" 
        id="email"
        name="email"
        aria-required="true"
        aria-describedby="email-error"
        required
      >
      <div id="email-error" class="error-message" role="alert"></div>
    </div>
    
    <div class="form-group">
      <label for="phone">
        Phone Number
      </label>
      <input 
        type="tel" 
        id="phone"
        name="phone"
        aria-describedby="phone-help"
        pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}"
      >
      <small id="phone-help">Format: 123-456-7890</small>
      <div id="phone-error" class="error-message" role="alert"></div>
    </div>
    
    <button type="submit">Submit</button>
  </form>
  
  <script>
    const form = document.getElementById('myForm');
    const formStatus = document.getElementById('form-status');
    
    // Real-time validation
    const fields = form.querySelectorAll('input');
    
    fields.forEach(field => {
      field.addEventListener('blur', () => validateField(field));
      field.addEventListener('input', () => {
        if (field.validity.valid) {
          clearError(field);
        }
      });
    });
    
    function validateField(field) {
      const errorDiv = document.getElementById(field.id + '-error');
      
      if (field.validity.valueMissing) {
        showError(field, errorDiv, `${field.labels[0].textContent.replace('*', '').trim()} is required`);
      } else if (field.validity.tooShort) {
        showError(field, errorDiv, `Must be at least ${field.minLength} characters`);
      } else if (field.validity.typeMismatch) {
        showError(field, errorDiv, 'Please enter a valid format');
      } else if (field.validity.patternMismatch) {
        const help = field.getAttribute('aria-describedby');
        const helpText = document.getElementById(help)?.textContent || 'Invalid format';
        showError(field, errorDiv, helpText);
      } else {
        clearError(field, errorDiv);
      }
    }
    
    function showError(field, errorDiv, message) {
      field.setAttribute('aria-invalid', 'true');
      errorDiv.textContent = message;
      errorDiv.classList.add('show');
      
      // Announce to screen reader
      formStatus.textContent = message;
    }
    
    function clearError(field, errorDiv) {
      field.setAttribute('aria-invalid', 'false');
      if (errorDiv) {
        errorDiv.textContent = '';
        errorDiv.classList.remove('show');
      }
    }
    
    // Form submission
    form.addEventListener('submit', (e) => {
      e.preventDefault();
      
      let isValid = true;
      
      fields.forEach(field => {
        validateField(field);
        if (!field.validity.valid) {
          isValid = false;
        }
      });
      
      if (isValid) {
        formStatus.textContent = 'Form submitted successfully!';
        alert('Form submitted!');
        form.reset();
        
        fields.forEach(field => {
          clearError(field);
        });
      } else {
        formStatus.textContent = 'Please fix errors before submitting';
        // Focus first invalid field
        const firstInvalid = form.querySelector('[aria-invalid="true"]');
        if (firstInvalid) firstInvalid.focus();
      }
    });
  </script>
</body>
</html>
```

## 10.5 Color & Contrast

**WCAG requires sufficient color contrast ratios.**

**WCAG AA (Minimum):**
- Normal text: **4.5:1**

- Normal text: **7:1**
- Large text: **4.5:1**

```html
      // Deactivate all tabs
```

## Key Takeaways from Chapter 10

### WCAG Principles (POUR):
1. **Perceivable** - Can users perceive the content?
2. **Operable** - Can users operate the interface?
3. **Understandable** - Can users understand the content?
4. **Robust** - Does it work with assistive technologies?

### Essential Practices:
5. **Use semantic HTML** - Built-in accessibility
6. **Proper heading hierarchy** - One h1, don't skip levels
7. **All images need alt text** - Descriptive, not generic
8. **All form inputs need labels** - Associated with for/id
9. **Keyboard accessible** - Tab, Enter, Space, Escape work
10. **Visible focus indicators** - Never remove outlines without replacement

### ARIA:
11. **Native HTML first** - ARIA is a supplement, not replacement
12. **aria-label** - When no visible label exists
13. **aria-describedby** - Additional context/instructions
14. **aria-live** - Announce dynamic changes
15. **aria-invalid** - Mark invalid form fields

### Color & Contrast:
16. **4.5:1 contrast ratio** - Normal text (WCAG AA)
17. **Don't rely on color alone** - Use icons + text
18. **Test with tools** - WebAIM, Chrome DevTools

### Testing:
19. **Test with keyboard only** - Unplug mouse, use Tab key
20. **Test with screen reader** - NVDA (free), JAWS, VoiceOver
21. **Use automated tools** - Lighthouse, axe, WAVE
22. **Test at 200% zoom** - Content should still be usable

### Forms:
23. **Group related fields** - Use fieldset and legend
24. **Mark required fields** - Visually and with aria-required
25. **Provide clear error messages** - Specific, actionable
26. **Associate errors with fields** - aria-describedby

### Media:
27. **Captions for videos** - Required for deaf users
28. **Transcripts for audio** - Text alternative
29. **Audio descriptions** - For blind users

### General:
30. **Accessibility is not optional** - It's a legal requirement
31. **Good a11y = good UX** - Everyone benefits
32. **Test with real users** - Nothing beats real feedback
