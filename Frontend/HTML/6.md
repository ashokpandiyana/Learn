# Chapter 6: Multimedia - In-Depth Explanation

## Introduction: Multimedia on the Web

Multimedia elements (images, audio, video) are essential for modern web experiences. This chapter covers everything you need to know about implementing multimedia content properly.

**Key Goals:**
- Accessibility for all users
- Fast loading and performance
- Responsive across devices
- SEO optimization
- Browser compatibility

## 6.1 Images

### The `<img>` Element

The most common way to display images on the web.

**Basic Syntax:**

```html
<img src="photo.jpg" alt="Description of image">
```

**Two Required Attributes:**
1. **`src`** - Source (URL) of the image
2. **`alt`** - Alternative text (for accessibility)

---

### The `src` Attribute - Image Source

Specifies where the image is located.

**Types of URLs:**

```html
<!-- Relative URL (same directory) -->
<img src="photo.jpg" alt="Photo">

<!-- Relative URL (subdirectory) -->
<img src="images/photo.jpg" alt="Photo">

<!-- Absolute URL (external) -->
<img src="https://example.com/photo.jpg" alt="Photo">

<!-- Root-relative URL -->
<img src="/assets/images/photo.jpg" alt="Photo">

<!-- Data URL (base64 encoded) -->
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUg..." alt="Icon">
```

**Best Practices:**

```html
<!-- ✅ Good: Organized structure -->
<img src="/images/products/laptop.jpg" alt="Gaming laptop">

<!-- ✅ Good: Descriptive filename -->
<img src="/team/john-doe-ceo.jpg" alt="John Doe, CEO">

<!-- ❌ Bad: Generic filename -->
<img src="IMG_1234.jpg" alt="Image">

<!-- ❌ Bad: Absolute URL for own images -->
<img src="https://mysite.com/images/logo.png" alt="Logo">
<!-- Use relative URLs for your own site -->
```

---

### The `alt` Attribute - Alternative Text

**Critical for Accessibility and SEO!**

Provides text alternative when:
- Image fails to load
- User is using screen reader
- Search engine crawlers
- Text-only browsers

**How to Write Good Alt Text:**

```html
<!-- ✅ GOOD: Descriptive and specific -->
<img src="golden-retriever.jpg" alt="Golden retriever playing with a red ball in a park">

<!-- ✅ GOOD: Context matters -->
<img src="john-doe.jpg" alt="John Doe, Senior Software Engineer">

<!-- ✅ GOOD: Decorative images (empty alt) -->
<img src="decorative-border.png" alt="">

<!-- ❌ BAD: Too vague -->
<img src="photo.jpg" alt="Photo">

<!-- ❌ BAD: Redundant -->
<img src="sunset.jpg" alt="Image of a sunset">
<!-- Just say "Sunset over ocean" -->

<!-- ❌ BAD: Missing alt (accessibility fail!) -->
<img src="photo.jpg">

<!-- ❌ BAD: "Image" or "picture" in alt text -->
<img src="logo.jpg" alt="Logo image">
<!-- Screen readers already say "image" -->
```

**Special Cases:**

```html
<!-- Decorative images: Use empty alt -->
<img src="divider-line.png" alt="">

<!-- Functional images (links): Describe function -->
<a href="/home">
  <img src="home-icon.svg" alt="Go to home page">
</a>

<!-- Complex images: Brief alt + detailed description -->
<img 
  src="sales-chart.png" 
  alt="Sales chart showing 50% growth in Q4"
  aria-describedby="chart-description"
>
<p id="chart-description">
  Detailed description: The chart shows quarterly sales...
</p>

<!-- Text in images: Include the text in alt -->
<img src="sale-banner.jpg" alt="50% Off Sale - Limited Time Only">

<!-- Charts/Graphs: Summarize the data -->
<img src="pie-chart.jpg" alt="Market share: Apple 30%, Samsung 25%, Others 45%">
```

**Alt Text Length:**

```html
<!-- ✅ Short and descriptive (ideal) -->
<img src="cat.jpg" alt="Tabby cat sleeping on windowsill">

<!-- ⚠️ Long but necessary -->
<img 
  src="infographic.jpg" 
  alt="Infographic showing 5 steps to start a business: 1. Research market, 2. Write business plan, 3. Secure funding, 4. Register business, 5. Launch marketing campaign"
>

<!-- ✅ Better: Long descriptions separate -->
<img 
  src="complex-diagram.jpg" 
  alt="Network architecture diagram"
  aria-describedby="diagram-details"
>
<div id="diagram-details">
  <p>The diagram shows three layers...</p>
</div>
```

### The `width` and `height` Attributes

Specify image dimensions in pixels.

**Why Use Width and Height:**

1. **Prevent Layout Shift** - Browser reserves space before image loads
2. **Better Performance** - Page renders faster
3. **Avoid Content Jumping** - Improves user experience (Core Web Vitals)

```html
<!-- ✅ Good: Specified dimensions -->
<img 
  src="photo.jpg" 
  alt="Photo" 
  width="800" 
  height="600"
>

<!-- ❌ Bad: No dimensions -->
<img src="photo.jpg" alt="Photo">
```

**Visual Comparison:**

```
Without width/height:
┌────────────────┐
│ Text loads     │ ← Content here
│ [Image loads]  │ ← Then image loads and pushes content down
│ Text jumps ↓   │ ← Annoying!
└────────────────┘

With width/height:
┌────────────────┐
│ Text loads     │
│ [Space for     │ ← Browser reserves space
│  image]        │ ← Image loads into reserved space
│ Text stays     │ ← No jump!
└────────────────┘
```

**Responsive Images with CSS:**

```html
<!-- HTML: Set natural dimensions -->
<img 
  src="photo.jpg" 
  alt="Photo" 
  width="1200" 
  height="800"
  style="max-width: 100%; height: auto;"
>

<!-- CSS: Make responsive -->
<style>
  img {
    max-width: 100%;
    height: auto;
    display: block;
  }
</style>
```

**Aspect Ratio Preserved:**

```html
<img 
  src="photo.jpg" 
  alt="Photo" 
  width="1200" 
  height="800"
>
<!-- Aspect ratio: 1200:800 = 3:2 -->

<!-- CSS scales proportionally -->
<style>
  img {
    width: 100%;
    height: auto; /* Maintains 3:2 ratio */
  }
</style>
```

### The `loading` Attribute - Lazy Loading

Defers loading of off-screen images until user scrolls near them.

```html
<!-- Eager loading (default) - loads immediately -->
<img src="hero-image.jpg" alt="Hero" loading="eager">

<!-- Lazy loading - loads when near viewport -->
<img src="image-1.jpg" alt="Image 1" loading="lazy">
<img src="image-2.jpg" alt="Image 2" loading="lazy">
<img src="image-3.jpg" alt="Image 3" loading="lazy">
```

**When to Use:**

```html
<!-- ✅ Eager: Above the fold (immediately visible) -->
<img src="hero.jpg" alt="Hero" loading="eager">

<!-- ✅ Lazy: Below the fold (not immediately visible) -->
<img src="gallery-1.jpg" alt="Gallery" loading="lazy">
<img src="gallery-2.jpg" alt="Gallery" loading="lazy">

<!-- ✅ Lazy: Image galleries -->
<!-- ✅ Lazy: Long articles with many images -->
<!-- ✅ Lazy: Product listings -->
```

**Performance Benefits:**

```
Without lazy loading:
- Downloads all 50 images immediately
- Slower initial page load
- Wastes bandwidth

With lazy loading:
- Downloads only visible images
- Faster initial page load
- Downloads others as user scrolls
- Saves bandwidth
```

**Browser Support:**

```html
<!-- Modern browsers: Native lazy loading -->
<img src="image.jpg" alt="Image" loading="lazy">

<!-- Fallback for old browsers: Use JavaScript library -->
<script src="lazysizes.min.js"></script>
<img data-src="image.jpg" alt="Image" class="lazyload">
```

---

### The `decoding` Attribute

Hints how browser should decode image.

```html
<!-- sync: Decode synchronously (blocks page render) -->
<img src="important.jpg" alt="Important" decoding="sync">

<!-- async: Decode asynchronously (doesn't block render) -->
<img src="large-image.jpg" alt="Large" decoding="async">

<!-- auto: Browser decides (default) -->
<img src="image.jpg" alt="Image" decoding="auto">
```

**When to Use:**

```html
<!-- ✅ sync: Small, critical images -->
<img src="logo.png" alt="Logo" decoding="sync">

<!-- ✅ async: Large, non-critical images -->
<img src="high-res-photo.jpg" alt="Photo" decoding="async">
```

### Responsive Images - `srcset` and `sizes`

Serve different images based on screen size and resolution.

**Problem:** Same image for all devices wastes bandwidth.

```html
<!-- ❌ Bad: 3000px image on mobile -->
<img src="huge-image.jpg" alt="Image">
<!-- Mobile downloads huge file it doesn't need! -->
```

**Solution 1: Different Resolutions (Pixel Density)**

```html
<img 
  src="photo-1x.jpg"
  srcset="
    photo-1x.jpg 1x,
    photo-2x.jpg 2x,
    photo-3x.jpg 3x
  "
  alt="Photo"
  width="400"
  height="300"
>
```

**How it works:**
- `1x` - Normal screens (96 DPI)
- `2x` - Retina screens (192 DPI)
- `3x` - High-density screens (288 DPI)

Browser automatically picks the best one based on screen.

---

**Solution 2: Different Sizes (Width Descriptors)**

```html
<img 
  srcset="
    small.jpg 400w,
    medium.jpg 800w,
    large.jpg 1200w,
    xlarge.jpg 1600w
  "
  sizes="
    (max-width: 600px) 400px,
    (max-width: 1000px) 800px,
    1200px
  "
  src="medium.jpg"
  alt="Responsive image"
>
```

**Breaking it down:**

```html
srcset="
  small.jpg 400w,    ← Image is 400px wide
  medium.jpg 800w,   ← Image is 800px wide
  large.jpg 1200w    ← Image is 1200px wide
"

sizes="
  (max-width: 600px) 400px,   ← On screens ≤600px, image displays at 400px
  (max-width: 1000px) 800px,  ← On screens ≤1000px, image displays at 800px
  1200px                      ← On larger screens, image displays at 1200px
"
```

**Browser Logic:**

1. Check screen size
2. Look at `sizes` to determine display size
3. Pick appropriate image from `srcset`
4. Consider pixel density too

**Real-World Example:**

```html
<img 
  srcset="
    hero-400.jpg 400w,
    hero-800.jpg 800w,
    hero-1200.jpg 1200w,
    hero-1600.jpg 1600w,
    hero-2000.jpg 2000w
  "
  sizes="
    (max-width: 600px) 100vw,
    (max-width: 1200px) 50vw,
    33vw
  "
  src="hero-800.jpg"
  alt="Hero image"
  width="1200"
  height="800"
>
```

**Translation:**
- Mobile (≤600px): Image takes 100% viewport width
- Tablet (≤1200px): Image takes 50% viewport width
- Desktop (>1200px): Image takes 33% viewport width

## 6.2 Image Formats

Choosing the right image format is crucial for performance.

### Raster vs Vector

**Raster Images (Pixel-based):**
- Made of pixels (dots)
- Lose quality when scaled up
- Formats: JPEG, PNG, GIF, WebP, AVIF
- Use for: Photos, complex images

**Vector Images (Math-based):**
- Made of paths and shapes
- Scale infinitely without quality loss
- Formats: SVG
- Use for: Logos, icons, illustrations

**Visual Comparison:**

```
Raster (Zoom In):          Vector (Zoom In):
██████                     ╱────╲
██████  ← Pixelated       │    │  ← Still smooth
██████                     ╲────╱
```

---

### JPEG (Joint Photographic Experts Group)

**Best for:** Photographs, complex images with many colors

**Pros:**
- ✅ Small file size
- ✅ Millions of colors
- ✅ Excellent compression
- ✅ Universal browser support

**Cons:**
- ❌ Lossy compression (quality loss)
- ❌ No transparency
- ❌ Not ideal for text or sharp edges

```html
<img src="photo.jpg" alt="Photograph" width="800" height="600">
```

**When to use:**
- Photographs
- Product images
- Hero images
- Background images

---

### PNG (Portable Network Graphics)

**Best for:** Graphics with transparency, logos, icons, screenshots

**Pros:**
- ✅ Lossless compression
- ✅ Transparency support (alpha channel)
- ✅ Great for text and sharp edges
- ✅ Universal support

**Cons:**
- ❌ Larger file size than JPEG
- ❌ Not ideal for photos

```html
<img src="logo.png" alt="Company Logo" width="200" height="60">
```

**Two types:**
- **PNG-8**: 256 colors, smaller file size
- **PNG-24**: Millions of colors, larger file size

**When to use:**
- Logos
- Icons
- Images with transparency
- Screenshots with text
- Graphics with sharp edges

---

### GIF (Graphics Interchange Format)

**Best for:** Simple animations

**Pros:**
- ✅ Supports animation
- ✅ Small file size (for simple images)
- ✅ Universal support

**Cons:**
- ❌ Only 256 colors
- ❌ Poor compression for photos
- ❌ Outdated for most uses

```html
<img src="animation.gif" alt="Loading animation" width="50" height="50">
```

**When to use:**
- Simple animations (loading spinners)
- Very simple graphics

**⚠️ For complex animations, use video instead!**

---

### WebP

**Best for:** Modern web - photos and graphics

**Pros:**
- ✅ 25-35% smaller than JPEG/PNG
- ✅ Supports transparency
- ✅ Supports animation
- ✅ Both lossy and lossless

**Cons:**
- ❌ Not supported in very old browsers (IE, old Safari)

```html
<img src="photo.webp" alt="Modern photo" width="800" height="600">
```

**With Fallback:**

```html
<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Image with fallback">
</picture>
```

**Browser Support:**
- ✅ Chrome 23+
- ✅ Firefox 65+
- ✅ Edge 18+
- ✅ Safari 14+
- ❌ IE (use fallback)

---

### AVIF (AV1 Image File Format)

**Best for:** Next-generation images (newest format)

**Pros:**
- ✅ 50% smaller than JPEG
- ✅ Better than WebP
- ✅ Supports HDR
- ✅ Transparency support

**Cons:**
- ❌ Limited browser support (cutting edge)
- ❌ Slower encoding/decoding

```html
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Next-gen image with fallbacks">
</picture>
```

**Browser Support:**
- ✅ Chrome 85+
- ✅ Firefox 93+
- ✅ Safari 16+
- ❌ Older browsers (use fallbacks)

---

### Format Comparison Table

| Format | Best For | Transparency | Animation | File Size | Browser Support |
|--------|----------|--------------|-----------|-----------|----------------|
| **JPEG** | Photos | ❌ No | ❌ No | Small | ✅ Universal |
| **PNG** | Logos, Graphics | ✅ Yes | ❌ No | Large | ✅ Universal |
| **GIF** | Simple Animations | Partial | ✅ Yes | Small | ✅ Universal |
| **WebP** | Modern Web | ✅ Yes | ✅ Yes | Very Small | ✅ Modern |
| **AVIF** | Next-Gen | ✅ Yes | ✅ Yes | Smallest | ⚠️ Limited |
| **SVG** | Icons, Logos | ✅ Yes | ✅ Yes | Tiny | ✅ Universal |

---

### Choosing the Right Format - Decision Tree

```
Start: What type of image?
    │
    ├─ Photo/Complex Image?
    │   │
    │   ├─ Need transparency? → PNG
    │   │
    │   └─ No transparency?
    │       │
    │       ├─ Modern browsers only? → AVIF (with WebP fallback)
    │       ├─ Modern + older browsers? → WebP (with JPEG fallback)
    │       └─ Maximum compatibility? → JPEG
    │
    ├─ Logo/Icon/Simple Graphics?
    │   │
    │   ├─ Can use vector? → SVG
    │   └─ Need raster? → PNG (or WebP with PNG fallback)
    │
    └─ Animation?
        │
        ├─ Complex animation? → Video (MP4)
        ├─ Simple spinner/icon? → GIF or SVG animation
        └─ Modern animation? → WebP or AVIF
```

## 6.3 The `<picture>` Element

Provides multiple image sources for different scenarios.

**Use Cases:**
1. **Art Direction** - Different images for different screen sizes
2. **Format Fallbacks** - Modern formats with fallbacks
3. **Resolution Switching** - Different resolutions for different screens

**Basic Structure:**

```html
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Fallback image">
</picture>
```

**How it works:**
1. Browser checks each `<source>` from top to bottom
2. Uses first one that matches and is supported
3. Falls back to `<img>` if none match

---

### Format Fallbacks

Serve modern formats with legacy fallbacks.

```html
<picture>
  <!-- Try AVIF first (smallest, newest) -->
  <source srcset="photo.avif" type="image/avif">
  
  <!-- Try WebP second (small, modern) -->
  <source srcset="photo.webp" type="image/webp">
  
  <!-- Fallback to JPEG (universal) -->
  <img src="photo.jpg" alt="Photo" width="800" height="600">
</picture>
```

**Result:**
- Modern browsers: Get AVIF (smallest)
- Older modern browsers: Get WebP
- Old browsers: Get JPEG

---

### Responsive Images (Art Direction)

Different images for different screen sizes.

```html
<picture>
  <!-- Mobile: Portrait crop -->
  <source 
    media="(max-width: 600px)" 
    srcset="mobile-portrait.jpg"
  >
  
  <!-- Tablet: Square crop -->
  <source 
    media="(max-width: 1200px)" 
    srcset="tablet-square.jpg"
  >
  
  <!-- Desktop: Landscape crop -->
  <img src="desktop-landscape.jpg" alt="Responsive image">
</picture>
```

**Visual Example:**

```
Mobile (≤600px):          Tablet (≤1200px):        Desktop (>1200px):
┌────────┐                ┌──────────┐              ┌─────────────────┐
│        │                │          │              │                 │
│ Portrait│               │  Square  │              │   Landscape     │
│        │                │          │              │                 │
│        │                └──────────┘              └─────────────────┘
└────────┘
```

---

### Art Direction + Resolution

Combine art direction with resolution switching.

```html
<picture>
  <!-- Mobile: Portrait, multiple resolutions -->
  <source 
    media="(max-width: 600px)"
    srcset="
      mobile-portrait-400.jpg 400w,
      mobile-portrait-800.jpg 800w
    "
    sizes="100vw"
  >
  
  <!-- Tablet: Square, multiple resolutions -->
  <source 
    media="(max-width: 1200px)"
    srcset="
      tablet-square-600.jpg 600w,
      tablet-square-1200.jpg 1200w
    "
    sizes="100vw"
  >
  
  <!-- Desktop: Landscape, multiple resolutions -->
  <source 
    srcset="
      desktop-landscape-1200.jpg 1200w,
      desktop-landscape-1600.jpg 1600w,
      desktop-landscape-2000.jpg 2000w
    "
    sizes="100vw"
  >
  
  <!-- Fallback -->
  <img 
    src="desktop-landscape-1200.jpg" 
    alt="Responsive image with art direction"
    width="1200"
    height="800"
  >
</picture>
```

## 6.6 Audio

The `<audio>` element plays audio files.

**Basic Syntax:**

```html
<audio src="audio.mp3" controls></audio>
```

**Common Attributes:**
- `controls` - Show playback controls
- `autoplay` - Start playing automatically (often blocked by browsers)
- `loop` - Repeat when finished
- `muted` - Start muted
- `preload` - Hint for loading strategy

---

### Audio with Controls

```html

```

**Multiple Sources:**
- Browser picks first format it supports
- Fallback text for very old browsers

---

### Audio Formats

| Format | MIME Type | Browser Support |
|--------|-----------|----------------|
| **MP3** | audio/mpeg | ✅ Universal (best choice) |
| **WAV** | audio/wav | ✅ Universal (large files) |
| **OGG** | audio/ogg | ✅ Most (not Safari/IE) |
| **AAC** | audio/aac | ✅ Most modern browsers |

**Recommendation:** Use MP3 for best compatibility.

---

### Audio Attributes

```html
<audio 
  src="audio.mp3"
  controls
  autoplay
  loop
  muted
  preload="auto"
>
</audio>
```

**Attribute Details:**

```html
<!-- controls: Show play/pause, volume, etc. -->
<audio src="audio.mp3" controls></audio>

<!-- autoplay: Start playing automatically -->
<!-- ⚠️ Often blocked by browsers unless muted -->
<audio src="audio.mp3" autoplay muted></audio>

<!-- loop: Repeat indefinitely -->
<audio src="background-music.mp3" loop></audio>

<!-- muted: Start with volume at 0 -->
<audio src="audio.mp3" muted></audio>

<!-- preload options: -->
<audio src="audio.mp3" preload="none"></audio>     <!-- Don't preload -->
<audio src="audio.mp3" preload="metadata"></audio> <!-- Load metadata only -->
```

### Custom Audio Player

```html

      border-radius: 4px;
      cursor: pointer;
    }
    
    button:hover {
      background: #0052a3;
    }
    
    input[type="range"] {
      flex: 1;
    }
    
    .time {
      font-family: monospace;
    }
  </style>
</head>
<body>
  <div class="audio-player">
    <h3>Audio Player</h3>
    <audio id="audio" src="audio.mp3"></audio>
    
    <div class="controls">
      <button id="playPause">▶ Play</button>
      <span class="time" id="currentTime">0:00</span>
      <input type="range" id="progress" value="0" step="0.1">
      <span class="time" id="duration">0:00</span>
      <input type="range" id="volume" min="0" max="100" value="100" style="width: 80px;">
    </div>

  <script>
    const audio = document.getElementById('audio');
    const playPause = document.getElementById('playPause');
    const progress = document.getElementById('progress');
    const volume = document.getElementById('volume');
    const currentTime = document.getElementById('currentTime');
    const duration = document.getElementById('duration');
    
    // Play/Pause
    playPause.addEventListener('click', () => {
      if (audio.paused) {
        audio.play();
        playPause.textContent = '⏸ Pause';
      } else {
        audio.pause();
        playPause.textContent = '▶ Play';
      }
    });
    
    // Update progress
    audio.addEventListener('timeupdate', () => {
      progress.value = (audio.currentTime / audio.duration) * 100;
      currentTime.textContent = formatTime(audio.currentTime);
    });
    
    // Set duration when loaded
    audio.addEventListener('loadedmetadata', () => {
      duration.textContent = formatTime(audio.duration);
      progress.max = 100;
    });
    
    // Seek
    progress.addEventListener('input', () => {
      audio.currentTime = (progress.value / 100) * audio.duration;
    });
    
    // Volume
    volume.addEventListener('input', () => {
      audio.volume = volume.value / 100;
    });
    
    // Format time (seconds to mm:ss)
    function formatTime(seconds) {
      const mins = Math.floor(seconds / 60);
      const secs = Math.floor(seconds % 60);
      return `${mins}:${secs.toString().padStart(2, '0')}`;
    }
  </script>
</body>
</html>
```

```html
    src="subtitles-fr.vtt" 
    srclang="fr" 
    label="Français"
  >
</video>
```

### WebVTT Format

Subtitle files use WebVTT (Web Video Text Tracks) format.

**Example: subtitles-en.vtt**

```

00:00:00.000 --> 00:00:03.000
Hello, welcome to our video.

00:00:03.500 --> 00:00:07.000
In this tutorial, we'll learn about HTML5 video.

Let's get started with the basics.
```

**Format:**
- `WEBVTT` - Header (required)
- Timestamps in format: `HH:MM:SS.mmm`
- Text appears between timestamps

## 6.9 Embedded Content - `<iframe>`

Embeds external content (websites, videos, maps) into your page.

**Basic Syntax:**

```html
<iframe src="https://example.com"></iframe>
```

⚠️ **Security Warning:** iframes can be dangerous! Use carefully.

---

### iframe Attributes

```html
<iframe 
  src="https://example.com"
  width="800"
  height="600"
  title="Example website"
  loading="lazy"
  sandbox="allow-scripts allow-same-origin"
  allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture"
></iframe>
```

---

### The `sandbox` Attribute (Security)

Restricts what iframe can do.

```html
<!-- No restrictions (dangerous!) -->
<iframe src="https://example.com"></iframe>

<!-- All restrictions (safest) -->
<iframe src="https://example.com" sandbox></iframe>

<!-- Allow specific features -->
<iframe 
  src="https://example.com"
  sandbox="allow-scripts allow-same-origin"
></iframe>
```

**Sandbox Values:**
- `allow-scripts` - Allow JavaScript
- `allow-same-origin` - Allow same-origin access
- `allow-forms` - Allow form submission
- `allow-popups` - Allow pop-ups
- `allow-top-navigation` - Allow navigation of top window

**Recommended:**

```html
<!-- Safe for untrusted content -->
<iframe 
  src="https://untrusted-site.com"
  sandbox="allow-scripts"
></iframe>

<!-- For trusted embeds (YouTube, etc.) -->
<iframe 
  src="https://www.youtube.com/embed/..."
  sandbox="allow-scripts allow-same-origin"
></iframe>
```

---

### Embedding YouTube Videos

```html
<iframe 
  width="560" 
  height="315" 
  src="https://www.youtube.com/embed/VIDEO_ID" 
  title="YouTube video player"
  frameborder="0"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
  allowfullscreen
></iframe>
```

**YouTube Parameters:**

```html
<!-- Autoplay (muted required) -->
<iframe src="https://www.youtube.com/embed/VIDEO_ID?autoplay=1&mute=1"></iframe>

<!-- Loop -->
<iframe src="https://www.youtube.com/embed/VIDEO_ID?loop=1&playlist=VIDEO_ID"></iframe>

<!-- No controls -->
<iframe src="https://www.youtube.com/embed/VIDEO_ID?controls=0"></iframe>

<!-- Start at specific time (90 seconds) -->
<iframe src="https://www.youtube.com/embed/VIDEO_ID?start=90"></iframe>
```

## Key Takeaways from Chapter 6

### Images:
1. **Always provide `alt` text** - Critical for accessibility and SEO
2. **Specify `width` and `height`** - Prevents layout shift (CLS)
3. **Use `loading="lazy"`** - For images below the fold
4. **Responsive images** - Use `srcset` and `sizes` for different screens
5. **Modern formats** - Use WebP/AVIF with fallbacks
6. **Use `<picture>`** - For art direction and format fallbacks

### SVG:
7. **SVG for logos and icons** - Scales perfectly, small file size
8. **Inline SVG** - Most flexible, can style with CSS
9. **Optimize SVGs** - Remove unnecessary metadata

### Audio/Video:
10. **Always use `controls`** - Users need playback controls
11. **Provide multiple formats** - MP4 + WebM for video
12. **Add subtitles** - Use `<track>` for accessibility
13. **Use `poster`** - Thumbnail for videos
14. **`autoplay` needs `muted`** - Browsers block unmuted autoplay

### iframes:
15. **Use `sandbox`** - Security for untrusted content
16. **Responsive iframes** - Use aspect ratio padding trick
17. **`loading="lazy"`** - For iframes below the fold

### Performance:
18. **Compress all media** - Smaller files = faster loading
19. **Use CDN** - For faster delivery
20. **Lazy load** - Don't load what users can't see

Master multimedia and your sites will be accessible, performant, and engaging! 🎬📸🎵
