## 1.1 Document Structure

### Understanding DOCTYPE Declaration

The DOCTYPE declaration is the **very first thing** in your HTML document. It's not an HTML tagâ€”it's an instruction to the web browser about what version of HTML the page is written in.

**Why DOCTYPE is Critical:**

1. **Prevents Quirks Mode**: Without a DOCTYPE, browsers render pages in "quirks mode" (backward-compatible mode), which can cause unpredictable layout issues
2. **Ensures Standards Mode**: With DOCTYPE, browsers use standards-compliant rendering
3. **Cross-browser Consistency**: Helps ensure your page looks the same across different browsers

**HTML5 DOCTYPE (Current Standard):**

```html
<!DOCTYPE html>
```

**Why HTML5's DOCTYPE is Simple:**

Previous HTML versions had complex DOCTYPEs:

```html
<!-- HTML 4.01 Strict (old, don't use) -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

<!-- XHTML 1.0 (old, don't use) -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
```

HTML5 simplified this to just `<!DOCTYPE html>` - it's case-insensitive, but lowercase is conventional.

### The HTML Element (`<html>`)

The `<html>` element is the **root element** that wraps all other content on your page. Think of it as the container for your entire document.

**Basic Structure:**

```html
<!DOCTYPE html>
<html lang="en">
  <!-- Everything else goes here -->
</html>
```

**The `lang` Attribute - Why It Matters:**

The `lang` attribute specifies the language of the document's content. This is crucial for:

1. **Screen Readers**: Helps assistive technologies pronounce content correctly
2. **Search Engines**: Helps with regional search results
3. **Translation Tools**: Browsers can offer to translate pages
4. **Styling**: CSS can apply different styles based on language

**Common Language Codes:**

```html
<html lang="en">     <!-- English -->
<html lang="es">     <!-- Spanish -->
<html lang="fr">     <!-- French -->
<html lang="de">     <!-- German -->
<html lang="ja">     <!-- Japanese -->
<html lang="zh">     <!-- Chinese -->
<html lang="ar">     <!-- Arabic -->
<html lang="hi">     <!-- Hindi -->
<html lang="en-US">  <!-- English (United States) -->
<html lang="en-GB">  <!-- English (United Kingdom) -->
```

### The Head Section (`<head>`)

The `<head>` element contains **metadata**â€”information about the document that isn't displayed directly on the page but is essential for browsers, search engines, and other services.

**What Goes in the Head:**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">
  
  <!-- Viewport for responsive design -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Page title (appears in browser tab) -->
  <title>My Awesome Website</title>
  
  <!-- Meta tags for SEO -->
  <meta name="description" content="A description of my website">
  <meta name="keywords" content="html, web development, tutorial">
  <meta name="author" content="Your Name">
  
  <!-- Favicon -->
  <link rel="icon" href="favicon.ico">
  
  <!-- CSS Stylesheets -->
  <link rel="stylesheet" href="styles.css">
  
  <!-- JavaScript files -->
  <script src="script.js" defer></script>
  
  <!-- Open Graph for social media -->
  <meta property="og:title" content="My Awesome Website">
  <meta property="og:description" content="Check out my website!">
  <meta property="og:image" content="preview-image.jpg">
</head>
<body>
  <!-- Visible content goes here -->
</body>
</html>
```

**Key Points:**

- Nothing in `<head>` is visible on the page itself
- Order matters for some elements (charset should be first)
- Multiple `<link>` and `<script>` tags are allowed
- Only ONE `<title>` tag per page

### The Body Section (`<body>`)

The `<body>` element contains **all visible content** that users see and interact with.

**Complete Example:**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Complete HTML Structure</title>
</head>
<body>
  <!-- Header section -->
  <header>
    <h1>Welcome to My Website</h1>
    <nav>
      <a href="#home">Home</a>
      <a href="#about">About</a>
      <a href="#contact">Contact</a>
    </nav>
  </header>
  
  <!-- Main content -->
  <main>
    <h2>Main Content Area</h2>
    <p>This is where your visible content goes.</p>
  </main>
  
  <!-- Footer -->
  <footer>
    <p>&copy; 2024 My Website</p>
  </footer>
</body>
</html>
```

**Important Rules:**

- Only ONE `<body>` element per document
- All visible content must be inside `<body>`
- `<body>` comes after `<head>`, never inside it

### SEO Meta Tags

**Description Meta Tag:**

```html

```

**Best Practices:**

- Unique for each page

- Include relevant keywords naturally
- Think of it as your "ad copy" in search results

**Example of Search Result:**

**Keywords Meta Tag:**

```html

```

âš ï¸ **Note**: Google doesn't use this anymore, but some search engines still do. Use sparingly.

**Author Meta Tag:**

```html
<meta name="author" content="John Doe">
```

**Robots Meta Tag:**

```html
<!-- Tell search engines to index this page and follow links -->
<meta name="robots" content="index, follow">

<!-- Don't index this page (e.g., admin pages) -->
<meta name="robots" content="noindex, nofollow">

<!-- Index but don't follow links -->
<meta name="robots" content="index, nofollow">
```

**Values:**
- `index`: Allow search engines to index this page
- `noindex`: Don't index this page
- `follow`: Follow links on this page
- `nofollow`: Don't follow links on this page

## 1.3 Title & Favicon

The `<title>` tag is one of the most important elements for both SEO and user experience.

```html

```

**Where Title Appears:**

1. **Browser Tab**: Shows in the browser tab/window
2. **Search Results**: Main clickable link in Google
3. **Bookmarks**: Default bookmark name
4. **Social Shares**: Sometimes used in social previews

**Title Best Practices:**

âœ… **Good Titles:**

## 1.4 Block vs Inline Elements

Understanding the difference between block and inline elements is fundamental to HTML layout.

### Block-Level Elements

**Characteristics:**

- Start on a **new line**
- Take up the **full width** available (100% of parent)
- Can have width and height set
- Can contain other block or inline elements (with some exceptions)
- Stack vertically by default

- Can only contain text and other inline elements

### Block vs Inline Comparison

| Feature | Block | Inline |
|---------|-------|--------|
| **New Line** | Yes, always starts new line | No, stays in line |
| **Width** | Takes full width (100%) | Only width of content |
| **Height** | Can be set | Cannot be set |
| **Width** | Can be set | Cannot be set |
| **Margin** | All sides work | Only left/right work |
| **Padding** | All sides work | All sides work (but vertical may overlap) |
| **Contains** | Block or inline elements | Only inline elements |
| **Examples** | `<div>`, `<p>`, `<h1>`, `<section>` | `<span>`, `<a>`, `<strong>`, `<img>` |

### Nesting Rules

**âœ… ALLOWED:**

```html
<!-- Block can contain block -->
<div>
  <p>Paragraph inside div</p>
  <section>Section inside div</section>
</div>

<!-- Block can contain inline -->
<p>This is <strong>bold text</strong> in a paragraph</p>

<!-- Inline can contain inline -->
<span>This is <strong>bold</strong> inside span</span>
```

**âŒ GENERALLY NOT ALLOWED:**

```html
<!-- Inline cannot contain block (invalid HTML) -->

  <div>This is wrong!</div>
</span>

<!-- Paragraph cannot contain paragraph -->
<p>
  <p>This is wrong!</p>
</p>

<!-- Links cannot contain other links -->
<a href="#">
  <a href="#">This is wrong!</a>
</a>
```

**âš ï¸ SPECIAL EXCEPTIONS:**

```html
<!-- Links CAN contain block elements in HTML5 -->

  <div class="product-card">
    <img src="product.jpg" alt="Product">
    <h3>Product Name</h3>
    <p>Description</p>
  </div>

<!-- This is valid in HTML5, but not in older versions -->
```

## 1.5 HTML Comments

Comments are notes in your code that browsers ignore completely. They're only visible when viewing the source code.

### Basic Comment Syntax

```html
<!-- This is a comment -->

<!-- 
  This is a 
  multi-line
  comment
-->
```

**Important Rules:**

- Comments start with `<!--` and end with `-->`
- Everything between is ignored by the browser
- Can span multiple lines
- Cannot be nested (comments inside comments don't work)

---

### Uses of Comments

**1. Documentation:**

```html
<!-- Header Section: Contains logo and main navigation -->
<header>
  <img src="logo.png" alt="Company Logo">
  <nav>
    <!-- Main navigation links -->
    <a href="/">Home</a>
    <a href="/about">About</a>
    <a href="/contact">Contact</a>
  </nav>
</header>
```

**2. TODOs and Notes:**

```html
<!-- TODO: Add search functionality here -->
<div class="search-container">
  <!-- FIXME: Search button not working on mobile -->
  <input type="text" placeholder="Search...">
  <button>Search</button>
</div>

<!-- NOTE: This section needs review before launch -->
<section class="beta-feature">
  <p>New feature coming soon!</p>
</section>
```

**3. Temporary Debugging (Commenting Out Code):**

```html
<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>
  <!-- <a href="/blog">Blog</a> -->  <!-- Temporarily hidden -->
  <a href="/contact">Contact</a>
</nav>
```

**4. Version Control and Changes:**

```html
<!-- Updated: 2024-11-26 - Added new hero section -->
<section class="hero">
  <h1>Welcome</h1>
</section>

<!-- Removed: 2024-11-20 - Old pricing table
<table class="pricing">
  ...
</table>
-->
```

**5. Section Markers:**

```html
<!-- ========================================
     HEADER SECTION
     ======================================== -->
<header>
  <h1>My Website</h1>
</header>

<!-- ========================================
     MAIN CONTENT
     ======================================== -->
<main>
  <p>Content goes here</p>
</main>

<!-- ========================================
     FOOTER SECTION
     ======================================== -->
<footer>
  <p>&copy; 2024</p>
</footer>
```

## Key Takeaways from Chapter 1

1. **Always start with `<!DOCTYPE html>`** - Ensures standards mode
2. **Set `lang` attribute on `<html>`** - Critical for accessibility
3. **Put `charset` FIRST in `<head>`** - Prevents encoding issues
4. **Always include viewport meta tag** - Mobile responsiveness
5. **Write unique, descriptive titles** - SEO and user experience
6. **Block elements stack vertically** - Start new lines, full width
7. **Inline elements flow horizontally** - Stay in line with text
8. **Use comments for documentation** - Not for sensitive information
9. **Meta tags are invisible but essential** - SEO, social media, mobile
10. **One `<head>`, one `<body>`** - Basic document structure

This foundation is critical before moving to advanced HTML concepts!
