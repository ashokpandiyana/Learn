# Chapter 2: Semantic HTML - In-Depth Explanation

## 2.1 Why Semantic HTML Matters

### What is Semantic HTML?

**Semantic HTML** means using HTML elements that clearly describe their meaning and purpose, rather than just their appearance.

**Non-Semantic (Bad):**
```html
<div class="header">
  <div class="navigation">
    <div class="link">Home</div>
    <div class="link">About</div>
  </div>
</div>
```

**Semantic (Good):**
```html
<header>
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
</header>
```

Both might *look* the same with CSS, but the second one **communicates meaning** to browsers, search engines, and assistive technologies.

---

### The Four Pillars of Why Semantic HTML Matters

#### 1. Accessibility (A11y)

**Screen Readers Understand Semantics:**

```html
<!-- Screen reader announces: "navigation landmark" -->
<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>
</nav>

<!-- Screen reader announces: "main content landmark" -->
<main>
  <h1>Welcome to Our Site</h1>
  <p>Content here...</p>
</main>

<!-- vs. Non-semantic (screen reader just reads content) -->
<div>
  <a href="/">Home</a>
  <a href="/about">About</a>
</div>
```

**Screen reader users can:**
- Jump directly to `<nav>` elements
- Skip to `<main>` content
- Navigate by heading levels (`<h1>`, `<h2>`, etc.)
- List all landmarks on the page

**Real Impact:**
- 285 million people worldwide are visually impaired
- Many use screen readers to browse the web
- Semantic HTML makes their experience exponentially better

---

#### 2. Search Engine Optimization (SEO)

**Search engines prioritize semantic structure:**

```html
<!-- Google understands this is a navigation menu -->
<nav>
  <a href="/products">Products</a>
  <a href="/services">Services</a>
</nav>

<!-- Google knows this is the main content -->
<main>
  <article>
    <h1>Best Practices for Web Development</h1>
    <p>Content that Google will prioritize...</p>
  </article>
</main>

<!-- Google knows this is sidebar content (less important) -->
<aside>
  <h2>Related Articles</h2>
</aside>
```

**SEO Benefits:**
- Better content understanding
- Improved ranking signals
- Rich snippets in search results
- Featured snippets eligibility
- Better mobile search rankings

**Example of Rich Snippets:**

```html
<article>
  <h1>How to Bake Chocolate Chip Cookies</h1>
  <p>Published: <time datetime="2024-11-26">November 26, 2024</time></p>
  <p>By: <span itemprop="author">Jane Doe</span></p>
  
  <!-- Google can extract this as a recipe -->
  <section>
    <h2>Ingredients</h2>
    <ul>
      <li>2 cups flour</li>
      <li>1 cup sugar</li>
    </ul>
  </section>
</article>
```

---

#### 3. Code Maintainability

**Easier to Read and Understand:**

```html
<!-- ❌ Hard to understand structure -->
<div class="wrapper">
  <div class="top-section">
    <div class="brand">Logo</div>
    <div class="links">
      <div><a href="/">Home</a></div>
    </div>
  </div>
  <div class="content-area">
    <div class="main-stuff">Content</div>
    <div class="side-stuff">Sidebar</div>
  </div>
  <div class="bottom">Footer</div>
</div>

<!-- ✅ Crystal clear structure -->
<div class="wrapper">
  <header>
    <div class="brand">Logo</div>
    <nav>
      <a href="/">Home</a>
    </nav>
  </header>
  
  <main>
    <article>Content</article>
    <aside>Sidebar</aside>
  </main>
  
  <footer>Footer</footer>
</div>
```

**Benefits for Development Teams:**
- New developers understand code faster
- Less documentation needed
- Easier debugging
- Reduced technical debt

---

#### 4. Future-Proofing

**Standards Evolve, Semantics Persist:**

```html
<!-- This will always work, regardless of future changes -->
<article>
  <header>
    <h1>Article Title</h1>
  </header>
  <section>
    <p>Content...</p>
  </section>
  <footer>
    <p>Author info...</p>
  </footer>
</article>
```

**Why This Matters:**
- Browser updates won't break semantic HTML
- New assistive technologies understand semantic elements
- Web standards build on semantic foundations
- Your code stays relevant for years

### `<header>` - Introductory Content

**Purpose:** Contains introductory content or navigational aids for its parent section.

**Where to Use:**
- Top of the page (site header)
- Top of an `<article>` (article header)
- Top of a `<section>` (section header)

**Typical Contents:**
- Logo
- Site title
- Navigation menu (`<nav>`)
- Search bar
- User account links

**Example 1: Site Header**

```html
<header class="site-header">
  <!-- Logo/Brand -->
  <div class="brand">
    <img src="logo.svg" alt="Company Logo">
    <h1>My Awesome Website</h1>
  </div>
  
  <!-- Main Navigation -->
  <nav class="main-nav">
    <a href="/">Home</a>
    <a href="/products">Products</a>
    <a href="/about">About</a>
    <a href="/contact">Contact</a>
  </nav>
  
  <!-- User Actions -->
  <div class="user-actions">
    <button>Search</button>
    <a href="/login">Login</a>
  </div>
</header>
```

**Example 2: Article Header**

```html
<article>
  <header>
    <h2>Understanding JavaScript Closures</h2>
    <p class="meta">
      Published: <time datetime="2024-11-26">Nov 26, 2024</time>
      By: <span class="author">John Doe</span>
    </p>
    <p class="summary">A deep dive into one of JavaScript's most powerful features.</p>
  </header>
  
  <section>
    <p>Article content goes here...</p>
  </section>
</article>
```

**Example 3: Section Header**

```html
<section class="pricing">
  <header>
    <h2>Our Pricing Plans</h2>
    <p>Choose the plan that's right for you</p>
  </header>
  
  <!-- Pricing cards -->
  <div class="pricing-grid">
    <!-- ... -->
  </div>
</section>
```

**Important Notes:**
- ✅ Can have multiple `<header>` elements per page
- ✅ Can be nested in `<article>`, `<section>`, etc.
- ❌ Cannot be placed inside another `<header>`, `<footer>`, or `<address>`

### `<nav>` - Navigation Links

**Purpose:** Contains navigation links for the site or current page.

**When to Use:**
- Main site navigation
- Table of contents
- Breadcrumbs
- Pagination

**When NOT to Use:**
- Footer links (unless major navigation)
- Social media links (typically just regular links)
- Tags or categories (unless primary navigation)

**Example 1: Main Navigation**

```html
<nav class="primary-navigation" aria-label="Main">
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li>
      <a href="/services">Services</a>
      <!-- Dropdown submenu -->
      <ul>
        <li><a href="/services/consulting">Consulting</a></li>
        <li><a href="/services/development">Development</a></li>
      </ul>
    </li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>
```

**Example 2: Table of Contents**

```html
<article>
  <h1>Complete HTML Guide</h1>
  
  <nav aria-label="Table of contents">
    <h2>Table of Contents</h2>
    <ol>
      <li><a href="#introduction">Introduction</a></li>
      <li><a href="#basics">HTML Basics</a></li>
      <li><a href="#advanced">Advanced Topics</a></li>
      <li><a href="#conclusion">Conclusion</a></li>
    </ol>
  </nav>
  
  <section id="introduction">
    <h2>Introduction</h2>
    <!-- ... -->
  </section>
</article>
```

**Example 3: Breadcrumb Navigation**

```html
<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/products/laptops">Laptops</a></li>
    <li aria-current="page">MacBook Pro</li>
  </ol>
</nav>
```

**Example 4: Pagination**

```html
<nav aria-label="Pagination">
  <ul class="pagination">
    <li><a href="?page=1" aria-label="Previous page">Previous</a></li>
    <li><a href="?page=1">1</a></li>
    <li><a href="?page=2" aria-current="page">2</a></li>
    <li><a href="?page=3">3</a></li>
    <li><a href="?page=3" aria-label="Next page">Next</a></li>
  </ul>
</nav>
```

**Accessibility Best Practices:**

```html
<!-- Use aria-label when you have multiple navs -->
<nav aria-label="Main">
  <a href="/">Home</a>
  <a href="/about">About</a>
</nav>

<nav aria-label="Secondary">
  <a href="/privacy">Privacy</a>
  <a href="/terms">Terms</a>
</nav>

<!-- Use aria-current to indicate current page -->
<nav>
  <a href="/" aria-current="page">Home</a>  <!-- Current page -->
  <a href="/about">About</a>
  <a href="/contact">Contact</a>
</nav>
```

### `<main>` - Primary Content

**Purpose:** Contains the **main content** of the document. There should be only ONE `<main>` per page.

**Critical Rules:**
- ❌ Only ONE `<main>` element per page
- ❌ Cannot be descendant of `<article>`, `<aside>`, `<footer>`, `<header>`, or `<nav>`
- ✅ Should be direct child of `<body>` (or wrapper div)
- ✅ Content that is unique to this page

**What Goes in `<main>`:**
- Primary article content
- Search results
- Product listings
- Dashboard data
- Anything that's the "point" of the page

**What DOESN'T Go in `<main>`:**
- Site header/logo (use `<header>`)
- Site navigation (use `<nav>`)
- Sidebar (use `<aside>`)
- Footer (use `<footer>`)
- Content repeated across pages

**Example 1: Blog Post Page**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>How to Learn HTML</title>
</head>
<body>
  <!-- Site header (NOT in main) -->
  <header>
    <h1>My Blog</h1>
    <nav>
      <a href="/">Home</a>
      <a href="/about">About</a>
    </nav>
  </header>
  
  <!-- MAIN CONTENT - unique to this page -->
  <main>
    <article>
      <h1>How to Learn HTML in 30 Days</h1>
      <p>Published: <time datetime="2024-11-26">November 26, 2024</time></p>
      
      <section>
        <h2>Introduction</h2>
        <p>Learning HTML is the first step...</p>
      </section>
      
      <section>
        <h2>Day 1: Basic Structure</h2>
        <p>Start with the fundamentals...</p>
      </section>
      
      <!-- More content sections -->
    </article>
    
    <!-- Related articles (still part of main content) -->
    <section class="related">
      <h2>Related Articles</h2>
      <ul>
        <li><a href="/css-guide">CSS Guide</a></li>
        <li><a href="/js-basics">JavaScript Basics</a></li>
      </ul>
    </section>
  </main>
  
  <!-- Sidebar (NOT in main) -->
  <aside>
    <h2>About the Author</h2>
    <p>John Doe is a web developer...</p>
  </aside>
  
  <!-- Footer (NOT in main) -->
  <footer>
    <p>&copy; 2024 My Blog</p>
  </footer>
</body>
</html>
```

**Example 2: Product Page**

```html
<body>
  <header><!-- Site header --></header>
  
  <main>
    <!-- Product details - unique to this product -->
    <article>
      <h1>MacBook Pro 16-inch</h1>
```

```html

    <h1>Main Content Starts Here</h1>
    <!-- Content -->
  </main>
</body>
```

### `<article>` - Self-Contained Content

**Purpose:** Represents a **self-contained composition** that could be independently distributed or reused.

**Key Question:** *"Could this content make sense on its own, even if removed from the page?"*

If **YES** → Use `<article>`
If **NO** → Use `<section>` instead

**Perfect Use Cases:**
- Blog posts
- News articles
- Forum posts
- Product cards
- Comments
- Tweets/social media posts
- Individual items in a feed

**Example 1: Blog Post**

```html
<article>
  <header>
    <h1>10 Tips for Better CSS</h1>
    <p class="byline">
      By <span class="author">Jane Smith</span> on 
      <time datetime="2024-11-26">November 26, 2024</time>
    </p>
  </header>
  
  <section class="introduction">
    <p>CSS can be tricky, but these tips will help...</p>
  </section>
  
  <section class="tip">
    <h2>1. Use CSS Variables</h2>
    <p>CSS variables make your code more maintainable...</p>
  </section>
  
  <section class="tip">
    <h2>2. Master Flexbox</h2>
    <p>Flexbox simplifies layouts...</p>
  </section>
  
  <!-- More tips -->
  
  <footer>
    <p>Tags: <a href="/tag/css">CSS</a>, <a href="/tag/tips">Tips</a></p>
    <p>Share: 
      <a href="#">Twitter</a>
      <a href="#">Facebook</a>
    </p>
  </footer>
</article>
```

### `<section>` - Thematic Grouping

**Purpose:** Represents a **thematic grouping of content**, typically with a heading.

**Key Difference from `<article>`:**
- `<article>` = Self-contained, reusable content
- `<section>` = Thematic grouping within a larger context

**Rule of Thumb:** If you can't give it a meaningful heading, it shouldn't be a `<section>`. Use `<div>` instead.

**Example 1: Sectioning a Long Article**

```html
<article>
  <h1>Complete Guide to Web Performance</h1>
  
  <section id="introduction">
    <h2>Introduction</h2>
    <p>Web performance is critical for user experience...</p>
  </section>
  
  <section id="metrics">
    <h2>Key Performance Metrics</h2>
    <p>Several metrics help measure performance...</p>
    
    <section id="lcp">
      <h3>Largest Contentful Paint (LCP)</h3>
      <p>Measures loading performance...</p>
    </section>
    
    <section id="fid">
      <h3>First Input Delay (FID)</h3>
      <p>Measures interactivity...</p>
    </section>
  </section>
  
  <section id="optimization">
    <h2>Optimization Techniques</h2>
    <p>Here are proven techniques to improve performance...</p>
  </section>
  
  <section id="conclusion">
    <h2>Conclusion</h2>
    <p>Performance optimization is an ongoing process...</p>
  </section>
</article>
```

## 2.3 Text Semantics

Text semantic elements give meaning to inline text content.

### Heading Hierarchy (`<h1>` to `<h6>`)

**Purpose:** Define heading levels from most important (h1) to least important (h6).

**Critical Rules:**
1. ❗ **Only ONE `<h1>` per page** (for main heading/title)
2. ❗ **Don't skip levels** (h1 → h2 → h3, not h1 → h3)
3. ❗ **Use for structure, not size** (style with CSS if needed)

**Correct Heading Hierarchy:**

```html
<article>
  <h1>Complete Guide to Web Development</h1>  <!-- Page title -->
  
  <section>
    <h2>Frontend Development</h2>  <!-- Major section -->
    
    <section>
      <h3>HTML</h3>  <!-- Subsection -->
      
      <h4>Semantic Elements</h4>  <!-- Sub-subsection -->
      <p>Content about semantic elements...</p>
      
      <h4>Forms</h4>
      <p>Content about forms...</p>
    </section>
    
    <section>
      <h3>CSS</h3>
      
      <h4>Flexbox</h4>
      <p>Content about flexbox...</p>
      
      <h4>Grid</h4>
      <p>Content about grid...</p>
    </section>
  </section>
  
  <section>
    <h2>Backend Development</h2>
    
    <h3>Node.js</h3>
    <p>Content about Node.js...</p>
    
    <h3>Python</h3>
    <p>Content about Python...</p>
  </section>
</article>
```

**Visual Hierarchy:**

```
h1: Complete Guide to Web Development
  h2: Frontend Development
    h3: HTML
      h4: Semantic Elements
      h4: Forms
    h3: CSS
      h4: Flexbox
      h4: Grid
  h2: Backend Development
    h3: Node.js
    h3: Python
```

**❌ Common Mistakes:**

```html
<!-- ❌ WRONG: Multiple h1 tags -->
<h1>Main Title</h1>
<section>
  <h1>Section Title</h1>  <!-- Should be h2 -->
</section>

<!-- ❌ WRONG: Skipping levels -->
<h1>Title</h1>
<h3>Subtitle</h3>  <!-- Skipped h2! -->

<!-- ❌ WRONG: Using for size, not structure -->
<h3>Small heading</h3>  <!-- Just to make it smaller -->
<h1>Bigger text</h1>    <!-- Just to make it bigger -->
```

**✅ Correct Way to Change Size:**

```html
<!-- ✅ Use CSS to change size, not heading level -->
<h2 class="small-heading">Section Title</h2>

<style>
  .small-heading {
    font-size: 1.2rem;  /* Make h2 smaller if needed */
  }
</style>
```

### `<p>` - Paragraphs

**Purpose:** Represents a paragraph of text.

```html
<p>This is a paragraph. Paragraphs are block elements that contain flowing text content.</p>

<p>Each paragraph is a separate block. They automatically create space between each other.</p>
```

**Important:** Don't use multiple `<br>` tags to create space. Use proper paragraphs and CSS margin instead.

```html
<!-- ❌ WRONG -->
<p>
  First line<br><br><br>
  Second line with lots of breaks
</p>

<!-- ✅ CORRECT -->
<p>First paragraph</p>
<p>Second paragraph</p>

<style>
  p {
    margin-bottom: 1.5rem;
  }
</style>
```

### `<strong>` vs `<b>` - Important Text

**`<strong>` - Semantic Importance**

```html
<p><strong>Warning:</strong> This action cannot be undone.</p>
<p>It is <strong>critically important</strong> to back up your data.</p>
```

- **Meaning:** Strong importance, seriousness, or urgency
- **Screen readers:** May emphasize with different tone
- **Default appearance:** Bold
- **SEO:** Search engines understand this is important

**`<b>` - Visual Bold (No Semantic Meaning)**

```html
<p>The <b>bold text</b> is just visually different, not semantically important.</p>
<p>Article published in <b>2024</b>.</p>
```

- **Meaning:** None, just visual styling
- **Screen readers:** Read normally
- **Use case:** Keywords, product names in reviews

**When to use which:**

```html
<!-- ✅ Use <strong> for important content -->
<p><strong>Warning:</strong> Do not mix these chemicals!</p>
<p>The <strong>most important point</strong> is safety.</p>

<!-- ✅ Use <b> for stylistic bold without importance -->
<p>In the document titled <b>Introduction to HTML</b>, we learn...</p>
<p>The font size is <b>16px</b> by default.</p>
```

---

### `<em>` vs `<i>` - Emphasized Text

**`<em>` - Semantic Emphasis**

```html
<p>I <em>really</em> love HTML!</p>
<p>You must <em>not</em> forget to close your tags.</p>
```

- **Meaning:** Stress emphasis, changes meaning
- **Screen readers:** May emphasize with intonation
- **Default appearance:** Italic

**`<i>` - Visual Italic (No Semantic Meaning)**

```html
<p>The term <i>semantics</i> comes from Greek.</p>
<p>My favorite book is <i>The Great Gatsby</i>.</p>
```

- **Meaning:** None, just visual styling
- **Use cases:** Foreign words, technical terms, taxonomic names, titles

**Examples:**

```html
<!-- ✅ <em> for emphasis that changes meaning -->
<p>I love <em>you</em>.</p>  <!-- Emphasis on "you" specifically -->
<p>I <em>love</em> you.</p>  <!-- Emphasis on depth of feeling -->
<p><em>I</em> love you.</p>  <!-- Emphasis on "I" (not someone else) -->

<!-- ✅ <i> for technical terms, foreign words -->
<p>The scientific name is <i>Canis lupus</i>.</p>
<p>The French word <i>bonjour</i> means hello.</p>
<p>Read the book <i>1984</i> by George Orwell.</p>
```

### Other Text Semantic Elements

**`<mark>` - Highlighted/Marked Text**

```html
<p>Search results for "HTML":</p>
<p>Learn <mark>HTML</mark> and CSS for web development.</p>
<p><mark>HTML</mark> is the backbone of the web.</p>
```

- **Use case:** Search results, referenced text
- **Default appearance:** Yellow background

**`<small>` - Fine Print**

```html
<p>Buy now for only $99!</p>

<footer>
  <p>&copy; 2024 Company Name</p>
  <p><small>All rights reserved. Patent pending.</small></p>
</footer>
```

- **Use case:** Side comments, legal text, copyright
- **Default appearance:** Smaller font

**`<del>` and `<ins>` - Deleted and Inserted Text**

```html
<!-- Track changes in documents -->
<p>The price is <del>$100</del> <ins>$80</ins>.</p>

<p>
  My favorite color is <del>blue</del> <ins>red</ins>.
</p>

<!-- With datetime attributes -->
<p>
  Status: 
  <del datetime="2024-11-25">Pending</del>
  <ins datetime="2024-11-26">Approved</ins>
</p>
```

- **`<del>`:** Represents deleted/removed content
- **`<ins>`:** Represents inserted/added content
- **Attributes:** `cite` (URL explaining change), `datetime` (when changed)

---

**`<sub>` and `<sup>` - Subscript and Superscript**

```html
<!-- Chemical formulas -->
<p>Water is H<sub>2</sub>O.</p>
<p>Carbon dioxide is CO<sub>2</sub>.</p>

<!-- Mathematical expressions -->
<p>E = mc<sup>2</sup></p>
<p>2<sup>3</sup> = 8</p>

<!-- Footnotes -->
<p>This is an important fact<sup>1</sup>.</p>
<p><small><sup>1</sup> Source: Academic Journal, 2024</small></p>

<!-- Ordinals -->
<p>She finished 1<sup>st</sup> in the race.</p>
```

---

**`<abbr>` - Abbreviations**

```html
<!-- With title attribute showing full form -->
<p>
  <abbr title="HyperText Markup Language">HTML</abbr> is the standard 
  markup language for web pages.
</p>

<p>
  The <abbr title="World Health Organization">WHO</abbr> was founded in 1948.
</p>

<p>
  I work at <abbr title="International Business Machines">IBM</abbr>.
</p>
```

- **`title` attribute:** Shows tooltip on hover
- **Accessibility:** Screen readers may announce full form
- **Default appearance:** Dotted underline (in some browsers)

---

**`<cite>` - Citations**

```html
<p>
  As mentioned in <cite>The Art of Computer Programming</cite> by Donald Knuth...
</p>

<blockquote>
  <p>To be or not to be, that is the question.</p>
  <footer>— <cite>Hamlet</cite> by William Shakespeare</footer>
</blockquote>

<p>
  According to <cite>The New York Times</cite>, the economy is growing.
</p>
```

- **Use for:** Titles of creative works (books, movies, songs, etc.)
- **Don't use for:** Author names (use plain text or `<span>`)

---

**`<q>` - Short Inline Quotation**

```html
<p>Benjamin Franklin once said <q>Tell me and I forget, teach me and I may remember, involve me and I learn.</q></p>

<!-- With cite attribute -->
```

## 2.5 Time & Data Elements

### `<time>` - Machine-Readable Dates/Times

**Purpose:** Represents a specific time or date in a machine-readable format.

**Why It Matters:**
- Search engines can extract dates
- Browsers can add to calendar
- Screen readers can format appropriately
- Enables better date parsing

**Basic Usage:**

```html
<!-- Just displaying a date -->
<p>Published: <time datetime="2024-11-26">November 26, 2024</time></p>

<!-- Date and time -->
<p>Event starts: <time datetime="2024-12-25T10:00">December 25 at 10 AM</time></p>

<!-- Time only -->
<p>Meeting at <time datetime="14:30">2:30 PM</time></p>
```

**`datetime` Attribute Format:**

```html
<!-- Full date -->
<time datetime="2024-11-26">Nov 26, 2024</time>

<!-- Date with time -->
<time datetime="2024-11-26T15:30">Nov 26, 2024 at 3:30 PM</time>

<!-- Date with time and timezone -->
<time datetime="2024-11-26T15:30:00-05:00">Nov 26, 2024 3:30 PM EST</time>

<!-- Date with time and UTC -->
<time datetime="2024-11-26T20:30:00Z">Nov 26, 2024 8:30 PM UTC</time>

<!-- Year and month -->
<time datetime="2024-11">November 2024</time>

<!-- Year only -->
<time datetime="2024">2024</time>

<!-- Duration -->
<time datetime="PT2H30M">2 hours 30 minutes</time>
```

**Real-World Examples:**

```html
<!-- Blog post metadata -->
<article>
  <header>
    <h1>My Blog Post</h1>
    <p>
      Published: <time datetime="2024-11-26">November 26, 2024</time>
      Updated: <time datetime="2024-11-27">November 27, 2024</time>
    </p>
  </header>
</article>

<!-- Event listing -->
<article class="event">
  <h2>Web Development Conference</h2>
  <p>
    <time datetime="2024-12-15T09:00">December 15, 2024 at 9:00 AM</time> - 
    <time datetime="2024-12-15T17:00">5:00 PM</time>
  </p>
  <p>Duration: <time datetime="PT8H">8 hours</time></p>
</article>

<!-- Comment timestamp -->
<article class="comment">
  <p>Great article!</p>
  <footer>
    Posted by John on 
    <time datetime="2024-11-26T10:23:45-05:00" title="10:23:45 AM EST">
      2 hours ago
    </time>
  </footer>
</article>

<!-- Historical dates -->
<p>
  World War II ended on 
  <time datetime="1945-09-02">September 2, 1945</time>.
</p>
```

**Duration Format (ISO 8601):**

```html
<!-- P = Period, T = Time -->
<p>Movie length: <time datetime="PT2H30M">2 hours 30 minutes</time></p>
<p>Cook for <time datetime="PT45M">45 minutes</time></p>
<p>Project timeline: <time datetime="P3M">3 months</time></p>
<p>Trip duration: <time datetime="P2W">2 weeks</time></p>

<!-- P3Y2M10D = 3 years, 2 months, 10 days -->
<time datetime="P3Y2M10D">3 years, 2 months, and 10 days</time>
```

## Key Takeaways from Chapter 2

1. **Semantic HTML provides meaning**, not just appearance
2. **Use the most specific element** available for your content
3. **Only ONE `<main>` per page** - it's the primary content
4. **`<article>` for self-contained** content (blog posts, products, comments)
5. **`<section>` for thematic groupings** with headings
6. **`<aside>` for tangential** content (sidebars, pull quotes)
7. **Heading hierarchy is crucial** - one `<h1>`, don't skip levels
8. **`<strong>` vs `<b>`** - importance vs appearance
9. **`<em>` vs `<i>`** - emphasis vs appearance
10. **Lists have three types** - `<ul>`, `<ol>`, `<dl>` for different purposes
11. **`<time>` makes dates machine-readable** - better for SEO and accessibility
12. **`<address>` for contact info** - not for random addresses

Master these semantic elements, and your HTML will be accessible, SEO-friendly, and maintainable!
