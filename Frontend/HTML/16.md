### Category 1: HTML Fundamentals

**Q1: What is the purpose of DOCTYPE?**

**Answer:**
```html
<!DOCTYPE html>
```

The DOCTYPE declaration tells the browser which HTML version the page uses. In HTML5, it's simplified to just `<!DOCTYPE html>`. 

**Why it matters:**
- Triggers **standards mode** (not quirks mode)
- Ensures consistent rendering across browsers
- Required for HTML5 features to work properly
- Must be the very first line (before `<html>`)

**Without DOCTYPE:** Browser enters quirks mode, rendering inconsistently, breaking modern CSS layouts and causing cross-browser issues.

**Q3: What's the difference between `<div>` and `<span>`?**

- Takes full width (100%)
- Starts on new line
- Can have width/height set
- Can contain block or inline elements

- Only takes width of content
- Stays in line with text
- Cannot have width/height (ignored)
- Can only contain inline elements

### Category 2: Semantic HTML

**Q4: Why use semantic HTML?**

**Answer:**

Semantic HTML uses elements that clearly describe their meaning:

**1. Accessibility:**
```html
<!-- Screen reader announces "navigation landmark" -->
<nav>
  <a href="/">Home</a>
</nav>

<!-- Screen reader just reads content -->
<div class="navigation">
  <a href="/">Home</a>
</div>
```

**2. SEO:**
- Search engines understand structure better
- Content prioritization (main > aside)
- Rich snippets eligibility

**3. Maintainability:**
- Code is self-documenting
- Easier for teams to understand
- Faster debugging

**4. Future-proofing:**
- Standards-based
- Works with assistive technologies
- Browser updates won't break it

**Q5: Explain the difference between `<article>`, `<section>`, and `<div>`.**

**Answer:**

**`<article>` - Self-Contained Content:**
- Could stand alone and make sense
- Examples: blog post, news article, product card, comment

```html
<article>
  <h2>Blog Post Title</h2>
  <p>This could be distributed independently...</p>
</article>
```

**`<section>` - Thematic Grouping:**
- Groups related content with a heading
- Part of a larger whole
- Examples: chapters, tabbed content, numbered sections

```html
<article>
  <h1>Complete Guide</h1>
  
  <section>
    <h2>Chapter 1</h2>
    <p>Content...</p>
  </section>
  
  <section>
    <h2>Chapter 2</h2>
    <p>Content...</p>
  </section>
</article>
```

**`<div>` - Generic Container:**
- No semantic meaning
- Use for styling/layout only
- When no semantic element fits

```html
<div class="wrapper">  <!-- Layout container -->
  <div class="container">  <!-- Another layout div -->
    <section>  <!-- Now semantic content starts -->
      <h2>Actual Content</h2>
    </section>
  </div>
</div>
```

**Decision Tree:**
```
Can it stand alone? 
  ‚Üí Yes: <article>

Part of a themed group with heading?
  ‚Üí Yes: <section>

Just for styling/layout?
  ‚Üí Yes: <div>
```

**Q7: How do you validate a form in HTML5?**

**Answer:**

**HTML5 Validation Attributes:**

```html

  <!-- Required field -->
  <input type="text" required>
  
  <!-- Length validation -->
  <input type="text" minlength="3" maxlength="20">

```

**Custom Validation with JavaScript:**

```javascript
const input = document.getElementById('email');

input.addEventListener('invalid', (e) => {
  e.preventDefault();
  
  if (input.validity.valueMissing) {
    input.setCustomValidity('Email is required');
  } else if (input.validity.typeMismatch) {
    input.setCustomValidity('Please enter a valid email');
  }
});

input.addEventListener('input', () => {
  input.setCustomValidity(''); // Clear custom error
});
```

### Category 4: Attributes & Meta Tags

**Q8: Explain the difference between `id` and `class`.**

**Answer:**

**`id` Attribute:**
- **Unique** - Only one element per page can have a specific ID
- **High specificity** - Harder to override in CSS
- **JavaScript** - Fast selection with `getElementById()`
- **Anchor links** - Can be target of `#id`

**`class` Attribute:**
- **Reusable** - Multiple elements can have same class
- **Lower specificity** - Easier to override
- **Multiple classes** - One element can have many classes
- **Best for styling** - Recommended approach

```html
<!-- ID: Unique, one per page -->
<div id="header">Only one header</div>
<div id="header">ERROR - Duplicate ID!</div>

<!-- Class: Reusable, many per page -->
<div class="card">Card 1</div>
<div class="card">Card 2</div>
<div class="card">Card 3</div>

<!-- Multiple classes -->
<div class="card featured premium">Featured Premium Card</div>
```

**When to use which:**

```html
<!-- ‚úÖ Use ID for: -->
<div id="main-content">  <!-- Unique landmarks -->
<form id="contactForm">  <!-- JavaScript reference -->
<section id="pricing">   <!-- Anchor link target -->

<!-- ‚úÖ Use Class for: -->
<div class="card">       <!-- Reusable components -->
<button class="btn">     <!-- Styling -->
<p class="text-large">   <!-- Multiple elements -->
```

**Q9: What is the purpose of the `data-*` attribute?**

**Answer:**

Custom data attributes store extra information on HTML elements.

**Syntax:**
```html

  data-user-name="John Doe"
```

**Q11: How do you optimize images for web performance?**

**Answer:**

**Multiple optimization techniques:**

**1. Choose the right format:**
```html
<!-- Photos: JPEG or WebP -->
<img src="photo.jpg" alt="Photo">

<!-- Logos/Icons: PNG or SVG -->
<img src="logo.svg" alt="Logo">

<!-- Modern: WebP with fallback -->
<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Image">
</picture>
```

**2. Compress images:**
- Use tools: ImageOptim, TinyPNG, Squoosh
- Target: 80-90% quality (visually lossless)
- WebP: 25-35% smaller than JPEG

**3. Specify dimensions:**
```html
<img src="photo.jpg" alt="Photo" width="800" height="600">
<!-- Prevents layout shift (CLS) -->
```

**4. Lazy loading:**
```html
<img src="image.jpg" alt="Image" loading="lazy" width="800" height="600">
<!-- Only loads when near viewport -->
```

**5. Responsive images:**
```html
<img 
  srcset="small-400.jpg 400w, large-1200.jpg 1200w"
  sizes="(max-width: 600px) 400px, 1200px"
  src="large-1200.jpg"
  alt="Image"
>
<!-- Serves appropriate size for device -->
```

**6. Modern formats:**
```html
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Image">
</picture>
```

### Category 6: Performance

**Q12: What are `defer` and `async` attributes? When would you use each?**

**Answer:**

Both prevent script blocking, but work differently:

**`defer`:**
```html
<script src="app.js" defer></script>
```
- Downloads in parallel with HTML parsing
- Executes **after** HTML parsing completes
- **Maintains order** (if multiple scripts)
- Executes before `DOMContentLoaded`

**`async`:**
```html
<script src="analytics.js" async></script>
```
- Downloads in parallel with HTML parsing
- Executes **as soon as** downloaded
- **Doesn't maintain order**
- May execute before DOM ready

**Visual Timeline:**

```
defer:
‚îú‚îÄ HTML parsing ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚î§
‚îú‚îÄ Download script ‚îÄ‚îÄ‚îÄ‚îÄ‚î§
                        ‚îî‚îÄ Execute script ‚îÄ‚î§

async:
‚îú‚îÄ HTML parsing ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚î§ PAUSE ‚îú‚îÄ Resume ‚îÄ‚îÄ‚îÄ‚îÄ‚î§
‚îú‚îÄ Download script ‚îÄ‚î§
                    ‚îî‚îÄ Execute ‚îÄ‚î§ (interrupts parsing)
```

**When to use:**

```html
<!-- ‚úÖ defer: Application scripts (need full DOM) -->
<script src="main.js" defer></script>
<script src="components.js" defer></script>
<!-- Executes in order -->

<!-- ‚úÖ async: Independent scripts (analytics, ads) -->
<script src="google-analytics.js" async></script>
<script src="ads.js" async></script>
<!-- Order doesn't matter -->

<!-- ‚ùå Neither: Blocking (avoid!) -->
<script src="script.js"></script>
```

**Q13: What are Web Vitals?**

**Answer:**

**Core Web Vitals** are Google's metrics for user experience:

**1. LCP (Largest Contentful Paint):**
- Measures **loading** performance
- When largest element becomes visible
- **Good:** <2.5 seconds
- **Needs improvement:** 2.5-4 seconds
- **Poor:** >4 seconds

**Optimize:**
```html
<!-- Preload hero image -->
<link rel="preload" href="hero.jpg" as="image">

<!-- Use modern formats -->
<picture>
  <source srcset="hero.webp" type="image/webp">
  <img src="hero.jpg" alt="Hero" width="1200" height="600">
</picture>
```

**2. FID (First Input Delay):**
- Measures **interactivity**
- Time from user interaction to browser response
- **Good:** <100 milliseconds
- **Needs improvement:** 100-300ms
- **Poor:** >300ms

**Optimize:**
```html
<!-- Defer JavaScript -->
<script src="app.js" defer></script>

<!-- Break up long tasks -->
<script>
  async function processData(data) {
    for (let i = 0; i < data.length; i++) {
      processItem(data[i]);
      if (i % 100 === 0) {
        await new Promise(resolve => setTimeout(resolve, 0));
      }
    }
  }
</script>
```

**3. CLS (Cumulative Layout Shift):**
- Measures **visual stability**
- How much content shifts while loading
- **Good:** <0.1
- **Needs improvement:** 0.1-0.25
- **Poor:** >0.25

**Optimize:**
```html
<!-- Always specify dimensions -->
<img src="photo.jpg" alt="Photo" width="800" height="600">

<!-- Reserve space for dynamic content -->
<div style="min-height: 400px;">
  <!-- Content loads here -->
</div>
```

### Category 7: Security

**Q14: What is XSS and how do you prevent it?**

**Answer:**

**XSS (Cross-Site Scripting):** Injecting malicious scripts into websites.

**Types:**

**1. Stored XSS:**
```html
<!-- Attacker submits: -->
<script>steal_cookies()</script>

<!-- Stored in database, shown to all users -->
```

**2. Reflected XSS:**
```html
<!-- URL: example.com/search?q=<script>alert('XSS')</script> -->
<!-- Page displays: -->
You searched for: <script>alert('XSS')</script>
```

**Prevention:**

**1. Never use innerHTML with user input:**
```javascript
// ‚ùå DANGEROUS
element.innerHTML = userInput;

// ‚úÖ SAFE
element.textContent = userInput;
```

**2. Escape HTML:**
```javascript
function escapeHtml(unsafe) {
  return unsafe
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#039;");
}
```

**3. Content Security Policy:**
```html
<meta http-equiv="Content-Security-Policy" content="
  default-src 'self';
  script-src 'self';
">
<!-- Blocks inline scripts -->
```

**4. Validate and sanitize input:**
```javascript
// Server-side (example)
const sanitizedInput = sanitizeHtml(userInput);
```

**Q15: Why use `rel="noopener noreferrer"` with `target="_blank"`?**

**Answer:**

**Security vulnerability:** Without it, the new page can access your page via `window.opener`.

**The Attack:**
```html
<!-- Your site: -->
<a href="https://malicious-site.com" target="_blank">
  Click here
</a>

<!-- Malicious site can: -->
<script>
  window.opener.location = 'https://fake-login-page.com';
  // Your tab now shows fake login!
</script>
```

**The Fix:**
```html
<a 
  href="https://external-site.com" 
  target="_blank" 
  rel="noopener noreferrer"
>
  External Link
</a>
```

**What each does:**
- `noopener` - Prevents `window.opener` access
- `noreferrer` - Doesn't send referrer information

**Modern browsers** (Chrome 88+) automatically apply `noopener`, but include it for older browsers.

### Q17: `localStorage` vs `sessionStorage` vs Cookies

### Q19: GET vs POST

**Answer:**

**GET:**
- Data in URL query string
- Visible in browser history/logs
- Bookmarkable
- Cacheable
- Length limit (~2000 characters)
- **Use for:** Searches, filters, non-sensitive data

```html
<form action="/search" method="GET">
  <input type="text" name="q">
  <button>Search</button>
</form>
<!-- Submits to: /search?q=keywords -->
```

**POST:**
- Data in request body
- Not visible in URL
- Not bookmarkable
- Not cached
- No length limit
- **Use for:** Forms, sensitive data, file uploads, creating/updating data

```html
<form action="/login" method="POST">
  <input type="email" name="email">
  <input type="password" name="password">
  <button>Login</button>
</form>
<!-- Data sent in body, URL stays /login -->
```

### Scenario 4: Preventing Layout Shift

**Answer:**

**CLS Prevention Techniques:**

**1. Always specify image dimensions:**


```html
<style>

    padding-bottom: 56.25%; /* 16:9 */

</div>
```

**4. Avoid inserting content above existing content:**
```javascript
// ‚ùå Bad: Inserts at top (shifts everything down)
element.insertAdjacentHTML('afterbegin', newContent);

// ‚úÖ Good: Appends at bottom (no shift)
element.insertAdjacentHTML('beforeend', newContent);
```

### Q20: Explain the Critical Rendering Path

**Answer:**

**The sequence of steps browsers use to render a page:**

**Steps:**
1. **Parse HTML** ‚Üí Build DOM tree
2. **Parse CSS** ‚Üí Build CSSOM tree
3. **Combine DOM + CSSOM** ‚Üí Render tree

**Optimization Strategy:**
- Inline critical CSS (above-the-fold)
- Defer non-critical CSS
- Defer/async JavaScript
- Preload critical resources
- Minimize blocking resources

```html
  <link rel="preload" href="styles.css" as="style" onload="this.rel='stylesheet'">

    <source srcset="hero.webp" type="image/webp">
```

### Demonstrating Deep Knowledge

**Show you understand trade-offs:**

**Interviewer:** "Should you use localStorage or sessionStorage?"

**Good Answer:**
"It depends on the use case. 

**localStorage:**
- Persists across browser restarts
- Shared across tabs
- Good for user preferences, shopping carts, cached data

**sessionStorage:**
- Cleared when tab closes
- Tab-specific
- Good for multi-step forms, temporary filters, session-specific data

**However,** neither should store sensitive data (vulnerable to XSS). For authentication tokens, I'd use httpOnly cookies. For large datasets, I'd consider IndexedDB.

I'd also implement proper error handling since storage can fail (quota exceeded, disabled in incognito mode)."
