# Chapter 8: HTML Attributes - In-Depth Explanation

## Introduction

HTML attributes provide additional information about elements. Some attributes work on specific elements (like `href` on `<a>`), while others work on any element (global attributes).

**This Chapter Covers:**
- Global attributes (work on all elements)
- Data attributes (custom data storage)
- ARIA attributes (accessibility)
- Microdata attributes (structured data for SEO)

---

## 8.1 Global Attributes

Global attributes can be used on **any HTML element**.

### `id` - Unique Identifier

**Purpose:** Provides a unique identifier for an element.

**Rules:**
- Must be **unique** on the page (only one element can have a specific ID)
- Case-sensitive
- Must start with a letter (a-z or A-Z)
- Can contain letters, digits, hyphens, underscores, periods
- No spaces allowed

```html
<!-- ‚úÖ Valid IDs -->
<div id="header">Header</div>
<div id="main-content">Content</div>
<div id="section_1">Section 1</div>
<div id="user-profile-123">Profile</div>

<!-- ‚ùå Invalid IDs -->
<div id="123-start"><!-- Can't start with number --></div>
<div id="my section"><!-- No spaces allowed --></div>
<div id="user@email"><!-- Special chars not allowed --></div>

<!-- ‚ùå Duplicate IDs (BAD!) -->
<div id="main">First</div>
<div id="main">Second</div>  <!-- ERROR: Duplicate ID! -->
```

**Use Cases:**

1. **CSS Targeting (High Specificity)**

```html
<div id="hero">Hero Section</div>

<style>
  #hero {
    background: blue;
    /* ID selector has high specificity */
  }
</style>
```

2. **JavaScript Selection**

```html
<button id="submitBtn">Submit</button>

<script>
  const btn = document.getElementById('submitBtn');
  btn.addEventListener('click', () => {
    alert('Clicked!');
  });
</script>
```

3. **Anchor Links (Fragment Identifiers)**

```html
<a href="#section-2">Jump to Section 2</a>

<div id="section-2">
  <h2>Section 2</h2>
  <p>Content here...</p>
</div>
```

4. **Label Association**

```html
<label for="username">Username:</label>
<input type="text" id="username" name="username">
```

**Best Practices:**

```html
<!-- ‚úÖ Good: Descriptive, semantic -->
<div id="user-profile-header"></div>
<div id="shopping-cart"></div>
<div id="comment-form"></div>

<!-- ‚ùå Bad: Generic, meaningless -->
<div id="div1"></div>
<div id="content"></div>
<div id="section"></div>

<!-- ‚úÖ Good: Kebab-case (recommended) -->
<div id="main-navigation"></div>

<!-- ‚úÖ Also acceptable: camelCase -->
<div id="mainNavigation"></div>

<!-- ‚úÖ Also acceptable: snake_case -->
<div id="main_navigation"></div>
```

**ID vs Class:**

```html
<!-- ID: Unique, one per page -->
<div id="header">Only one header</div>

<!-- Class: Reusable, many per page -->
<div class="card">Card 1</div>
<div class="card">Card 2</div>
<div class="card">Card 3</div>
```

### `class` - CSS Classes

**Purpose:** Assigns one or more class names for styling and JavaScript.

**Rules:**
- Can be **reused** on multiple elements
- Multiple classes separated by spaces
- Same naming rules as ID

```html
<!-- Single class -->
<div class="container">Content</div>

<!-- Multiple classes -->
<div class="card featured highlighted">Featured Card</div>

<!-- Same class on multiple elements -->
<button class="btn">Button 1</button>
<button class="btn">Button 2</button>
<button class="btn">Button 3</button>
```

**CSS Targeting:**

```html
<div class="card featured">Featured Card</div>
<div class="card">Normal Card</div>
<div class="card featured premium">Premium Featured Card</div>

<style>
  /* All cards */
  .card {
    border: 1px solid #ddd;
    padding: 20px;
  }
  
  /* Featured cards only */
  .featured {
    background: yellow;
  }
  
  /* Premium cards only */
  .premium {
    border-color: gold;
  }
  
  /* Elements with both featured AND premium */
  .featured.premium {
    background: linear-gradient(to right, yellow, gold);
  }
</style>
```

**JavaScript Selection:**

```javascript
// Get all elements with class
const cards = document.getElementsByClassName('card');
const cards2 = document.querySelectorAll('.card');

// Check if element has class
element.classList.contains('active');

// Add class
element.classList.add('active');

// Remove class
element.classList.remove('active');

// Toggle class
element.classList.toggle('active');

// Replace class
element.classList.replace('old-class', 'new-class');
```

**Common Naming Conventions:**

```html
<!-- BEM (Block Element Modifier) -->
<div class="card">
  <div class="card__header">
    <h3 class="card__title">Title</h3>
  </div>
  <div class="card__body">Content</div>
  <div class="card__footer">
    <button class="card__button card__button--primary">Action</button>
  </div>
</div>

<!-- Utility Classes -->
<div class="mt-4 p-2 bg-blue text-white rounded">
  Utility-styled content
</div>

<!-- Semantic Classes -->
<div class="user-profile">
  <div class="profile-header">Header</div>
  <div class="profile-content">Content</div>
</div>
```

### `style` - Inline CSS

**Purpose:** Apply CSS directly to an element.

```html
<div style="color: red; font-size: 20px;">Red text</div>

<p style="
  background: blue;
  color: white;
  padding: 10px;
  border-radius: 5px;
">
  Styled paragraph
</p>
```

**‚ö†Ô∏è Warning: Avoid Inline Styles in Production!**

```html
<!-- ‚ùå Bad: Hard to maintain, no reusability -->
<div style="color: red;">Text 1</div>
<div style="color: red;">Text 2</div>
<div style="color: red;">Text 3</div>

<!-- ‚úÖ Good: Use classes instead -->
<div class="text-red">Text 1</div>
<div class="text-red">Text 2</div>
<div class="text-red">Text 3</div>

<style>
  .text-red {
    color: red;
  }
</style>
```

**When Inline Styles Are Acceptable:**

```html
<!-- Dynamic styles from JavaScript -->
<div id="box"></div>

<script>
  const box = document.getElementById('box');
  const userColor = getUserPreference(); // From database/API
  box.style.backgroundColor = userColor;
</script>

<!-- CSS variables (better approach) -->
<div style="--user-color: blue;" class="themed-box">
  Content
</div>

<style>
  .themed-box {
    background: var(--user-color);
  }
</style>
```

### `hidden` - Hide Element

**Purpose:** Hides the element from display and accessibility tree.

```html
<!-- Hidden element -->
<div hidden>
  This content is completely hidden
</div>

<!-- Conditionally hidden -->
<div id="message" hidden>
  Success message
</div>

<script>
  function showMessage() {
    document.getElementById('message').hidden = false;
  }
  
  function hideMessage() {
    document.getElementById('message').hidden = true;
  }
</script>
```

**Equivalent to CSS:**

```css
[hidden] {
  display: none;
}
```

**Hidden vs CSS display:none:**

```html
<!-- Hidden attribute (semantic) -->
<div hidden>Content</div>

<!-- CSS display none (presentational) -->
<div style="display: none;">Content</div>

<!-- Visually hidden but accessible to screen readers -->
<div class="sr-only">Content</div>

<style>
  .sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    border: 0;
  }
</style>
```

**Use Cases:**

```html
<!-- Toggle visibility -->
<button onclick="toggleDetails()">Show Details</button>
<div id="details" hidden>
  <p>Additional details...</p>
</div>

<script>
  function toggleDetails() {
    const details = document.getElementById('details');
    details.hidden = !details.hidden;
  }
</script>

<!-- Conditional rendering -->
<div id="error-message" hidden>
  An error occurred!
</div>

<script>
  if (hasError) {
    document.getElementById('error-message').hidden = false;
  }
</script>
```

### `tabindex` - Keyboard Navigation

**Purpose:** Controls tab order and focusability.

**Values:**
- `tabindex="0"` - Normal tab order, focusable
- `tabindex="-1"` - Not in tab order, but programmatically focusable
- `tabindex="1+"` - Custom tab order (avoid!)

```html
<!-- Make non-interactive element focusable -->
<div tabindex="0">
  This div is now keyboard focusable
</div>

<!-- Programmatically focusable only -->
<div tabindex="-1" id="alert">
  Alert message
</div>

<script>
  // Focus programmatically
  document.getElementById('alert').focus();
</script>
```

**Tab Order Examples:**

```html
<!-- ‚ùå Bad: Custom tab order (avoid!) -->
<input type="text" tabindex="3">
<input type="text" tabindex="1">
<input type="text" tabindex="2">
<!-- Confusing for users! -->

<!-- ‚úÖ Good: Natural tab order -->
<input type="text">  <!-- Tab order: 1 -->
<input type="text">  <!-- Tab order: 2 -->
<input type="text">  <!-- Tab order: 3 -->

<!-- ‚úÖ Good: Remove from tab order -->
<button tabindex="-1">Not tabbable</button>

<!-- ‚úÖ Good: Make custom element focusable -->
<div role="button" tabindex="0" onclick="handleClick()">
  Custom Button
</div>
```

**Common Use Cases:**

```html
<!-- Skip link (for accessibility) -->
<a href="#main-content" class="skip-link" tabindex="1">
  Skip to main content
</a>

<!-- Modal dialog focus management -->
<div role="dialog" tabindex="-1" id="modal">
  <h2 tabindex="0">Dialog Title</h2>
  <button>Close</button>
</div>

<script>
  function openModal() {
    const modal = document.getElementById('modal');
    modal.focus(); // Focus on modal container
  }
</script>

<!-- Accordion panel -->
<button tabindex="0" onclick="togglePanel()">
  Expand Section
</button>
<div tabindex="-1" hidden>
  Panel content
</div>
```

**‚ö†Ô∏è Tabindex Best Practices:**

```html
<!-- ‚úÖ Use tabindex="0" for custom interactive elements -->
<div role="button" tabindex="0">Custom Button</div>

<!-- ‚úÖ Use tabindex="-1" for programmatic focus -->
<div tabindex="-1" id="skip-target">Content</div>

<!-- ‚ùå Avoid positive tabindex values -->
<input tabindex="5">  <!-- Don't do this! -->

<!-- ‚úÖ Let native elements handle their own focus -->
<button>Native Button</button>  <!-- No tabindex needed -->
```

### `draggable` - Drag and Drop

**Purpose:** Indicates whether element can be dragged.

```html
<!-- Draggable element -->
<div draggable="true">
  Drag me!
</div>

<!-- Not draggable (default for most elements) -->
<div draggable="false">
  Cannot drag
</div>
```

**Complete Drag and Drop Example:**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Drag and Drop</title>
  <style>
    .draggable {
      padding: 20px;
      margin: 10px;
      background: #0066cc;
      color: white;
      border-radius: 4px;
      cursor: move;
      display: inline-block;
    }
    
    .draggable:active {
      opacity: 0.5;
    }
    
    .dropzone {
      min-height: 200px;
      padding: 20px;
      margin: 20px 0;
      border: 2px dashed #ddd;
      border-radius: 8px;
      background: #f9f9f9;
    }
    
    .dropzone.drag-over {
      background: #e3f2fd;
      border-color: #0066cc;
    }
  </style>
</head>
<body>
  <h1>Drag and Drop Demo</h1>
  
  <div draggable="true" class="draggable" id="item1">
    Drag Item 1
  </div>
  
  <div draggable="true" class="draggable" id="item2">
    Drag Item 2
  </div>
  
  <div draggable="true" class="draggable" id="item3">
    Drag Item 3
  </div>
  
  <div class="dropzone" id="dropzone">
    Drop items here
  </div>
  
  <script>
    const draggables = document.querySelectorAll('.draggable');
    const dropzone = document.getElementById('dropzone');
    
    draggables.forEach(draggable => {
      // When drag starts
      draggable.addEventListener('dragstart', (e) => {
        e.dataTransfer.effectAllowed = 'move';
        e.dataTransfer.setData('text/html', e.target.outerHTML);
        e.dataTransfer.setData('text/plain', e.target.id);
      });
      
      // While dragging
      draggable.addEventListener('drag', (e) => {
        console.log('Dragging...');
      });
      
      // When drag ends
      draggable.addEventListener('dragend', (e) => {
        console.log('Drag ended');
      });
    });
    
    // Drop zone events
    dropzone.addEventListener('dragover', (e) => {
      e.preventDefault(); // Allow drop
      dropzone.classList.add('drag-over');
    });
    
    dropzone.addEventListener('dragleave', (e) => {
      dropzone.classList.remove('drag-over');
    });
    
    dropzone.addEventListener('drop', (e) => {
      e.preventDefault();
      dropzone.classList.remove('drag-over');
      
      const id = e.dataTransfer.getData('text/plain');
      const element = document.getElementById(id);
      
      if (element) {
        dropzone.appendChild(element);
      }
    });
  </script>
</body>
</html>
```

## 8.2 Data Attributes (`data-*`)

**Purpose:** Store custom data on HTML elements.

**Rules:**
- Must start with `data-`
- Followed by lowercase letters, numbers, hyphens, dots, underscores
- Accessed via JavaScript `dataset` property

```html
<!-- Basic data attributes -->
<div 
  data-user-id="12345"
  data-user-name="John Doe"
  data-user-role="admin"
>
  User Info
</div>

<!-- Accessing with JavaScript -->
<script>
  const div = document.querySelector('div');
  
  console.log(div.dataset.userId);     // "12345"
  console.log(div.dataset.userName);   // "John Doe"
  console.log(div.dataset.userRole);   // "admin"
  
  // Setting data attributes
  div.dataset.userStatus = 'active';
  
  // Removing data attributes
  delete div.dataset.userRole;
</script>
```

**Naming Convention:**

```html
<!-- HTML: kebab-case -->
<div data-user-name="John"></div>
<div data-product-id="123"></div>
```

## 8.3 ARIA Attributes (Accessibility)

**Purpose:** Enhance accessibility for assistive technologies (screen readers).

**Key Principle:** Use **native HTML first**, ARIA second!

```html
<!-- ‚ùå Bad: Using ARIA unnecessarily -->
<div role="button" tabindex="0" onclick="submit()">Submit</div>

<!-- ‚úÖ Good: Native button element -->
<button onclick="submit()">Submit</button>
```

---

### `role` Attribute

Defines the element's purpose for assistive technologies.

**Common Roles:**

```html
<!-- Navigation -->
<div role="navigation">
  <a href="/">Home</a>
</div>
<!-- Better: -->
<nav>
  <a href="/">Home</a>
</nav>

<!-- Button -->
<div role="button" tabindex="0">Click me</div>
<!-- Better: -->
<button>Click me</button>

<!-- Alert -->
<div role="alert">
  Error: Please fill all fields
</div>

<!-- Dialog -->
<div role="dialog" aria-labelledby="dialog-title">
  <h2 id="dialog-title">Confirm Action</h2>
</div>

<!-- Menu -->
<ul role="menu">
  <li role="menuitem">Option 1</li>
  <li role="menuitem">Option 2</li>
</ul>

<!-- Tab panel -->
<div role="tabpanel">
  Content for this tab
</div>
```

---

```html
<!-- Search button with icon -->

  üîç
</button>

<!-- Social media links -->

  <svg><!-- Twitter icon --></svg>
</a>

<!-- Navigation buttons -->

<button aria-label="Next page">‚Üí</button>
```

```html
</div>

  <caption id="table-caption">Sales Data 2024</caption>

</table>
```

### `aria-describedby` - Additional Description

Provides additional description by referencing another element.

```html

  type="password" 
  id="password"
  aria-describedby="password-help"

  type="email" 
  id="email"
```

### `aria-hidden` - Hide from Screen Readers

Hides element from assistive technologies.

```html
<!-- Decorative icons -->

  <span aria-hidden="true">üîç</span>
  Search
</button>

<!-- Duplicate content for visual effect -->

  Welcome
  <span aria-hidden="true" class="shadow-text">Welcome</span>
</h1>

```

**‚ö†Ô∏è Important:**

```html
<!-- ‚ùå Bad: Hiding interactive content -->

<!-- Screen reader users can't access this! -->

<!-- ‚úÖ Good: Only hide decorative elements -->

  <span aria-hidden="true">‚Üí</span>
  Next
</button>
```

### `aria-expanded` - Expansion State

Indicates whether content is expanded or collapsed.

```html
<button 
  aria-expanded="false" 
  aria-controls="menu"
  onclick="toggleMenu()"
>
  Menu
</button>

<ul id="menu" hidden>
  <li>Option 1</li>
  <li>Option 2</li>
</ul>

<script>
  function toggleMenu() {
    const button = document.querySelector('button');
    const menu = document.getElementById('menu');
    const expanded = button.getAttribute('aria-expanded') === 'true';
    
    button.setAttribute('aria-expanded', !expanded);
    menu.hidden = expanded;
  }
</script>
```

### `aria-selected`, `aria-checked` - Selection State

```html

  <button role="tab" aria-selected="true">Tab 1</button>

  tabindex="0"

</div>
```

### `aria-live` - Live Regions

Announces dynamic content changes to screen readers.

```html
<!-- Polite: Waits for user to finish -->
<div aria-live="polite" role="status">
  <!-- Status updates -->
</div>

<!-- Assertive: Interrupts immediately -->
<div aria-live="assertive" role="alert">
  <!-- Error messages -->
</div>

<!-- Off: No announcements (default) -->
<div aria-live="off">
  <!-- Regular content -->
</div>
```

**Example - Form Validation:**

```html
<form>
  <label for="username">Username:</label>
  <input type="text" id="username" required>
  
  <div aria-live="polite" role="status" id="username-status"></div>
  
  <button type="submit">Submit</button>
</form>

<script>
  const input = document.getElementById('username');
  const status = document.getElementById('username-status');
  
  input.addEventListener('blur', () => {
    if (input.value.length < 3) {
      status.textContent = 'Username must be at least 3 characters';
      input.setAttribute('aria-invalid', 'true');
    } else {
      status.textContent = 'Username is valid';
      input.setAttribute('aria-invalid', 'false');
    }
  });
</script>
```

## Key Takeaways from Chapter 8

### Global Attributes:
1. **`id`** - Unique identifier (only one per page)
2. **`class`** - Reusable styling hooks (many per page)
3. **`style`** - Avoid inline styles (use classes instead)
4. **`title`** - Tooltips (not for critical info)
5. **`lang`** - Language code (helps screen readers)
6. **`tabindex`** - Control keyboard navigation
7. **`contenteditable`** - Make elements editable
8. **`draggable`** - Enable drag and drop

### Data Attributes:
9. **`data-*`** - Store custom data on elements
10. **Access via `dataset`** - JavaScript API
11. **Use for behavior, not styling** - Classes for styles

### ARIA:
12. **Use native HTML first** - ARIA is a supplement
13. **`aria-label`** - Label when no visible text
14. **`aria-hidden`** - Hide from screen readers
15. **`aria-live`** - Announce dynamic changes
16. **Always test with screen readers** - NVDA, JAWS, VoiceOver

### Microdata:
17. **Schema.org** - Structured data vocabulary
18. **Better SEO** - Rich snippets in search results
19. **Product, Article, Organization** - Common types
20. **Use for business-critical pages** - Homepage, products, articles

Master these attributes and you'll build semantic, accessible, SEO-friendly, and interactive web pages! üöÄ
