# Chapter 4: Tables - In-Depth Explanation

## Introduction: What Are Tables For?

HTML tables are designed to display **tabular data** - information that naturally fits in rows and columns, like spreadsheets.

**âœ… Tables Should Be Used For:**
- Spreadsheet-like data
- Financial reports
- Product comparisons
- Schedules and timetables
- Statistical data
- Pricing tables

**âŒ Tables Should NOT Be Used For:**
- Page layout (use CSS Grid or Flexbox instead)
- Positioning images
- Creating columns for text
- Navigation menus

âš ï¸ **Historical Note:** In the early days of the web (1990s-2000s), tables were commonly used for page layout. This is now considered **bad practice** because it:
- Hurts accessibility (screen readers get confused)
- Makes responsive design difficult
- Creates maintenance nightmares
- Hurts SEO

## 4.1 Table Structure

### Basic Table Elements

Every HTML table is built from these elements:

```html
<table>       <!-- Container for entire table -->
  <thead>     <!-- Table header group (optional but recommended) -->
    <tr>      <!-- Table row -->
      <th>    <!-- Table header cell (bold, centered by default) -->
  <tbody>     <!-- Table body group (optional but recommended) -->
    <tr>      <!-- Table row -->
      <td>    <!-- Table data cell (normal text, left-aligned) -->
  <tfoot>     <!-- Table footer group (optional) -->
```

---

### The `<table>` Element

The `<table>` element is the container for the entire table.

**Basic Syntax:**

```html
<table>
  <!-- All table content goes here -->
</table>
```

**Common Attributes (mostly deprecated, use CSS instead):**

```html
<!-- âŒ Deprecated (don't use) -->
<table border="1" cellpadding="10" cellspacing="5" width="100%">

<!-- âœ… Modern approach (use CSS) -->
<table class="data-table">
```

```css
.data-table {
  width: 100%;
  border-collapse: collapse;
}

.data-table th,
.data-table td {
  padding: 10px;
  border: 1px solid #ddd;
}
```

---

### The `<thead>` Element - Table Header Group

Groups header rows together. Should contain one or more `<tr>` elements with `<th>` cells.

**Why Use `<thead>`:**
1. **Accessibility** - Screen readers announce "table with X columns"
2. **Styling** - Easy to style headers differently
3. **Printing** - Headers can repeat on each printed page
4. **Scrolling** - Can make headers sticky while body scrolls

**Example:**

```html
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>John Doe</td>
      <td>30</td>
      <td>New York</td>
    </tr>
  </tbody>
</table>
```

---

### The `<tbody>` Element - Table Body Group

Groups the main data rows. Most tables have one `<tbody>`, but you can have multiple for grouping.

**Single tbody:**

```html
<table>
  <thead>
    <tr>
      <th>Product</th>
      <th>Price</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Laptop</td>
      <td>$999</td>
    </tr>
    <tr>
      <td>Mouse</td>
      <td>$29</td>
    </tr>
  </tbody>
</table>
```

**Multiple tbody (for grouping):**

```html
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Score</th>
    </tr>
  </thead>
  
  <tbody>
    <tr>
      <th colspan="2">Team A</th>
    </tr>
    <tr>
      <td>Alice</td>
      <td>95</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>87</td>
    </tr>
  </tbody>
  
  <tbody>
    <tr>
      <th colspan="2">Team B</th>
    </tr>
    <tr>
      <td>Charlie</td>
      <td>92</td>
    </tr>
    <tr>
      <td>Diana</td>
      <td>88</td>
    </tr>
  </tbody>
</table>
```

### The `<tfoot>` Element - Table Footer Group

Groups footer rows (totals, summaries, etc.). 

**Important:** `<tfoot>` can be placed **before or after** `<tbody>` in HTML5, but it always renders at the bottom.

**Example:**

```html
<table>
  <thead>
    <tr>
      <th>Item</th>
      <th>Quantity</th>
      <th>Price</th>
    </tr>
  </thead>
  
  <tbody>
    <tr>
      <td>Laptop</td>
      <td>2</td>
      <td>$1,998</td>
    </tr>
    <tr>
      <td>Mouse</td>
      <td>5</td>
      <td>$145</td>
    </tr>
  </tbody>
  
  <tfoot>
    <tr>
      <th colspan="2">Total</th>
      <td>$2,143</td>
    </tr>
  </tfoot>
</table>
```

**Why place `<tfoot>` before `<tbody>`?**
- Historical reasons (older browsers)
- Long tables: footer info loads first while data loads
- Modern HTML5: placement doesn't matter

```html
<!-- Both are valid in HTML5 -->

<!-- tfoot before tbody -->
<table>
  <thead>...</thead>
  <tfoot>...</tfoot>
  <tbody>...</tbody>
</table>

<!-- tfoot after tbody (more logical) -->
<table>
  <thead>...</thead>
  <tbody>...</tbody>
  <tfoot>...</tfoot>
</table>
```

---

### The `<tr>` Element - Table Row

Defines a row in the table. Contains `<th>` and/or `<td>` cells.

```html
<tr>
  <td>Cell 1</td>
  <td>Cell 2</td>
  <td>Cell 3</td>
</tr>
```

**All cells in a row should align:**

```html
<!-- âœ… Good: 3 cells per row -->
<table>
  <tr>
    <td>A</td>
    <td>B</td>
    <td>C</td>
  </tr>
  <tr>
    <td>D</td>
    <td>E</td>
    <td>F</td>
  </tr>
</table>

<!-- âŒ Bad: Mismatched cells (will look broken) -->
<table>
  <tr>
    <td>A</td>
    <td>B</td>
    <td>C</td>
  </tr>
  <tr>
    <td>D</td>
    <td>E</td>
    <!-- Missing cell! -->
  </tr>
</table>
```

---

### The `<th>` Element - Table Header Cell

Represents a header cell. By default, text is **bold** and **centered**.

**Use `<th>` for:**
- Column headers
- Row headers
- Any cell that describes data

**Basic Column Headers:**

```html
<table>
  <thead>
    <tr>
      <th>First Name</th>
      <th>Last Name</th>
      <th>Email</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>John</td>
      <td>Doe</td>
      <td>john@example.com</td>
    </tr>
  </tbody>
</table>
```

**Row Headers:**

```html
<table>
  <thead>
    <tr>
      <th>Month</th>
      <th>Sales</th>
      <th>Expenses</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>January</th>  <!-- Row header -->
      <td>$50,000</td>
      <td>$30,000</td>
    </tr>
    <tr>
      <th>February</th>  <!-- Row header -->
      <td>$55,000</td>
      <td>$32,000</td>
    </tr>
  </tbody>
</table>
```

---

### The `<td>` Element - Table Data Cell

Represents a data cell. Contains the actual data.

```html
<tr>
  <td>Regular text</td>
  <td>123</td>
  <td>$99.99</td>
</tr>
```

**`<td>` can contain any HTML:**

```html
<tr>
  <td>
    <!-- Images -->
    <img src="photo.jpg" alt="Product" width="100">
  </td>
  <td>
    <!-- Links -->
    <a href="details.html">View Details</a>
  </td>
  <td>
    <!-- Forms -->
    <button>Add to Cart</button>
  </td>
  <td>
    <!-- Lists -->
    <ul>
      <li>Feature 1</li>
      <li>Feature 2</li>
    </ul>
  </td>
</tr>
```

### The `scope` Attribute - For Accessibility

Tells screen readers whether a header cell applies to a row, column, or group.

**Values:**
- `scope="col"` - Header for a column
- `scope="row"` - Header for a row
- `scope="colgroup"` - Header for a group of columns
- `scope="rowgroup"` - Header for a group of rows

**Column Headers:**

```html
<table>
  <thead>
    <tr>
      <th scope="col">Name</th>
      <th scope="col">Age</th>
      <th scope="col">City</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>John</td>
      <td>30</td>
      <td>New York</td>
    </tr>
  </tbody>
</table>
```

**Row Headers:**

```html
<table>
  <thead>
    <tr>
      <th scope="col">Month</th>
      <th scope="col">Sales</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">January</th>
      <td>$50,000</td>
    </tr>
    <tr>
      <th scope="row">February</th>
      <td>$55,000</td>
    </tr>
  </tbody>
</table>
```

**Why `scope` Matters:**

When a screen reader encounters a data cell, it announces the associated headers:
- With `scope`: "January, Sales: $50,000"
- Without `scope`: "50,000" (no context!)

**Column Groups:**

```html
<table>
  <thead>
    <tr>
      <th></th>
      <th scope="colgroup" colspan="2">Q3</th>
      <th scope="colgroup" colspan="2">Q4</th>
    </tr>
    <tr>
      <th scope="col">Product</th>
      <th scope="col">Sales</th>
      <th scope="col">Profit</th>
      <th scope="col">Sales</th>
      <th scope="col">Profit</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Product A</th>
      <td>$50K</td>
      <td>$15K</td>
      <td>$60K</td>
      <td>$18K</td>
    </tr>
  </tbody>
</table>
```

- Headers not in same row/column

## 4.3 Table Accessibility

### The `<caption>` Element - Table Title

Provides a title/description for the table. Should be the **first element** inside `<table>`.

**Basic Usage:**

```html
<table>
  <caption>Employee Sales Data - Q4 2024</caption>
  <thead>
    <tr>
      <th>Name</th>
      <th>Sales</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>$50,000</td>
    </tr>
  </tbody>
</table>
```

**Why Use `<caption>`:**
1. **Accessibility** - Screen readers announce caption first
2. **Context** - Users understand table purpose before reading data
3. **SEO** - Search engines understand table content
4. **Visual** - Can be styled for presentation

**Styling Captions:**

```html
<style>
  caption {
    caption-side: top; /* or bottom */
    text-align: left;
    font-size: 1.2em;
    font-weight: bold;
    padding: 10px;
    background-color: #f0f0f0;
  }
</style>

<table>
  <caption>Monthly Sales Report</caption>
  <!-- table content -->
</table>
```

**Caption Position:**

```css
/* Caption above table (default) */
caption {
  caption-side: top;
}

/* Caption below table */
caption {
  caption-side: bottom;
}
```

### Limitations of `<colgroup>` and `<col>`

âš ï¸ **Important:** You can only apply **limited CSS properties** to `<col>`:

### Strategy 2: Card Layout on Mobile

```html
<style>
  @media (max-width: 768px) {
    /* Hide table headers */
    thead {
      display: none;
    }
    
    /* Display rows as cards */
    tbody tr {
      display: block;
      margin-bottom: 20px;
      border: 1px solid #ddd;
      border-radius: 8px;
      padding: 10px;
    }
    
    /* Display cells as rows */
    tbody td {
      display: block;
      text-align: right;
      padding: 10px;
      border-bottom: 1px solid #eee;
    }
    
    /* Add labels using data attributes */
    tbody td::before {
      content: attr(data-label);
      float: left;
      font-weight: bold;
    }
  }
</style>
```

## Key Takeaways from Chapter 4

1. **Tables are for tabular data only** - Not for layout!
2. **Always use semantic structure** - `<thead>`, `<tbody>`, `<tfoot>`
3. **`<th>` for headers**, `<td>` for data
4. **`<caption>` is essential** for accessibility
5. **Use `scope` attribute** on all `<th>` elements
6. **`colspan`** spans columns, **`rowspan`** spans rows
7. **Use `headers` attribute** for complex tables
8. **`<colgroup>` and `<col>`** for column styling (limited properties)
9. **Make tables responsive** with overflow or card layout
10. **Always test with screen readers** for accessibility
