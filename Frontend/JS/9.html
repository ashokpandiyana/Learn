üéØ 1. ES6 Classes - Modern Syntax

ES6 Classes provide a cleaner, more intuitive syntax for creating objects and implementing inheritance. They're syntactic sugar over JavaScript's prototype-based inheritance.

üí° Important: Classes are NOT a new object model! They're just a nicer syntax for the prototype pattern you learned in Chapter 8. Under the hood, it's still prototypes!

1.1 Basic Class Syntax

// ES5 Constructor Function (old way) function PersonOld(name, age) { this.name = name; this.age = age; } PersonOld.prototype.greet = function() { return `Hello, I'm ${this.name}`; }; // ES6 Class (modern way) class Person { constructor(name, age) { this.name = name; this.age = age; } greet() { return `Hello, I'm ${this.name}`; } } // Both work the same way const alice = new Person("Alice", 25); console.log(alice.greet()); // "Hello, I'm Alice"

1.2 Class Declaration vs Expression

// Class Declaration class Person { constructor(name) { this.name = name; } } // Class Expression (Anonymous) const Person2 = class { constructor(name) { this.name = name; } }; // Class Expression (Named) const Person3 = class PersonClass { constructor(name) { this.name = name; } }; // All work the same way const p1 = new Person("Alice"); const p2 = new Person2("Bob"); const p3 = new Person3("Charlie");
‚ö†Ô∏è Important Differences from Functions:
  • Classes are NOT hoisted (unlike function declarations)
  • Classes always run in strict mode
  • Class methods are non-enumerable by default
  • Must call classes with new - can't call as regular function
// Classes are NOT hoisted const p = new MyClass(); // ‚ùå ReferenceError class MyClass {} // Must use 'new' class Person { constructor(name) { this.name = name; } } const alice = new Person("Alice"); // ‚úÖ Works const bob = Person("Bob"); // ‚ùå TypeError: Cannot call class as function

üî® 2. Constructor and Methods

2.1 The Constructor

class Person { // Constructor runs when new instance is created constructor(name, age, email) { // Initialize instance properties this.name = name; this.age = age; this.email = email; // Can include validation if (age < 0) { throw new Error("Age cannot be negative"); } // Can call methods this.logCreation(); } logCreation() { console.log(`Person ${this.name} created`); } } const alice = new Person("Alice", 25, "alice@example.com"); // Logs: "Person Alice created"
üí° Constructor Rules:
  • Only ONE constructor per class
  • Constructor is optional (default one is provided)
  • Constructor automatically returns this
  • Can return different object (rarely needed)

2.2 Instance Methods

class Calculator { constructor(value = 0) { this.value = value; } // Instance methods (on prototype) add(n) { this.value += n; return this; // Enable chaining } subtract(n) { this.value -= n; return this; } multiply(n) { this.value *= n; return this; } getValue() { return this.value; } } const calc = new Calculator(10); // Method chaining const result = calc .add(5) // 15 .multiply(2) // 30 .subtract(10) // 20 .getValue(); console.log(result); // 20 // Methods are on prototype (shared) const calc2 = new Calculator(); console.log(calc.add === calc2.add); // true ‚úÖ
üí° Method Chaining: Return this from methods to enable chaining. This pattern is used in many libraries (jQuery, Lodash, etc.).

üéõÔ∏è 3. Getters and Setters

Getters and setters allow you to define methods that are accessed like properties, enabling validation and computed properties.

3.1 Basic Getters and Setters

class Person { constructor(firstName, lastName) { this.firstName = firstName; this.lastName = lastName; } // Getter - accessed like a property get fullName() { return `${this.firstName} ${this.lastName}`; } // Setter - assigned like a property set fullName(name) { const parts = name.split(" "); this.firstName = parts[0]; this.lastName = parts[1]; } } const person = new Person("Alice", "Johnson"); // Use getter (no parentheses!) console.log(person.fullName); // "Alice Johnson" // Use setter (assign like property) person.fullName = "Bob Smith"; console.log(person.firstName); // "Bob" console.log(person.lastName); // "Smith"

3.2 Validation with Setters

class User { constructor(username, age) { this.username = username; this._age = age; // Convention: _ prefix for "private" } // Getter with validation get age() { return this._age; } // Setter with validation set age(value) { if (typeof value !== 'number') { throw new TypeError("Age must be a number"); } if (value < 0 || value > 150) { throw new RangeError("Age must be between 0 and 150"); } this._age = value; } } const user = new User("alice", 25); console.log(user.age); // 25 (uses getter) user.age = 30; // ‚úÖ Valid (uses setter) console.log(user.age); // 30 user.age = 200; // ‚ùå RangeError: Age must be between 0 and 150 user.age = "thirty"; // ‚ùå TypeError: Age must be a number

3.3 Computed Properties

class Rectangle { constructor(width, height) { this.width = width; this.height = height; } // Computed property - calculated on access get area() { return this.width * this.height; } get perimeter() { return 2 * (this.width + this.height); } // Read-only property (no setter) get isSquare() { return this.width === this.height; } } const rect = new Rectangle(5, 10); console.log(rect.area); // 50 (computed) console.log(rect.perimeter); // 30 (computed) console.log(rect.isSquare); // false // Changes automatically reflect rect.width = 10; console.log(rect.area); // 100 (updated!) console.log(rect.isSquare); // true (now it's a square)
‚úÖ Benefits of Getters/Setters:
  • Validation - ensure data integrity
  • Computed properties - calculate on demand
  • Encapsulation - hide internal implementation
  • Backwards compatibility - can change internal structure

‚ö° 4. Static Methods and Properties

Static members belong to the class itself, not to instances. They're called on the class, not on objects.

4.1 Static Methods

class MathUtils { // Static method - called on class, not instance static add(a, b) { return a + b; } static multiply(a, b) { return a * b; } static max(...numbers) { return Math.max(...numbers); } } // Call static methods on CLASS console.log(MathUtils.add(5, 3)); // 8 ‚úÖ console.log(MathUtils.multiply(4, 7)); // 28 ‚úÖ console.log(MathUtils.max(1, 5, 3, 9)); // 9 ‚úÖ // Can't call on instances const util = new MathUtils(); console.log(util.add(5, 3)); // ‚ùå TypeError: util.add is not a function

4.2 Static Properties (ES2022)

class Config { // Static properties static appName = "MyApp"; static version = "1.0.0"; static maxUsers = 1000; static getInfo() { return `${this.appName} v${this.version}`; } } console.log(Config.appName); // "MyApp" console.log(Config.version); // "1.0.0" console.log(Config.getInfo()); // "MyApp v1.0.0"

4.3 Real-World Example: Factory Methods

class User { constructor(name, email, role) { this.name = name; this.email = email; this.role = role; } // Static factory methods static createAdmin(name, email) { return new User(name, email, "admin"); } static createGuest(name) { return new User(name, `${name}@guest.com`, "guest"); } static fromJSON(json) { const data = JSON.parse(json); return new User(data.name, data.email, data.role); } } // Using factory methods const admin = User.createAdmin("Alice", "alice@example.com"); const guest = User.createGuest("Bob"); console.log(admin.role); // "admin" console.log(guest.email); // "Bob@guest.com" const jsonUser = '{"name":"Charlie","email":"c@test.com","role":"user"}'; const user = User.fromJSON(jsonUser); console.log(user.name); // "Charlie"
üí° Use Static Methods For:
  • Utility functions that don't need instance data
  • Factory methods that create instances
  • Helper functions related to the class
  • Constants and configuration

üß¨ 5. Inheritance with extends

ES6 classes make inheritance much cleaner with the extends keyword.

5.1 Basic Inheritance

// Parent class class Animal { constructor(name, age) { this.name = name; this.age = age; } speak() { return `${this.name} makes a sound`; } eat() { return `${this.name} is eating`; } } // Child class inherits from Animal class Dog extends Animal { constructor(name, age, breed) { super(name, age); // Call parent constructor this.breed = breed; } // Override parent method speak() { return `${this.name} barks`; } // New method specific to Dog fetch() { return `${this.name} fetches the ball`; } } const dog = new Dog("Buddy", 3, "Golden Retriever"); console.log(dog.speak()); // "Buddy barks" (overridden) console.log(dog.eat()); // "Buddy is eating" (inherited) console.log(dog.fetch()); // "Buddy fetches the ball" (new) // Check inheritance console.log(dog instanceof Dog); // true console.log(dog instanceof Animal); // true

5.2 The super Keyword

class Animal { constructor(name) { this.name = name; } speak() { return `${this.name} makes a sound`; } } class Dog extends Animal { constructor(name, breed) { // MUST call super() before using 'this' super(name); // Call parent constructor this.breed = breed; // Now we can use 'this' } speak() { // Call parent method and extend it const parentSpeak = super.speak(); return `${parentSpeak} and barks!`; } } const dog = new Dog("Buddy", "Golden Retriever"); console.log(dog.speak()); // "Buddy makes a sound and barks!"
‚ö†Ô∏è super() Rules:
  • MUST call super() before accessing this in child constructor
  • If child has no constructor, super() is called automatically
  • super.method() calls parent method
  • super() can only be called in constructor
// ‚ùå WRONG: Using 'this' before super() class Dog extends Animal { constructor(name, breed) { this.breed = breed; // ‚ùå ReferenceError! super(name); } } // ‚úÖ CORRECT: super() first class Dog extends Animal { constructor(name, breed) { super(name); // ‚úÖ Call parent first this.breed = breed; // ‚úÖ Now we can use 'this' } }

5.3 Multi-level Inheritance

class Animal { constructor(name) { this.name = name; } move() { return `${this.name} moves`; } } class Mammal extends Animal { constructor(name, hasFur) { super(name); this.hasFur = hasFur; } breathe() { return `${this.name} breathes air`; } } class Dog extends Mammal { constructor(name, breed) { super(name, true); this.breed = breed; } bark() { return `${this.name} barks`; } } const dog = new Dog("Buddy", "Golden Retriever"); console.log(dog.move()); // From Animal console.log(dog.breathe()); // From Mammal console.log(dog.bark()); // From Dog // Check inheritance chain console.log(dog instanceof Dog); // true console.log(dog instanceof Mammal); // true console.log(dog instanceof Animal); // true

‚ö° 6. Static Methods - Advanced Usage

6.1 Static Methods in Inheritance

class Animal { static kingdom = "Animalia"; static getKingdom() { return this.kingdom; } static compareAnimals(a, b) { return a.age - b.age; } } class Dog extends Animal { static species = "Canis familiaris"; static getSpecies() { return this.species; } } // Static methods are inherited! console.log(Dog.getKingdom()); // "Animalia" (inherited) console.log(Dog.getSpecies()); // "Canis familiaris" // Use for utility functions const dog1 = new Dog(); dog1.age = 3; const dog2 = new Dog(); dog2.age = 5; const result = Dog.compareAnimals(dog1, dog2); console.log(result); // -2 (dog1 is younger)

üîí 7. Private Fields and Methods

ES2022 introduced true private fields using the # prefix. They're truly private and cannot be accessed from outside the class.

7.1 Private Fields

class BankAccount { // Private fields (must be declared) #balance = 0; #pin; constructor(initialBalance, pin) { this.#balance = initialBalance; this.#pin = pin; } // Public method to access private field getBalance() { return this.#balance; } deposit(amount) { if (amount > 0) { this.#balance += amount; return true; } return false; } withdraw(amount, pin) { if (!this.#verifyPin(pin)) { throw new Error("Invalid PIN"); } if (amount <= this.#balance) { this.#balance -= amount; return amount; } throw new Error("Insufficient funds"); } // Private method #verifyPin(pin) { return pin === this.#pin; } } const account = new BankAccount(1000, "1234"); console.log(account.getBalance()); // 1000 ‚úÖ account.deposit(500); console.log(account.getBalance()); // 1500 ‚úÖ // Cannot access private fields directly console.log(account.#balance); // ‚ùå SyntaxError: Private field console.log(account.#pin); // ‚ùå SyntaxError: Private field account.#verifyPin("1234"); // ‚ùå SyntaxError: Private method
‚úÖ True Privacy: Private fields with # are truly private! They cannot be accessed from outside, even with tricks like bracket notation or Object.keys().

6.2 Private vs Convention-based Privacy

// Old way: Convention with underscore (not truly private) class OldWay { constructor() { this._privateData = "secret"; // Convention, not enforced } } const old = new OldWay(); console.log(old._privateData); // "secret" - still accessible! ‚ùå // New way: True private fields with # class NewWay { #privateData = "secret"; // Truly private getData() { return this.#privateData; } } const newObj = new NewWay(); console.log(newObj.#privateData); // ‚ùå SyntaxError console.log(newObj.getData()); // "secret" ‚úÖ (via public method)

‚ú® 9. Best Practices and Patterns

9.1 Composition over Inheritance

// Prefer composition (combining behaviors) over deep inheritance // Instead of deep inheritance: class Animal extends LivingThing {} class Mammal extends Animal {} class Dog extends Mammal {} // Deep hierarchy ‚ùå // Use composition with mixins: const canEat = { eat(food) { return `${this.name} eats ${food}`; } }; const canWalk = { walk() { return `${this.name} walks`; } }; const canSwim = { swim() { return `${this.name} swims`; } }; class Dog { constructor(name) { this.name = name; } } // Mix in behaviors Object.assign(Dog.prototype, canEat, canWalk); const dog = new Dog("Buddy"); console.log(dog.eat("kibble")); // "Buddy eats kibble" console.log(dog.walk()); // "Buddy walks"

10.1 Computed Method Names

// Dynamic method names using [] const methodName = "greet"; const prefix = "get"; class Person { constructor(name) { this.name = name; } // Computed method name [methodName]() { return `Hello, I'm ${this.name}`; } [prefix + "Name"]() { return this.name; } } const person = new Person("Alice"); console.log(person.greet()); // "Hello, I'm Alice" console.log(person.getName()); // "Alice"

10.2 Class Fields (Public)

// Public class fields (ES2022) class Counter { // Public field (without constructor) count = 0; // Can be initialized with expression id = Math.random(); increment() { this.count++; } } const counter = new Counter(); console.log(counter.count); // 0 counter.increment(); console.log(counter.count); // 1

10.3 Instance Methods vs Arrow Functions as Properties

class Component { constructor() { this.data = "important"; } // Regular method (on prototype, shared) regularMethod() { console.log(this.data); } // Arrow function as property (on instance, not shared) arrowMethod = () => { console.log(this.data); } } const comp1 = new Component(); const comp2 = new Component(); // Regular methods are shared console.log(comp1.regularMethod === comp2.regularMethod); // true // Arrow functions are per-instance console.log(comp1.arrowMethod === comp2.arrowMethod); // false
üí° When to use Arrow Functions as Properties:
  • Event handlers (automatically binds this)
  • Callbacks that need access to instance
  • React component methods
Trade-off: Uses more memory (not shared) but avoids binding issues.

10.4 Extending Built-in Classes

// Extend Array to create custom collection class MyArray extends Array { // Add custom methods first() { return this[0]; } last() { return this[this.length - 1]; } sum() { return this.reduce((a, b) => a + b, 0); } } const arr = new MyArray(1, 2, 3, 4, 5); console.log(arr.first()); // 1 console.log(arr.last()); // 5 console.log(arr.sum()); // 15 // Still has all Array methods arr.push(6); console.log(arr.length); // 6 // Custom Error class class ValidationError extends Error { constructor(message) { super(message); this.name = "ValidationError"; } } throw new ValidationError("Invalid input"); // ValidationError: Invalid input

üìù Chapter Summary & Interview Questions

Key Takeaways:

  1. ES6 Classes are syntactic sugar over prototypes
  2. Constructor initializes new instances (only one per class)
  3. Methods are automatically added to prototype (shared)
  4. Getters/Setters provide controlled property access with validation
  5. Static members belong to class, called on class not instances
  6. extends creates clean inheritance (vs manual prototype setup)
  7. super() must be called before using this in child constructor
  8. # private fields are truly private and enforced (ES2022)
  9. Classes must use new - can't be called as regular functions
  10. Not hoisted - declare before use

ES5 vs ES6 Quick Comparison:

Creating an object with methods:

ES5: function + prototype + manual setup (10+ lines)
ES6: class + clean syntax (5 lines)

Inheritance:
ES5: Object.create + call + constructor fix (complex)
ES6: extends + super (simple)

Privacy:
ES5: Closures or _ convention (not enforced)
ES6: # private fields (enforced)

Class Pattern Template:

Common Interview Questions:

  1. Q: Are ES6 classes different from prototypes?
    A: No, they're syntactic sugar. Classes are converted to constructor functions and prototypes under the hood. JavaScript still uses prototypal inheritance.
  2. Q: What's the difference between static and instance methods?
    A: Static methods belong to the class itself and are called on the class (ClassName.method()). Instance methods belong to instances and are called on objects (obj.method()).
  3. Q: Why must super() be called before using this?
    A: The parent constructor needs to initialize the object first. In JavaScript, the parent creates the instance, and the child constructor extends it. You can't use this until the parent has set up the object.
  4. Q: What's the difference between getters/setters and regular methods?
    A: Getters/setters are accessed like properties (no parentheses): obj.prop vs obj.method(). They enable validation, computed properties, and controlled access.
  5. Q: How do private fields (#) differ from _ convention?
    A: # creates truly private fields enforced by JavaScript (SyntaxError if accessed outside). _ is just a naming convention with no enforcement - still accessible.
  6. Q: Can you call a class without new?
    A: No, classes must be called with new, or they throw a TypeError. This prevents accidental global pollution that was possible with constructor functions.
  7. Q: What happens if you don't define a constructor?
    A: JavaScript provides a default empty constructor. For child classes, the default constructor automatically calls super() with all arguments.
  8. Q: Can static methods access instance properties?
    A: No, static methods can't access instance properties/methods directly because they're called on the class, not on instances. They don't have access to this referring to an instance.
  9. Q: What's method chaining and how do you implement it?
    A: Method chaining allows calling multiple methods in sequence: obj.method1().method2(). Implement by returning this from each method.
  10. Q: Why use composition over inheritance?
    A: Deep inheritance hierarchies become rigid and hard to maintain. Composition (mixing in behaviors) is more flexible and avoids the fragile base class problem.
