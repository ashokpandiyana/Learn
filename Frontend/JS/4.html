1.2 Accessing Object Properties

const user = { firstName: "John", lastName: "Doe", age: 30, "favorite-color": "blue" // Key with special characters }; // Dot notation (most common) console.log(user.firstName); // "John" console.log(user.age); // 30 // Bracket notation console.log(user["firstName"]); // "John" console.log(user["favorite-color"]); // "blue" (special chars require brackets) // Dynamic property access const propertyName = "age"; console.log(user[propertyName]); // 30 // Accessing non-existent property console.log(user.email); // undefined (not an error!)

1.3 Adding, Modifying, and Deleting Properties

const product = { name: "Laptop", price: 999 }; // Adding new property product.brand = "Dell"; product["stock"] = 50; // Modifying existing property product.price = 899; // Deleting property delete product.stock; console.log(product); // { name: "Laptop", price: 899, brand: "Dell" } // Checking if property exists console.log("name" in product); // true console.log("stock" in product); // false (deleted) console.log(product.hasOwnProperty("price")); // true

1.4 Object Methods

// Methods are functions stored as object properties const calculator = { value: 0, // Method shorthand (ES6) add(num) { this.value += num; return this; // Method chaining }, // Traditional method syntax subtract: function(num) { this.value -= num; return this; }, // Arrow function (careful with 'this'!) getValue: () => { return this.value; // ‚ùå 'this' doesn't work here! }, // Getter get current() { return this.value; }, // Setter set current(newValue) { this.value = newValue; } }; // Method chaining calculator.add(10).add(5).subtract(3); console.log(calculator.current); // 12 calculator.current = 100; // Using setter console.log(calculator.current); // 100
‚ö†Ô∏è Arrow Functions in Objects: Don't use arrow functions for object methods if you need to access this. Arrow functions don't have their own this binding!

1.5 Important Object Methods

Object.keys(obj)

Returns an array of object's own property names.

const user = { name: "Alice", age: 25, city: "NYC" }; console.log(Object.keys(user)); // ["name", "age", "city"]

Object.values(obj)

Returns an array of object's own property values.

const user = { name: "Alice", age: 25, city: "NYC" }; console.log(Object.values(user)); // ["Alice", 25, "NYC"]

Object.entries(obj)

Returns an array of [key, value] pairs.

const user = { name: "Alice", age: 25 }; console.log(Object.entries(user)); // [["name", "Alice"], ["age", 25]] // Useful for iteration for (const [key, value] of Object.entries(user)) { console.log(`${key}: ${value}`); }

Object.assign(target, ...sources)

Copies properties from source objects to target (shallow copy).

const target = { a: 1, b: 2 }; const source = { b: 3, c: 4 }; const result = Object.assign(target, source); console.log(result); // { a: 1, b: 3, c: 4 } console.log(target); // { a: 1, b: 3, c: 4 } (modified!) // Clone without modifying original const clone = Object.assign({}, target);

üìä 2. Arrays - Ordered Collections

Arrays are ordered lists of values. They can hold any type of data and are zero-indexed.

2.1 Creating Arrays

// Array literal (most common) const fruits = ["apple", "banana", "orange"]; // Using Array constructor const numbers = new Array(1, 2, 3, 4, 5); // Empty array with specific length const empty = new Array(5); // [empty √ó 5] // Array of mixed types const mixed = [1, "hello", true, null, { name: "Alice" }]; // Array.of() - creates array from arguments const arr = Array.of(1, 2, 3); // [1, 2, 3] // Array.from() - creates array from iterable const str = Array.from("hello"); // ["h", "e", "l", "l", "o"]

2.2 Accessing and Modifying Arrays

const colors = ["red", "green", "blue"]; // Accessing elements console.log(colors[0]); // "red" console.log(colors[1]); // "green" console.log(colors[10]); // undefined (out of bounds) // Array length console.log(colors.length); // 3 // Last element console.log(colors[colors.length - 1]); // "blue" // Modifying elements colors[1] = "yellow"; console.log(colors); // ["red", "yellow", "blue"] // Adding beyond length creates sparse array colors[5] = "purple"; console.log(colors); // ["red", "yellow", "blue", empty √ó 2, "purple"]

2.3 Array Methods - Adding/Removing Elements

push() - Add to end

Adds elements to end. Returns new length. Mutates array.

const arr = [1, 2, 3]; arr.push(4, 5); console.log(arr); // [1, 2, 3, 4, 5]

pop() - Remove from end

Removes last element. Returns removed element. Mutates array.

const arr = [1, 2, 3]; const last = arr.pop(); console.log(last); // 3 console.log(arr); // [1, 2]

unshift() - Add to beginning

Adds elements to start. Returns new length. Mutates array.

const arr = [3, 4, 5]; arr.unshift(1, 2); console.log(arr); // [1, 2, 3, 4, 5]

shift() - Remove from beginning

Removes first element. Returns removed element. Mutates array.

const arr = [1, 2, 3]; const first = arr.shift(); console.log(first); // 1 console.log(arr); // [2, 3]
üí° Performance Tip: push() and pop() are fast (O(1)). shift() and unshift() are slower (O(n)) because they reindex the entire array.

2.4 slice() vs splice()

‚ö†Ô∏è Most Confused Methods! These are commonly confused in interviews. Remember: slice doesn't mutate, splice does!

slice(start, end)

Returns shallow copy of portion. Does NOT mutate.

const arr = [1, 2, 3, 4, 5]; // Get elements from index 1 to 3 (exclusive) const sliced = arr.slice(1, 3); console.log(sliced); // [2, 3] console.log(arr); // [1,2,3,4,5] ‚úÖ unchanged // Negative indices arr.slice(-2); // [4, 5] // Copy entire array const copy = arr.slice();

splice(start, deleteCount, ...items)

Changes array by removing/replacing/adding. MUTATES array.

const arr = [1, 2, 3, 4, 5]; // Remove 2 elements starting at index 1 const removed = arr.splice(1, 2); console.log(removed); // [2, 3] console.log(arr); // [1,4,5] ‚ùå mutated! // Insert elements arr.splice(1, 0, 2, 3); console.log(arr); // [1,2,3,4,5]
Feature slice() splice()
Mutates array? ‚ùå No ‚úÖ Yes
Returns New array Removed elements
Purpose Extract portion Remove/insert/replace
Parameters (start, end) (start, deleteCount, ...items)

üîÑ 3. Array Iteration Methods

Modern array methods for functional programming. These methods don't mutate the original array (except forEach).

3.1 forEach() - Execute function for each element

const numbers = [1, 2, 3, 4, 5]; // Basic forEach numbers.forEach(function(num) { console.log(num); }); // With arrow function numbers.forEach(num => console.log(num)); // Access index and array numbers.forEach((num, index, arr) => { console.log(`Index ${index}: ${num}`); }); // forEach doesn't return anything (returns undefined) const result = numbers.forEach(num => num * 2); console.log(result); // undefined
üí° forEach: Cannot use break or continue. Cannot return early. Use for loop if you need these features.

3.2 map() - Transform each element

const numbers = [1, 2, 3, 4, 5]; // Double each number const doubled = numbers.map(num => num * 2); console.log(doubled); // [2, 4, 6, 8, 10] console.log(numbers); // [1, 2, 3, 4, 5] (unchanged) // Transform objects const users = [ { name: "Alice", age: 25 }, { name: "Bob", age: 30 } ]; const names = users.map(user => user.name); console.log(names); // ["Alice", "Bob"] // With index const indexed = numbers.map((num, index) => `${index}: ${num}`); console.log(indexed); // ["0: 1", "1: 2", ...]
üí° When to use map(): When you need a new array with the same length but transformed values. Always returns new array.

3.3 filter() - Select elements that pass test

const numbers = [1, 2, 3, 4, 5, 6]; // Get even numbers const evens = numbers.filter(num => num % 2 === 0); console.log(evens); // [2, 4, 6] // Filter objects const users = [ { name: "Alice", age: 25, active: true }, { name: "Bob", age: 17, active: false }, { name: "Charlie", age: 30, active: true } ]; // Get active adult users const activeAdults = users.filter(user => { return user.age >= 18 && user.active; }); console.log(activeAdults); // [Alice, Charlie] // Remove falsy values const mixed = [1, 0, null, "hello", undefined, 5]; const truthy = mixed.filter(Boolean); console.log(truthy); // [1, "hello", 5]

3.4 reduce() - Reduce array to single value

// Syntax: array.reduce((accumulator, currentValue) => ..., initialValue) const numbers = [1, 2, 3, 4, 5]; // Sum all numbers const sum = numbers.reduce((total, num) => total + num, 0); console.log(sum); // 15 // Find maximum const max = numbers.reduce((max, num) => num > max ? num : max, numbers[0]); console.log(max); // 5 // Count occurrences const fruits = ["apple", "banana", "apple", "orange", "banana", "apple"]; const count = fruits.reduce((acc, fruit) => { acc[fruit] = (acc[fruit] || 0) + 1; return acc; }, {}); console.log(count); // { apple: 3, banana: 2, orange: 1 } // Flatten array const nested = [[1, 2], [3, 4], [5]]; const flat = nested.reduce((acc, arr) => acc.concat(arr), []); console.log(flat); // [1, 2, 3, 4, 5]

reduce() Execution Flow

Sum example: [1, 2, 3, 4].reduce((acc, num) => acc + num, 0)

Step 1: acc = 0, num = 1 ‚Üí return 0 + 1 = 1
Step 2: acc = 1, num = 2 ‚Üí return 1 + 2 = 3
Step 3: acc = 3, num = 3 ‚Üí return 3 + 3 = 6
Step 4: acc = 6, num = 4 ‚Üí return 6 + 4 = 10
Final Result: 10
‚ö†Ô∏è Common reduce() Mistake: Always provide an initial value! Without it, reduce uses first element as initial value and starts from second element.

3.5 find() and findIndex()

find() - First match

Returns first element that passes test, or undefined.

const users = [ { id: 1, name: "Alice" }, { id: 2, name: "Bob" }, { id: 3, name: "Charlie" } ]; const user = users.find(u => u.id === 2); console.log(user); // { id: 2, name: "Bob" } const missing = users.find(u => u.id === 99); console.log(missing); // undefined

findIndex() - Index of first match

Returns index of first element that passes test, or -1.

const numbers = [10, 20, 30, 40]; const index = numbers.findIndex(n => n > 25); console.log(index); // 2 (element 30) const notFound = numbers.findIndex(n => n > 100); console.log(notFound); // -1

3.6 some() and every()

some() - At least one passes

Returns true if at least one element passes test.

const numbers = [1, 2, 3, 4, 5]; const hasEven = numbers.some(n => n % 2 === 0); console.log(hasEven); // true const hasNegative = numbers.some(n => n < 0); console.log(hasNegative); // false

every() - All must pass

Returns true only if all elements pass test.

const numbers = [2, 4, 6, 8]; const allEven = numbers.every(n => n % 2 === 0); console.log(allEven); // true const allPositive = numbers.every(n => n > 0); console.log(allPositive); // true

forEach undefined Execute side effects No
map New array (same length) Transform elements No
filter New array (‚â§ length) Select elements No
reduce Single value Aggregate/accumulate No
find Element or undefined Find first match No
some Boolean Check if any pass No
every Boolean Check if all pass No

üì¶ 4. Destructuring - Unpacking Values

4.1 Array Destructuring

// Basic array destructuring const colors = ["red", "green", "blue"]; const [first, second, third] = colors; console.log(first); // "red" console.log(second); // "green" // Skip elements const [primary, , tertiary] = colors; console.log(primary); // "red" console.log(tertiary); // "blue" // Rest operator const numbers = [1, 2, 3, 4, 5]; const [one, two, ...rest] = numbers; console.log(one); // 1 console.log(two); // 2 console.log(rest); // [3, 4, 5] // Default values const [a = 10, b = 20, c = 30] = [1, 2]; console.log(a, b, c); // 1, 2, 30 // Swapping variables let x = 1, y = 2; [x, y] = [y, x]; console.log(x, y); // 2, 1

4.2 Object Destructuring

// Basic object destructuring const user = { name: "Alice", age: 25, city: "NYC" }; const { name, age, city } = user; console.log(name); // "Alice" console.log(age); // 25 // Rename variables const { name: userName, age: userAge } = user; console.log(userName); // "Alice" // Default values const { name, country = "USA" } = user; console.log(country); // "USA" (not in object) // Nested destructuring const person = { name: "Bob", address: { city: "London", country: "UK" } }; const { name, address: { city, country } } = person; console.log(city); // "London" console.log(country); // "UK" // Rest in objects const { name, ...otherInfo } = user; console.log(otherInfo); // { age: 25, city: "NYC" }

4.3 Destructuring in Function Parameters

// Array destructuring in parameters function sumFirstTwo([a, b]) { return a + b; } console.log(sumFirstTwo([1, 2, 3])); // 3 // Object destructuring in parameters function greet({ name, age }) { return `Hello ${name}, you are ${age} years old`; } console.log(greet({ name: "Alice", age: 25 })); // With defaults function createUser({ name = "Guest", role = "user" } = {}) { return { name, role }; } console.log(createUser()); // { name: "Guest", role: "user" }
üí° Destructuring Benefits:
  • Cleaner, more readable code
  • Extract only needed values
  • Default values built-in
  • Great for function parameters

üåü 5. Spread Operator - Expanding Values

5.1 Spread with Arrays

// Copy array (shallow) const original = [1, 2, 3]; const copy = [...original]; console.log(copy); // [1, 2, 3] // Combine arrays const arr1 = [1, 2]; const arr2 = [3, 4]; const combined = [...arr1, ...arr2]; console.log(combined); // [1, 2, 3, 4] // Insert elements const nums = [1, 4, 5]; const inserted = [0, ...nums, 6]; console.log(inserted); // [0, 1, 4, 5, 6] // Convert string to array const str = "hello"; const chars = [...str]; console.log(chars); // ["h", "e", "l", "l", "o"] // Remove duplicates const numbers = [1, 2, 2, 3, 3, 4]; const unique = [...new Set(numbers)]; console.log(unique); // [1, 2, 3, 4]

5.2 Spread with Objects

// Copy object (shallow) const user = { name: "Alice", age: 25 }; const userCopy = { ...user }; console.log(userCopy); // { name: "Alice", age: 25 } // Merge objects const defaults = { theme: "light", language: "en" }; const userPrefs = { language: "es", fontSize: 14 }; const settings = { ...defaults, ...userPrefs }; console.log(settings); // { theme: "light", language: "es", fontSize: 14 } // Add/override properties const person = { name: "Bob", age: 30 }; const updated = { ...person, age: 31, city: "NYC" }; console.log(updated); // { name: "Bob", age: 31, city: "NYC" }
‚ö†Ô∏è Shallow Copy Warning: Spread creates shallow copies. Nested objects/arrays are still referenced, not copied!
const original = { name: "Alice", scores: [90, 85, 95] }; const copy = { ...original }; copy.scores[0] = 100; // Mutates original too! console.log(original.scores); // [100, 85, 95] ‚ùå

üìù Chapter Summary & Interview Questions

Key Takeaways:

  1. Objects: Key-value pairs, use dot or bracket notation
  2. Arrays: Ordered collections, zero-indexed
  3. push/pop: End operations (fast O(1))
  4. shift/unshift: Beginning operations (slow O(n))
  5. slice: Returns copy, doesn't mutate
  6. splice: Modifies array, mutates original
  7. map: Transform ‚Üí new array (same length)
  8. filter: Select ‚Üí new array (‚â§ length)
  9. reduce: Aggregate ‚Üí single value
  10. Destructuring: Unpack values cleanly
  11. Spread: Copy/combine (shallow!)

Common Interview Questions:

  1. Q: What's the difference between slice and splice?
    A: slice doesn't mutate, returns new array. splice mutates original, can add/remove elements.
  2. Q: Difference between map and forEach?
    A: map returns new array, forEach returns undefined. Use map for transformations, forEach for side effects.
  3. Q: How to remove duplicates from array?
    A: [...new Set(array)] or use filter with indexOf.
  4. Q: What does reduce do?
    A: Reduces array to single value by executing reducer function on each element, passing result to next iteration.
  5. Q: How to deep clone an object?
    A: Spread/Object.assign are shallow. Use JSON.parse(JSON.stringify(obj)) or structuredClone() for deep clone.
  6. Q: Difference between for...in and for...of?
    A: for...in iterates keys (use for objects), for...of iterates values (use for arrays).
