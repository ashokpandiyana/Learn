üî§ 1. String Basics

Strings are sequences of characters used to represent text. In JavaScript, strings are immutable primitives.

1.1 Creating Strings

// Single quotes const str1 = 'Hello World'; // Double quotes const str2 = "Hello World"; // Template literals (backticks) - ES6 const str3 = `Hello World`; // Escaping quotes const quote1 = 'He said, "Hello"'; const quote2 = "It's a beautiful day"; const quote3 = 'It\'s a beautiful day'; // Escape with backslash // Using String constructor (not recommended) const str4 = new String("Hello"); // Creates String object, not primitive
‚ö†Ô∏è Important: Use string literals (quotes/backticks), not new String(). The constructor creates an object, not a primitive, which can cause unexpected behavior with === comparison.

1.2 String Immutability

// Strings are IMMUTABLE - cannot be changed let str = "Hello"; str[0] = "J"; // ‚ùå Doesn't work (no error in non-strict mode) console.log(str); // "Hello" (unchanged) // To "change" a string, create a new one str = "J" + str.slice(1); // "Jello" console.log(str); // "Jello"

1.3 String Length and Indexing

const text = "JavaScript"; // Length property console.log(text.length); // 10 // Accessing characters (0-indexed) console.log(text[0]); // "J" console.log(text[4]); // "S" console.log(text[100]); // undefined (out of bounds) // charAt() method (older way) console.log(text.charAt(0)); // "J" console.log(text.charAt(100)); // "" (empty string, not undefined) // Last character console.log(text[text.length - 1]); // "t"

üîß 2. String Methods

2.1 Case Conversion

const text = "Hello World"; // Convert to uppercase console.log(text.toUpperCase()); // "HELLO WORLD" // Convert to lowercase console.log(text.toLowerCase()); // "hello world" // Original unchanged (strings are immutable) console.log(text); // "Hello World" // Practical use: Case-insensitive comparison const input = "HELLO"; const stored = "hello"; console.log(input.toLowerCase() === stored); // true

2.2 Extracting Substrings - slice, substring, substr

‚ö†Ô∏è Interview Favorite: These three methods are commonly confused. Know the differences!

slice(start, end)

Extracts section from start to end (exclusive). Supports negative indices.

slice(startIndex, endIndex)
const str = "JavaScript"; // Basic slice console.log(str.slice(0, 4)); // "Java" console.log(str.slice(4, 10)); // "Script" // Omit end (goes to end of string) console.log(str.slice(4)); // "Script" // Negative indices (count from end) console.log(str.slice(-6)); // "Script" (last 6 chars) console.log(str.slice(0, -6)); // "Java" (exclude last 6)

substring(start, end)

Similar to slice but doesn't support negative indices. Swaps arguments if start > end.

substring(startIndex, endIndex)
const str = "JavaScript"; // Basic substring console.log(str.substring(0, 4)); // "Java" // Swaps if start > end console.log(str.substring(4, 0)); // "Java" (same as 0, 4) // Negative treated as 0 console.log(str.substring(-3)); // "JavaScript" (starts at 0)

substr(start, length) - DEPRECATED ‚ö†Ô∏è

Extracts characters from start, for given length. Avoid using - use slice instead!

substr(startIndex, length)
const str = "JavaScript"; // substr uses LENGTH, not end index console.log(str.substr(0, 4)); // "Java" (start at 0, take 4 chars) console.log(str.substr(4, 6)); // "Script" (start at 4, take 6 chars) // ‚ö†Ô∏è DEPRECATED - use slice() instead!
Method Parameters Negative Indices Status
slice (start, end) ‚úÖ Supported ‚úÖ Recommended
substring (start, end) ‚ùå Treats as 0 ‚úÖ OK to use
substr (start, length) ‚úÖ For start only ‚ùå Deprecated

2.3 Searching and Matching

indexOf()

Returns first index of substring, or -1 if not found.

const str = "Hello World"; console.log(str.indexOf("o")); // 4 console.log(str.indexOf("World")); // 6 console.log(str.indexOf("xyz")); // -1 // Start search from index console.log(str.indexOf("o", 5)); // 7 (2nd "o")

lastIndexOf()

Returns last index of substring, searching backwards.

const str = "Hello World"; console.log(str.lastIndexOf("o")); // 7 console.log(str.lastIndexOf("l")); // 9 // Search backwards from index console.log(str.lastIndexOf("o", 5)); // 4

includes()

Returns true/false if string contains substring. (ES6)

const str = "Hello World"; console.log(str.includes("World")); // true console.log(str.includes("xyz")); // false // Case sensitive console.log(str.includes("world")); // false

startsWith() / endsWith()

Check if string starts/ends with substring. (ES6)

const str = "JavaScript"; console.log(str.startsWith("Java")); // true console.log(str.startsWith("Script")); // false console.log(str.endsWith("Script")); // true console.log(str.endsWith("Java")); // false

2.4 Trimming and Padding

// trim() - removes whitespace from both ends const str = " Hello World "; console.log(str.trim()); // "Hello World" console.log(str.trimStart()); // "Hello World " (ES2019) console.log(str.trimEnd()); // " Hello World" (ES2019) // padStart() / padEnd() - pad to specific length (ES2017) const num = "5"; console.log(num.padStart(3, "0")); // "005" console.log(num.padEnd(3, "0")); // "500" // Credit card masking const last4 = "1234"; const masked = last4.padStart(16, "*"); console.log(masked); // "************1234"

2.5 Replacing and Repeating

// replace() - replaces FIRST occurrence const text = "Hello World, Hello Everyone"; console.log(text.replace("Hello", "Hi")); // "Hi World, Hello Everyone" (only first) // replaceAll() - replaces ALL occurrences (ES2021) console.log(text.replaceAll("Hello", "Hi")); // "Hi World, Hi Everyone" // Replace with regex (global flag) console.log(text.replace(/Hello/g, "Hi")); // "Hi World, Hi Everyone" // repeat() - repeat string n times console.log("Ha".repeat(3)); // "HaHaHa" console.log("*".repeat(10)); // "**********"

2.6 Splitting and Joining

// split() - converts string to array const csv = "apple,banana,orange"; const fruits = csv.split(","); console.log(fruits); // ["apple", "banana", "orange"] // Split into characters const word = "hello"; console.log(word.split("")); // ["h", "e", "l", "l", "o"] // Split with limit const text = "one,two,three,four"; console.log(text.split(",", 2)); // ["one", "two"] // join() - converts array to string (opposite of split) const arr = ["Hello", "World"]; console.log(arr.join(" ")); // "Hello World" console.log(arr.join("-")); // "Hello-World" console.log(arr.join("")); // "HelloWorld"

‚ú® 3. Template Literals (ES6)

Template literals (backticks) provide powerful string features: interpolation, multi-line strings, and tagged templates.

3.1 String Interpolation

// Old way - string concatenation const name = "Alice"; const age = 25; const oldWay = "Hello, my name is " + name + " and I am " + age + " years old."; // New way - template literals const newWay = `Hello, my name is ${name} and I am ${age} years old.`; // Expressions inside ${} const a = 5, b = 10; console.log(`${a} + ${b} = ${a + b}`); // "5 + 10 = 15" // Function calls function getGreeting() { return "Hello"; } console.log(`${getGreeting()}, ${name}!`); // "Hello, Alice!" // Any expression works const items = [1, 2, 3]; console.log(`Items: ${items.length}`); // "Items: 3" console.log(`First: ${items[0]}`); // "First: 1"

3.2 Multi-line Strings

// Old way - escape sequences const oldMultiline = "Line 1\nLine 2\nLine 3"; // New way - template literals preserve line breaks const newMultiline = `Line 1 Line 2 Line 3`; // HTML templates const html = ` <div class="card"> <h2>${name}</h2> <p>Age: ${age}</p> </div> `; // SQL queries const query = ` SELECT name, email FROM users WHERE age > ${age} ORDER BY name `;
üí° Use Cases:
  • HTML/CSS/SQL templates
  • Multi-line messages
  • Complex string formatting
  • Dynamic content generation

3.3 Tagged Templates (Advanced)

// Tagged template - function processes template literal function highlight(strings, ...values) { let result = ""; strings.forEach((str, i) => { result += str; if (i < values.length) { result += `<strong>${values[i]}</strong>`; } }); return result; } const name = "Alice"; const age = 25; const result = highlight`Name: ${name}, Age: ${age}`; console.log(result); // "Name: <strong>Alice</strong>, Age: <strong>25</strong>" // Practical example: SQL query sanitization function sql(strings, ...values) { return { text: strings.join("?"), values: values }; } const userId = 123; const query = sql`SELECT * FROM users WHERE id = ${userId}`; // { text: "SELECT * FROM users WHERE id = ?", values: [123] }
üí° Tagged Templates: Advanced feature used by libraries like styled-components, GraphQL, and i18n for processing template literals.

üîç 4. Regular Expressions Basics

Regular expressions (regex) are patterns used to match character combinations in strings.

4.1 Creating Regex

// Literal notation (preferred) const regex1 = /pattern/; // Constructor notation (dynamic patterns) const regex2 = new RegExp("pattern"); // With flags const caseInsensitive = /hello/i; // i = case insensitive const global = /hello/g; // g = global (all matches) const multiline = /hello/m; // m = multiline const combined = /hello/gi; // gi = global + case insensitive

4.2 Basic Patterns

Common Regex Patterns

. (dot)

Matches any single character except newline

\d

Matches any digit (0-9)

\w

Matches any word character (a-z, A-Z, 0-9, _)

\s

Matches any whitespace character

^

Matches start of string

$

Matches end of string

*

Matches 0 or more times

+

Matches 1 or more times

?

Matches 0 or 1 time

{n}

Matches exactly n times

[abc]

Matches any character in brackets

[^abc]

Matches any character NOT in brackets

4.3 String Methods with Regex

// test() - returns true/false const emailRegex = /\w+@\w+\.\w+/; console.log(emailRegex.test("user@example.com")); // true console.log(emailRegex.test("invalid")); // false // match() - returns array of matches const text = "Call me at 555-1234 or 555-5678"; const phoneRegex = /\d{3}-\d{4}/g; console.log(text.match(phoneRegex)); // ["555-1234", "555-5678"] // replace() with regex const masked = text.replace(/\d/g, "*"); console.log(masked); // "Call me at ***-**** or ***-****" // search() - returns index of match console.log(text.search(/\d{3}/)); // 11 (first digit position) // split() with regex const words = "hello,world;foo:bar".split(/[,;:]/); console.log(words); // ["hello", "world", "foo", "bar"]

4.4 Common Regex Examples

// Email validation (basic) const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/; // Phone number (US format) const phoneRegex = /^\d{3}-\d{3}-\d{4}$/; // URL validation const urlRegex = /^https?:\/\/.+/; // Extract numbers from string const str = "Price: $123.45"; const numbers = str.match(/\d+\.?\d*/g); console.log(numbers); // ["123.45"] // Remove HTML tags const html = "<p>Hello <strong>World</strong></p>"; const plain = html.replace(/<[^>]+>/g, ""); console.log(plain); // "Hello World" // Validate password (min 8 chars, 1 uppercase, 1 number) const passwordRegex = /^(?=.*[A-Z])(?=.*\d).{8,}$/;
‚ö†Ô∏è Regex Complexity: Regular expressions can be powerful but complex. Test thoroughly and consider readability. For complex validations, sometimes simple string methods are clearer.

üé® 5. Common String Patterns

5.1 Capitalize First Letter

function capitalize(str) { return str.charAt(0).toUpperCase() + str.slice(1); } console.log(capitalize("hello")); // "Hello" // Title Case (capitalize each word) function titleCase(str) { return str.split(" ") .map(word => capitalize(word)) .join(" "); } console.log(titleCase("hello world")); // "Hello World"

5.2 Reverse a String

function reverseString(str) { return str.split("").reverse().join(""); } console.log(reverseString("hello")); // "olleh" // Or using spread operator function reverseString2(str) { return [...str].reverse().join(""); }

5.3 Check Palindrome

function isPalindrome(str) { const cleaned = str.toLowerCase().replace(/[^a-z0-9]/g, ""); const reversed = cleaned.split("").reverse().join(""); return cleaned === reversed; } console.log(isPalindrome("racecar")); // true console.log(isPalindrome("A man, a plan, a canal: Panama")); // true console.log(isPalindrome("hello")); // false

5.4 Count Vowels

function countVowels(str) { const matches = str.match(/[aeiou]/gi); return matches ? matches.length : 0; } console.log(countVowels("hello world")); // 3 // Alternative without regex function countVowels2(str) { const vowels = "aeiouAEIOU"; return [...str].filter(char => vowels.includes(char)).length; }

5.5 Truncate String

function truncate(str, maxLength) { if (str.length <= maxLength) return str; return str.slice(0, maxLength) + "..."; } console.log(truncate("Hello World", 5)); // "Hello..." console.log(truncate("Hi", 10)); // "Hi"

5.6 Slugify (URL-friendly strings)

function slugify(str) { return str .toLowerCase() .trim() .replace(/[^\w\s-]/g, "") // Remove special chars .replace(/[\s_-]+/g, "-") // Replace spaces with - .replace(/^-+|-+$/g, ""); // Remove leading/trailing - } console.log(slugify("Hello World!")); // "hello-world" console.log(slugify(" My Blog Post ")); // "my-blog-post"

üìù Chapter Summary & Interview Questions

Key Takeaways:

  1. Strings are immutable - methods return new strings
  2. slice() is most versatile (supports negative indices)
  3. Use template literals for interpolation and multi-line
  4. indexOf() returns -1 if not found
  5. includes() is modern, more readable than indexOf
  6. split/join convert between strings and arrays
  7. Regex is powerful but test thoroughly
  8. Case sensitive by default (use toLowerCase for comparison)

Common Interview Questions:

  1. Q: How do you reverse a string?
    A: str.split("").reverse().join("") or [...str].reverse().join("")
  2. Q: Difference between slice, substring, and substr?
    A: slice supports negative indices, substring swaps args if start>end, substr is deprecated.
  3. Q: How to check if string contains substring?
    A: Use includes() (modern) or indexOf() !== -1 (older).
  4. Q: How to make case-insensitive comparison?
    A: str1.toLowerCase() === str2.toLowerCase()
  5. Q: What's a template literal?
    A: Backtick strings with ${} interpolation, multi-line support, and tagged templates.
  6. Q: How to remove whitespace?
    A: trim() (both ends), trimStart(), trimEnd()
  7. Q: Write a function to check if string is palindrome.
    A: See section 5.3 above.
