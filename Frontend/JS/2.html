ü§î 1. Conditional Statements

Conditional statements allow your code to make decisions and execute different code blocks based on conditions.

1.1 if Statement

The most basic conditional - executes code if a condition is true.

Syntax:
if (condition) {
  // code to execute if condition is true
}
let age = 20; if (age >= 18) { console.log("You are an adult"); } // Multiple conditions let temperature = 30; if (temperature > 25 && temperature < 35) { console.log("Perfect weather!"); }

1.2 if-else Statement

Executes one block if condition is true, another if false.

let score = 75; if (score >= 60) { console.log("Pass"); } else { console.log("Fail"); } // Output: "Pass"

1.3 if-else if-else Ladder

Test multiple conditions in sequence.

let grade = 85; if (grade >= 90) { console.log("A - Excellent!"); } else if (grade >= 80) { console.log("B - Good!"); } else if (grade >= 70) { console.log("C - Average"); } else if (grade >= 60) { console.log("D - Pass"); } else { console.log("F - Fail"); } // Output: "B - Good!"
üí° Important: The conditions are checked in order from top to bottom. Once a condition is true, the corresponding block executes and the rest are skipped.

1.4 Nested if Statements

if statements inside other if statements for complex logic.

let age = 25; let hasLicense = true; if (age >= 18) { if (hasLicense) { console.log("You can drive"); } else { console.log("You need a license"); } } else { console.log("You are too young to drive"); }
‚ö†Ô∏è Avoid Deep Nesting: Too many nested if statements make code hard to read. Consider using logical operators (&&, ||) or early returns instead.

1.5 Ternary Operator

A shorthand for simple if-else statements.

Syntax: condition ? valueIfTrue : valueIfFalse
let age = 20; let status = age >= 18 ? "adult" : "minor"; console.log(status); // "adult" // Nested ternary (use sparingly!) let score = 85; let grade = score >= 90 ? "A" : score >= 80 ? "B" : score >= 70 ? "C" : "F"; // In function returns function canVote(age) { return age >= 18 ? "Yes" : "No"; }
üí° When to use: Use ternary for simple conditions. For complex logic, use if-else for better readability.

1.6 Switch Statement

Efficient way to check a variable against multiple values.

Syntax:
switch (expression) {
  case value1:
    // code
    break;
  default:
    // code
}
let day = 3; let dayName; switch (day) { case 1: dayName = "Monday"; break; case 2: dayName = "Tuesday"; break; case 3: dayName = "Wednesday"; break; case 4: dayName = "Thursday"; break; case 5: dayName = "Friday"; break; case 6: case 7: dayName = "Weekend"; break; default: dayName = "Invalid day"; } console.log(dayName); // "Wednesday"
‚ö†Ô∏è Don't Forget break! Without break, execution "falls through" to the next case. This is a common bug!
// Fall-through example (intentional) let month = "March"; switch (month) { case "December": case "January": case "February": console.log("Winter"); break; case "March": case "April": case "May": console.log("Spring"); break; // ... more cases }

üîÅ 2. Loops

Loops allow you to execute a block of code repeatedly. JavaScript has several types of loops.

2.1 for Loop

The most common loop - when you know how many iterations you need.

Syntax: for (initialization; condition; increment) { code }
// Basic for loop for (let i = 0; i < 5; i++) { console.log(`Iteration ${i}`); } // Output: Iteration 0, 1, 2, 3, 4 // Array iteration const fruits = ["apple", "banana", "orange"]; for (let i = 0; i < fruits.length; i++) { console.log(fruits[i]); } // Reverse loop for (let i = 5; i > 0; i--) { console.log(i); } // Output: 5, 4, 3, 2, 1 // Skip by 2 for (let i = 0; i < 10; i += 2) { console.log(i); } // Output: 0, 2, 4, 6, 8

2.2 while Loop

Executes while a condition is true - when you don't know the exact number of iterations.

let count = 0; while (count < 5) { console.log(`Count: ${count}`); count++; } // Practical example: Finding a value let numbers = [1, 3, 5, 7, 9]; let index = 0; let target = 7; while (index < numbers.length) { if (numbers[index] === target) { console.log(`Found ${target} at index ${index}`); break; } index++; }
‚ö†Ô∏è Infinite Loop Risk! Always ensure the condition will eventually become false, or you'll create an infinite loop that can crash the browser!
// ‚ùå BAD - Infinite loop! let x = 0; while (x < 10) { console.log(x); // Missing x++ - x never increases! } // ‚úÖ GOOD let x = 0; while (x < 10) { console.log(x); x++; }

2.3 do-while Loop

Similar to while, but executes at least once before checking the condition.

let num = 0; do { console.log(`Number: ${num}`); num++; } while (num < 5); // Key difference: runs at least once even if condition is false let x = 10; do { console.log("This runs once"); } while (x < 5); // Condition is false but code still ran!
Loop Type When to Use Checks Condition
for Known number of iterations Before each iteration
while Unknown iterations, may not run at all Before each iteration
do-while Unknown iterations, must run at least once After each iteration

2.4 for...in Loop

Iterates over enumerable properties of an object (including inherited ones).

// Best for objects const person = { name: "John", age: 30, city: "New York" }; for (let key in person) { console.log(`${key}: ${person[key]}`); } // Output: // name: John // age: 30 // city: New York // Can be used with arrays (but not recommended) const arr = ["a", "b", "c"]; for (let index in arr) { console.log(index); // "0", "1", "2" (strings, not numbers!) }
‚ö†Ô∏è Warning: for...in should NOT be used with arrays! It iterates over all enumerable properties, including inherited ones, and indices are strings.

2.5 for...of Loop (ES6)

Iterates over iterable values (arrays, strings, Maps, Sets, etc.).

// Best for arrays const fruits = ["apple", "banana", "orange"]; for (let fruit of fruits) { console.log(fruit); } // Output: apple, banana, orange // Works with strings const word = "hello"; for (let char of word) { console.log(char); } // Output: h, e, l, l, o // With array entries const colors = ["red", "green", "blue"]; for (let [index, color] of colors.entries()) { console.log(`${index}: ${color}`); }

for...in vs for...of - Quick Reference

‚èπÔ∏è 3. Break and Continue Statements

3.1 break Statement

Exits the loop completely, regardless of the condition.

// Exit loop when condition is met for (let i = 0; i < 10; i++) { if (i === 5) { break; // Stop the loop } console.log(i); } // Output: 0, 1, 2, 3, 4 (stops at 5) // Practical example: Finding first match const numbers = [10, 25, 30, 45, 50]; let found = false; for (let num of numbers) { if (num > 30) { console.log(`First number > 30 is ${num}`); found = true; break; // No need to check rest } } // Output: First number > 30 is 45

3.2 continue Statement

Skips the current iteration and moves to the next one.

// Skip specific values for (let i = 0; i < 10; i++) { if (i % 2 === 0) { continue; // Skip even numbers } console.log(i); } // Output: 1, 3, 5, 7, 9 (only odd numbers) // Practical example: Filter data const scores = [45, 78, -5, 92, 105, 68]; for (let score of scores) { if (score < 0 || score > 100) { console.log(`Invalid score: ${score}`); continue; // Skip invalid scores } console.log(`Valid score: ${score}`); }

3.3 Nested Loop Control

break and continue only affect the innermost loop by default.

// break in nested loop for (let i = 0; i < 3; i++) { for (let j = 0; j < 3; j++) { if (j === 1) { break; // Only breaks inner loop } console.log(`i=${i}, j=${j}`); } } // Output: i=0,j=0 i=1,j=0 i=2,j=0

üè∑Ô∏è 4. Labels

Labels allow you to identify loops and break/continue outer loops from within nested loops.

Syntax:
labelName: for (...) { ... }
// Breaking out of nested loops outerLoop: for (let i = 0; i < 3; i++) { for (let j = 0; j < 3; j++) { if (i === 1 && j === 1) { break outerLoop; // Breaks outer loop } console.log(`i=${i}, j=${j}`); } } // Output: i=0,j=0 i=0,j=1 i=0,j=2 i=1,j=0 // Practical example: 2D array search const matrix = [ [1, 2, 3], [4, 5, 6], [7, 8, 9] ]; let found = false; search: for (let i = 0; i < matrix.length; i++) { for (let j = 0; j < matrix[i].length; j++) { if (matrix[i][j] === 5) { console.log(`Found 5 at [${i}][${j}]`); found = true; break search; // Exit both loops } } } // Output: Found 5 at [1][1]
// Continue with labels outerLoop: for (let i = 0; i < 3; i++) { for (let j = 0; j < 3; j++) { if (j === 1) { continue outerLoop; // Skip to next i } console.log(`i=${i}, j=${j}`); } } // Output: i=0,j=0 i=1,j=0 i=2,j=0
üí° When to use labels:
  • Searching in 2D arrays/matrices
  • Complex nested loop logic
  • When you need to exit multiple loop levels
‚ö†Ô∏è Use Sparingly: Labels can make code harder to read. Consider refactoring into functions with early returns instead.
// Alternative to labels: Use functions function findInMatrix(matrix, target) { for (let i = 0; i < matrix.length; i++) { for (let j = 0; j < matrix[i].length; j++) { if (matrix[i][j] === target) { return { i, j }; // Early return exits all loops } } } return null; }

‚ú® 5. Best Practices & Common Patterns

5.1 Avoid Modifying Loop Variables

// ‚ùå BAD - Unpredictable behavior for (let i = 0; i < 10; i++) { i += 2; // Modifying inside loop body console.log(i); } // ‚úÖ GOOD - Use step in loop declaration for (let i = 0; i < 10; i += 3) { console.log(i); }

5.2 Cache Array Length

// ‚ùå Less efficient - recalculates length each iteration for (let i = 0; i < arr.length; i++) { // code } // ‚úÖ More efficient - caches length const len = arr.length; for (let i = 0; i < len; i++) { // code }

5.3 Choose the Right Loop

‚úÖ Modern JavaScript Preferences:
  • for...of for arrays and iterables
  • forEach() for functional approach
  • map(), filter(), reduce() for transformations
  • Traditional for when you need index control
  • for...in only for objects

5.4 Common Interview Patterns

// Pattern 1: Sum of array function sum(arr) { let total = 0; for (let num of arr) { total += num; } return total; } // Pattern 2: Find max value function findMax(arr) { let max = arr[0]; for (let num of arr) { if (num > max) { max = num; } } return max; } // Pattern 3: Reverse array in-place function reverseArray(arr) { for (let i = 0; i < arr.length / 2; i++) { let temp = arr[i]; arr[i] = arr[arr.length - 1 - i]; arr[arr.length - 1 - i] = temp; } return arr; } // Pattern 4: FizzBuzz (classic interview) for (let i = 1; i <= 100; i++) { if (i % 15 === 0) { console.log("FizzBuzz"); } else if (i % 3 === 0) { console.log("Fizz"); } else if (i % 5 === 0) { console.log("Buzz"); } else { console.log(i); } }

üìù Chapter Summary & Interview Questions

Key Takeaways:

  1. Conditionals: Use if-else for complex conditions, switch for multiple specific values, ternary for simple cases
  2. Loops: for (known iterations), while (unknown), do-while (at least once)
  3. for...in vs for...of: in for object keys, of for array values
  4. break: Exit loop completely
  5. continue: Skip current iteration
  6. Labels: Control nested loops (use sparingly)
  7. Best Practice: Always use strict equality (===), cache array lengths, choose appropriate loop type

Common Interview Questions:

  1. Q: What's the difference between for...in and for...of?
    A: for...in iterates over enumerable property names (keys), best for objects. for...of iterates over iterable values, best for arrays.
  2. Q: What's the difference between break and continue?
    A: break exits the loop entirely. continue skips the current iteration and moves to the next.
  3. Q: When would you use a do-while loop?
    A: When you need the code to execute at least once before checking the condition.
  4. Q: What's wrong with using var in loops?
    A: var is function-scoped, not block-scoped. In callbacks/closures, all iterations reference the same variable. Use let instead.
  5. Q: Write FizzBuzz.
    A: See Best Practices section above!
  6. Q: How do you break out of nested loops?
    A: Use labeled statements or extract into a function with early return.
