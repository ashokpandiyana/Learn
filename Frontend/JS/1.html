What are Variables?

Variables are containers for storing data values. Think of them as labeled boxes where you can store information and retrieve it later using the label (variable name).

Three Ways to Declare Variables

// 1. var (old way - function scoped) var name = "John"; // 2. let (modern way - block scoped, can be reassigned) let age = 25; // 3. const (modern way - block scoped, cannot be reassigned) const PI = 3.14159;

üí° Best Practice: Always use const by default. Only use let when you know the variable value will change. Avoid var in modern JavaScript.

Scope Demonstration

// Block Scope Example function scopeDemo() { var varVariable = "I'm function scoped"; if (true) { var varInside = "var ignores block"; let letInside = "let respects block"; const constInside = "const respects block"; } console.log(varInside); // ‚úÖ Works - "var ignores block" console.log(letInside); // ‚ùå Error - not defined console.log(constInside); // ‚ùå Error - not defined }
‚ö†Ô∏è Common Mistake: Using var in loops can lead to unexpected behavior due to function scoping. Always use let in loops.

üéØ 2. Primitive Data Types

JavaScript has 7 primitive data types. Primitives are immutable (cannot be changed) and stored directly in memory.

1. String

Represents textual data enclosed in quotes.

let singleQuote = 'Hello'; let doubleQuote = "World"; let backticks = `Template literals: ${singleQuote} ${doubleQuote}`; // String Methods let text = "JavaScript"; console.log(text.length); // 10 console.log(text.toUpperCase()); // "JAVASCRIPT" console.log(text[0]); // "J"

2. Number

Represents both integer and floating-point numbers.

let integer = 42; let float = 3.14; let negative = -10; let exponential = 2e3; // 2000 // Special numeric values let infinity = Infinity; let notANumber = NaN; // Not a Number console.log(1 / 0); // Infinity console.log("hello" * 2); // NaN
üí° Interview Tip: NaN is the only value in JavaScript that is not equal to itself. Use Number.isNaN() to check for NaN.

3. Boolean

Represents logical values: true or false.

let isActive = true; let isDeleted = false; // Boolean conversion console.log(Boolean(1)); // true console.log(Boolean(0)); // false console.log(Boolean("")); // false (empty string) console.log(Boolean("hello")); // true

4. Undefined

Represents a variable that has been declared but not assigned a value.

let x; console.log(x); // undefined console.log(typeof x); // "undefined"

5. Null

Represents intentional absence of any value.

let y = null; console.log(y); // null console.log(typeof y); // "object" (JavaScript bug!)
‚ö†Ô∏è JavaScript Quirk: typeof null returns "object" due to a historic bug in JavaScript that cannot be fixed without breaking existing code.

6. Symbol (ES6)

Creates unique identifiers for object properties.

let sym1 = Symbol("description"); let sym2 = Symbol("description"); console.log(sym1 === sym2); // false (each Symbol is unique)

7. BigInt (ES2020)

Represents integers larger than 2^53 - 1 (Number.MAX_SAFE_INTEGER).

let bigNum = 9007199254740991n; // Note the 'n' suffix let alsoHuge = BigInt(9007199254740991); console.log(bigNum + 1n); // 9007199254740992n

üîÑ 3. Type Coercion and Conversion

What is Type Coercion?

Type coercion is the automatic or implicit conversion of values from one data type to another.

Implicit Coercion (Automatic)

// String coercion console.log("5" + 3); // "53" (number to string) console.log("Hello" + 5); // "Hello5" // Number coercion console.log("5" - 3); // 2 (string to number) console.log("10" * "2"); // 20 (both strings to numbers) console.log("5" / 2); // 2.5 // Boolean coercion console.log(true + 1); // 2 (true becomes 1) console.log(false + 1); // 1 (false becomes 0)

Explicit Conversion (Manual)

// To String let num = 123; String(num); // "123" num.toString(); // "123" // To Number let str = "456"; Number(str); // 456 parseInt(str); // 456 parseFloat("3.14"); // 3.14 +str; // 456 (unary plus) // To Boolean Boolean(1); // true Boolean(0); // false Boolean(""); // false Boolean("hello"); // true

‚ûó 4. Operators

Arithmetic Operators

let a = 10, b = 3; console.log(a + b); // 13 (Addition) console.log(a - b); // 7 (Subtraction) console.log(a * b); // 30 (Multiplication) console.log(a / b); // 3.333... (Division) console.log(a % b); // 1 (Modulus - remainder) console.log(a ** b); // 1000 (Exponentiation - 10¬≥) // Increment/Decrement let x = 5; console.log(x++); // 5 (returns then increments) console.log(x); // 6 console.log(++x); // 7 (increments then returns)

Comparison Operators

// Equality Operators console.log(5 == "5"); // true (loose equality - type coercion) console.log(5 === "5"); // false (strict equality - no coercion) console.log(5 != "5"); // false (loose inequality) console.log(5 !== "5"); // true (strict inequality) // Relational Operators console.log(5 > 3); // true (greater than) console.log(5 < 3); // false (less than) console.log(5 >= 5); // true (greater than or equal) console.log(3 <= 5); // true (less than or equal)
‚ö†Ô∏è Critical: Always use === and !== instead of == and != to avoid unexpected type coercion bugs!

Logical Operators

// AND (&&) - both must be true console.log(true && true); // true console.log(true && false); // false // OR (||) - at least one must be true console.log(true || false); // true console.log(false || false); // false // NOT (!) - inverts boolean value console.log(!true); // false console.log(!false); // true // Short-circuit evaluation let user = null; let name = user && user.name; // null (stops at user) let defaultName = name || "Guest"; // "Guest"

Bitwise Operators

// Work on 32-bit binary representations let a = 5; // 0101 in binary let b = 3; // 0011 in binary console.log(a & b); // 1 (0001) - AND console.log(a | b); // 7 (0111) - OR console.log(a ^ b); // 6 (0110) - XOR console.log(~a); // -6 (inverts bits) console.log(a << 1); // 10 (left shift) console.log(a >> 1); // 2 (right shift)
üí° Practical Use: Bitwise operators are used for:
  • Permissions systems (checking flags)
  • Performance optimization (faster than math operations)
  • Color manipulation
  • Cryptography

üìù Chapter Summary & Interview Points

Key Takeaways:

  1. Variable Declaration: Use const by default, let when needed, avoid var
  2. Scope: let and const are block-scoped; var is function-scoped
  3. 7 Primitive Types: String, Number, Boolean, Null, Undefined, Symbol, BigInt
  4. Type Coercion: JavaScript automatically converts types - be aware of this behavior
  5. Equality: Always use === (strict) instead of == (loose)
  6. Falsy Values: false, 0, "", null, undefined, NaN
  7. Operators: Understand precedence and associativity

Common Interview Questions:

  1. Q: What's the difference between let, const, and var?
    A: Scope, hoisting, and reassignment capabilities differ.
  2. Q: What are the falsy values in JavaScript?
    A: false, 0, -0, 0n, "", null, undefined, NaN
  3. Q: What's the difference between == and ===?
    A: == does type coercion, === checks type and value
  4. Q: Why is typeof null equal to "object"?
    A: It's a JavaScript bug that can't be fixed without breaking the web
  5. Q: What is NaN and how do you check for it?
    A: "Not a Number" - use Number.isNaN() to check
