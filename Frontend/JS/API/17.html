One-Way Server Push over HTTP

üéØ What are Server-Sent Events?

Server-Sent Events (SSE) is a web technology that allows servers to push data to clients over HTTP. Unlike WebSockets, SSE is unidirectional - data flows only from server to client.

Key Characteristics:

  • üì§ One-Way: Server ‚Üí Client only
  • üåê HTTP-Based: Works over standard HTTP/HTTPS
  • üîÑ Auto-Reconnect: Automatic reconnection on disconnect
  • ‚ö° Simple: Easier than WebSockets for server push
  • üìù Text-Based: UTF-8 encoded text data
  • üéØ Event-Driven: Named events support

SSE Communication Flow

1. üåê Client

Opens connection

EventSource(url)
‚Üí

2. üîå Server

Keeps connection open

text/event-stream
‚Üí

3. üì§ Push Data

Server sends events

data: {...}
‚Üí

4. üì• Client

Receives events

onmessage

Connection stays open indefinitely. Server can push data anytime.

üí° Perfect For:
  • Live news feeds and updates
  • Social media timelines
  • Real-time notifications
  • Stock price updates
  • Live sports scores
  • Server monitoring dashboards
  • Progress updates for long operations

üöÄ Creating EventSource Connection

Basic Connection

// Create EventSource connection const eventSource = new EventSource('/api/events'); // Listen for messages eventSource.onmessage = (event) => { console.log('New message:', event.data); // Parse JSON if needed const data = JSON.parse(event.data); console.log(data); }; // Handle connection opened eventSource.onopen = (event) => { console.log('Connection opened'); }; // Handle errors eventSource.onerror = (event) => { if (event.target.readyState === EventSource.CLOSED) { console.error('Connection closed'); } else { console.error('Connection error'); } }; // Close connection eventSource.close();

With Credentials (Cookies)

// Include credentials (cookies) in requests const eventSource = new EventSource('/api/events', { withCredentials: true }); // This sends cookies with the connection request // Useful for authenticated endpoints
‚ö†Ô∏è CORS Note: When using withCredentials: true, the server must set appropriate CORS headers including Access-Control-Allow-Credentials: true.

üìä Connection States

EventSource has 3 ready states that indicate the connection status.

CONNECTING

0

Connection is being established

OPEN

1

Connection is open and receiving events

CLOSED

2

Connection is closed

// Check connection state console.log(eventSource.readyState); if (eventSource.readyState === EventSource.CONNECTING) { console.log('Connecting...'); } if (eventSource.readyState === EventSource.OPEN) { console.log('Connected and receiving events'); } if (eventSource.readyState === EventSource.CLOSED) { console.log('Connection closed'); } // Constants EventSource.CONNECTING // 0 EventSource.OPEN // 1 EventSource.CLOSED // 2

üì¨ Receiving Events

Default Message Event

// Listen to all messages (default event) eventSource.onmessage = (event) => { console.log('Data:', event.data); console.log('ID:', event.lastEventId); console.log('Origin:', event.origin); }; // Event properties event.data // Message data (string) event.lastEventId // Last event ID (for reconnection) event.origin // Origin of event source event.type // Event type ('message' by default)

Named Events

// Listen to specific named events eventSource.addEventListener('userUpdate', (event) => { const user = JSON.parse(event.data); console.log('User updated:', user); }); eventSource.addEventListener('notification', (event) => { const notification = JSON.parse(event.data); showNotification(notification.title, notification.body); }); eventSource.addEventListener('stockPrice', (event) => { const price = JSON.parse(event.data); updateStockDisplay(price.symbol, price.value); }); // Can listen to multiple event types on same connection

Error Handling

// Handle connection errors eventSource.onerror = (event) => { console.error('EventSource error'); // Check what happened if (event.target.readyState === EventSource.CONNECTING) { console.log('Reconnecting...'); } else if (event.target.readyState === EventSource.CLOSED) { console.log('Connection closed, not reconnecting'); } }; // EventSource automatically reconnects on failure // Unless server responds with HTTP 204 No Content

üìù Server-Side Event Format

The server must send events in a specific text format with proper headers.

Required Headers

// Server must send these HTTP headers Content-Type: text/event-stream Cache-Control: no-cache Connection: keep-alive // Optional but recommended Access-Control-Allow-Origin: * Access-Control-Allow-Credentials: true

Event Format

Basic Message

// Simple message (triggers onmessage) data: Hello World // Multi-line message data: First line data: Second line data: Third line // Must end with double newline

With Event ID

// ID used for reconnection (Last-Event-ID header) id: 123 data: Message with ID

Named Events

// Custom event type event: userUpdate data: {"id": 1, "name": "John"} event: notification data: {"title": "New Message", "body": "You have mail"}

With Retry Interval

// Set reconnection delay (milliseconds) retry: 3000 data: This sets retry to 3 seconds

Complete Example

// Full-featured event id: 456 event: stockUpdate retry: 5000 data: { data: "symbol": "AAPL", data: "price": 150.25, data: "change": +2.5 data: }
üí° Format Rules:
  • Each field starts with field name, colon, and value
  • Multi-line data uses multiple data: lines
  • Comments start with : (ignored by client)
  • Events must end with double newline (\n\n)
  • UTF-8 encoding required

üñ•Ô∏è Server-Side Implementation Examples

Node.js (Express)

const express = require('express'); const app = express(); app.get('/events', (req, res) => { // Set SSE headers res.setHeader('Content-Type', 'text/event-stream'); res.setHeader('Cache-Control', 'no-cache'); res.setHeader('Connection', 'keep-alive'); res.setHeader('Access-Control-Allow-Origin', '*'); // Send initial message res.write('data: Connected to event stream\n\n'); // Send updates every 2 seconds const interval = setInterval(() => { const data = { time: new Date().toISOString(), value: Math.random() }; res.write(`data: ${JSON.stringify(data)}\n\n`); }, 2000); // Clean up on client disconnect req.on('close', () => { clearInterval(interval); res.end(); }); }); app.listen(3000);

üéØ Practical Patterns

Pattern 1: Reconnection with Last Event ID

// Client automatically sends Last-Event-ID header on reconnect const eventSource = new EventSource('/events'); eventSource.onmessage = (event) => { console.log('Last ID:', event.lastEventId); console.log('Data:', event.data); }; // Server-side (Node.js) app.get('/events', (req, res) => { const lastEventId = req.headers['last-event-id']; if (lastEventId) { // Client reconnected, send missed events const missedEvents = getEventsSince(lastEventId); missedEvents.forEach(event => { res.write(`id: ${event.id}\n`); res.write(`data: ${JSON.stringify(event.data)}\n\n`); }); } // Continue sending new events... });

Pattern 2: Heartbeat/Keep-Alive

// Send periodic comments to keep connection alive // (prevents proxy timeout) // Server-side app.get('/events', (req, res) => { // Set headers... // Send heartbeat every 15 seconds const heartbeat = setInterval(() => { res.write(': heartbeat\n\n'); // Comment (ignored by client) }, 15000); // Send actual events... req.on('close', () => { clearInterval(heartbeat); }); });

Pattern 3: Graceful Shutdown

// Close connection gracefully const eventSource = new EventSource('/events'); // Listen for shutdown event eventSource.addEventListener('shutdown', (event) => { console.log('Server shutting down'); eventSource.close(); }); // Server sends shutdown event before closing // Server-side function shutdown() { clients.forEach(client => { client.write('event: shutdown\n'); client.write('data: Server maintenance\n\n'); client.end(); }); }

Pattern 4: Wrapper with Auto-Reconnect

class ReconnectingEventSource { constructor(url, options = {}) { this.url = url; this.options = options; this.reconnectDelay = options.reconnectDelay || 3000; this.maxRetries = options.maxRetries || Infinity; this.retries = 0; this.connect(); } connect() { this.eventSource = new EventSource(this.url, this.options); this.eventSource.onopen = (e) => { console.log('Connected'); this.retries = 0; this.onopen?.(e); }; this.eventSource.onmessage = (e) => { this.onmessage?.(e); }; this.eventSource.onerror = (e) => { console.error('Connection error'); this.onerror?.(e); if (this.eventSource.readyState === EventSource.CLOSED) { this.reconnect(); } }; } reconnect() { if (this.retries >= this.maxRetries) { console.error('Max retries reached'); return; } this.retries++; console.log(`Reconnecting in ${this.reconnectDelay}ms...`); setTimeout(() => this.connect(), this.reconnectDelay); } close() { this.eventSource.close(); } } // Usage const sse = new ReconnectingEventSource('/events', { reconnectDelay: 3000, maxRetries: 5 }); sse.onmessage = (e) => console.log(e.data);

‚úÖ Best Practices

DO ‚úì

  • Use HTTPS (not HTTP) in production
  • Set proper Content-Type header (text/event-stream)
  • Implement event IDs for reliable reconnection
  • Send periodic heartbeats to prevent timeout
  • Handle reconnection gracefully
  • Use named events for different message types
  • Close connections when no longer needed
  • Set appropriate retry intervals

DON'T ‚úó

  • Use SSE for bidirectional communication (use WebSockets)
  • Send binary data (SSE is text-only)
  • Forget to set Cache-Control headers
  • Open too many connections (browser limit ~6 per domain)
  • Send huge messages (split large data)
  • Ignore error handling
  • Forget to clean up on client disconnect
