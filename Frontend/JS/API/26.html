Display System Notifications to Users

üéØ What is the Notifications API?

Notifications API allows web applications to display system notifications outside the browser context. These are the same notifications you see from native apps.

Key Characteristics:

  • üîî System-Level: Shows OS notifications
  • üîí Permission-Based: Requires user consent
  • üì± Cross-Platform: Works on desktop & mobile
  • üé® Customizable: Title, body, icon, image
  • ‚ö° Interactive: Can respond to clicks
  • üåê Background: Works even when tab is closed (with Service Worker)

Notification Anatomy

üîî
Notification Title
This is the body of the notification. It can be longer and contain more details about what happened.

‚ö†Ô∏è Requirements:
  • Must run on HTTPS (except localhost)
  • Requires user permission
  • Must be triggered by user action (except with Service Workers)
  • Don't spam users (respect notification limits)
üí° Common Use Cases:
  • New message alerts (chat apps)
  • Social media updates (likes, comments)
  • Email notifications
  • Calendar reminders
  • Breaking news alerts
  • Order status updates
  • System alerts & warnings

üîê Requesting Permission

Before showing notifications, you must request and receive permission from the user.

Permission States

granted

User has granted permission to show notifications

denied

User has denied permission (cannot ask again)

default

User hasn't been asked yet

Checking Permission

// Check current permission state console.log(Notification.permission); // Returns: 'granted', 'denied', or 'default' if (Notification.permission === 'granted') { console.log('We have permission!'); } else if (Notification.permission === 'denied') { console.log('Permission denied'); } else { console.log('Need to request permission'); }

Requesting Permission

// Request permission (Promise-based) async function requestNotificationPermission() { if (!('Notification' in window)) { console.error('This browser does not support notifications'); return false; } const permission = await Notification.requestPermission(); if (permission === 'granted') { console.log('Permission granted!'); return true; } else { console.log('Permission not granted'); return false; } } // Usage const hasPermission = await requestNotificationPermission(); if (hasPermission) { // Can show notifications } // Legacy callback style (still supported) Notification.requestPermission(function(permission) { if (permission === 'granted') { console.log('Permission granted'); } });
‚ö†Ô∏è Best Practices for Permission:
  • Don't ask for permission immediately on page load
  • Explain why you need notification permission
  • Ask at a relevant moment (e.g., when user enables notifications feature)
  • If denied, don't ask again (browsers block repeated requests)
  • Provide alternative notification methods if denied

üé¨ Notification Events

// Create notification with event listeners const notification = new Notification('Event Demo', { body: 'Click me to open the app!', icon: '/icon.png', data: { url: '/messages' } }); // Click event notification.onclick = (event) => { console.log('Notification clicked'); event.preventDefault(); // Prevent default behavior // Open or focus window window.focus(); // Navigate to specific page if (notification.data.url) { window.location.href = notification.data.url; } // Close the notification notification.close(); }; // Show event (notification displayed) notification.onshow = () => { console.log('Notification shown'); }; // Close event notification.onclose = () => { console.log('Notification closed'); }; // Error event notification.onerror = (error) => { console.error('Notification error:', error); };

Closing Notifications

// Close notification programmatically const notification = new Notification('Auto-close', { body: 'This will close in 3 seconds' }); // Close after 3 seconds setTimeout(() => { notification.close(); }, 3000);

üè∑Ô∏è Tag System (Grouping Notifications)

The tag option allows you to group and replace notifications. Multiple notifications with the same tag will replace each other instead of stacking.

// Without tag - creates multiple notifications new Notification('Message 1', { body: 'First message' }); new Notification('Message 2', { body: 'Second message' }); new Notification('Message 3', { body: 'Third message' }); // Result: 3 separate notifications // With tag - replaces previous notification new Notification('Messages', { body: 'You have 1 new message', tag: 'messages' }); new Notification('Messages', { body: 'You have 2 new messages', tag: 'messages' }); new Notification('Messages', { body: 'You have 3 new messages', tag: 'messages' }); // Result: Only 1 notification showing "3 new messages" // Use renotify to alert user on replacement new Notification('Messages', { body: 'You have 4 new messages', tag: 'messages', renotify: true // Vibrate/sound even though replacing });

üíº Practical Examples

1. Chat Application

class ChatNotifications { constructor() { this.messageCount = 0; } async init() { if (Notification.permission !== 'granted') { await Notification.requestPermission(); } } showMessage(sender, message) { if (Notification.permission !== 'granted') return; const notification = new Notification(`${sender}`, { body: message, icon: '/user-icon.png', tag: `chat-${sender}`, requireInteraction: true }); notification.onclick = () => { window.focus(); // Open chat with sender openChat(sender); notification.close(); }; } showUnreadCount(count) { if (count === 0) return; new Notification('Unread Messages', { body: `You have ${count} unread message(s)`, tag: 'unread-count', icon: '/messages-icon.png' }); } }

2. Reminder System

function scheduleReminder(title, message, delayMs) { setTimeout(() => { if (Notification.permission === 'granted') { const notification = new Notification(title, { body: message, icon: '/reminder-icon.png', tag: 'reminder', requireInteraction: true, vibrate: [200, 100, 200] }); notification.onclick = () => { window.focus(); notification.close(); }; } }, delayMs); } // Schedule reminder in 1 minute scheduleReminder( 'Meeting Reminder', 'Your meeting starts in 1 minute', 60000 );

3. Progress Notifications

class ProgressNotifier { constructor(taskName) { this.taskName = taskName; this.notification = null; } start() { if (Notification.permission === 'granted') { this.notification = new Notification(this.taskName, { body: 'Starting...', tag: `progress-${this.taskName}`, icon: '/loading-icon.png' }); } } update(progress) { if (Notification.permission === 'granted') { this.notification = new Notification(this.taskName, { body: `${progress}% complete`, tag: `progress-${this.taskName}`, icon: '/loading-icon.png', silent: true }); } } complete() { if (this.notification) { this.notification.close(); } new Notification(this.taskName, { body: 'Completed successfully!', tag: `progress-${this.taskName}`, icon: '/success-icon.png', vibrate: [200] }); } } // Usage const uploader = new ProgressNotifier('File Upload'); uploader.start(); uploader.update(25); uploader.update(50); uploader.update(75); uploader.complete();

‚úÖ Best Practices

DO ‚úì

  • Ask for permission at relevant moments
  • Explain why you need notification permission
  • Use meaningful titles and bodies
  • Use tags to group related notifications
  • Close notifications when clicked
  • Handle click events appropriately
  • Test on multiple browsers/devices
  • Respect "Do Not Disturb" settings

DON'T ‚úó

  • Spam users with notifications
  • Ask for permission on page load
  • Show notifications without permission
  • Use misleading notification content
  • Forget to handle denied permission
  • Show too many notifications at once
  • Use notifications for non-important info
