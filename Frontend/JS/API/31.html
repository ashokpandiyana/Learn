Background JavaScript Threads

üéØ What are Web Workers?

Web Workers enable running JavaScript in background threads, separate from the main execution thread. This prevents heavy computations from blocking the UI.

Key Features:

  • üßµ Separate Thread: Runs independently
  • ‚ö° Non-Blocking: Doesn't freeze UI
  • üì® Message-Based: Communicate via messages
  • üîí No DOM Access: Cannot manipulate DOM
  • üéØ Dedicated/Shared: Per-page or cross-tab
  • üí™ CPU-Intensive: Perfect for heavy computation

Main Thread vs Worker Thread

  • Game physics calculations
  • Real-time data processing

üöÄ Creating Workers

Main Thread (Creating Worker)

// main.js - Create worker const worker = new Worker('worker.js'); // Send message to worker worker.postMessage({ type: 'start', data: [1, 2, 3] }); // Receive messages from worker worker.onmessage = (event) => { console.log('Result from worker:', event.data); }; // Handle errors worker.onerror = (error) => { console.error('Worker error:', error.message); console.error('File:', error.filename); console.error('Line:', error.lineno); }; // Terminate worker when done worker.terminate();

Worker Thread (worker.js)

// worker.js - Worker code self.onmessage = (event) => { const { type, data } = event.data; if (type === 'start') { // Perform heavy computation const result = heavyComputation(data); // Send result back to main thread self.postMessage(result); } }; function heavyComputation(data) { // CPU-intensive work here let result = 0; for (let i = 0; i < 1000000000; i++) { result += i; } return result; } // Worker can terminate itself self.close();

üì¶ Transferable Objects

Transferable Objects allow transferring ownership of data to the worker instead of copying it. This is much faster for large data.

// Create ArrayBuffer const buffer = new ArrayBuffer(1024 * 1024); // 1MB // Transfer ownership to worker (faster than copying) worker.postMessage({ buffer }, [buffer]); // buffer is now unusable in main thread! console.log(buffer.byteLength); // 0 (transferred away) // Worker receives and owns the buffer // worker.js self.onmessage = (e) => { const buffer = e.data.buffer; console.log(buffer.byteLength); // 1048576 (1MB) // Process buffer... // Transfer back if needed self.postMessage({ buffer }, [buffer]); };
‚ö†Ô∏è Transferable Types: Only certain types can be transferred: ArrayBuffer, MessagePort, ImageBitmap, OffscreenCanvas. Regular objects are copied, not transferred.

üíº Practical Examples

1. Prime Number Calculator

// main.js const primeWorker = new Worker('prime-worker.js'); function calculatePrimes(limit) { primeWorker.postMessage({ limit }); primeWorker.onmessage = (e) => { console.log(`Found ${e.data.count} primes`); console.log('Primes:', e.data.primes); }; } calculatePrimes(100000); // prime-worker.js self.onmessage = (e) => { const { limit } = e.data; const primes = []; for (let num = 2; num <= limit; num++) { let isPrime = true; for (let i = 2; i <= Math.sqrt(num); i++) { if (num % i === 0) { isPrime = false; break; } } if (isPrime) primes.push(num); } self.postMessage({ primes, count: primes.length }); };

2. Image Processing

// main.js const imageWorker = new Worker('image-worker.js'); function processImage(imageData) { imageWorker.postMessage({ imageData, filter: 'grayscale' }); imageWorker.onmessage = (e) => { const processedData = e.data.imageData; ctx.putImageData(processedData, 0, 0); }; } // image-worker.js self.onmessage = (e) => { const { imageData, filter } = e.data; const pixels = imageData.data; if (filter === 'grayscale') { for (let i = 0; i < pixels.length; i += 4) { const avg = (pixels[i] + pixels[i+1] + pixels[i+2]) / 3; pixels[i] = pixels[i+1] = pixels[i+2] = avg; } } self.postMessage({ imageData }); };

üîß Inline Workers

Create workers from strings instead of separate files using Blob URLs.

// Create worker from code string const workerCode = ` self.onmessage = (e) => { const result = e.data * 2; self.postMessage(result); }; `; const blob = new Blob([workerCode], { type: 'application/javascript' }); const workerUrl = URL.createObjectURL(blob); const worker = new Worker(workerUrl); worker.postMessage(5); worker.onmessage = (e) => { console.log('Result:', e.data); // 10 }; // Clean up URL.revokeObjectURL(workerUrl);

‚úÖ Best Practices

DO ‚úì

  • Use for CPU-intensive tasks
  • Terminate workers when done
  • Handle errors in both threads
  • Use transferables for large data
  • Keep worker code simple

DON'T ‚úó

  • Access DOM from workers (impossible)
  • Create too many workers
  • Use for simple tasks
  • Forget to terminate workers
