Chapter 5: Attributes & Properties

Understanding the Crucial Difference Between HTML Attributes and DOM Properties

This chapter covers one of the most confusing topics in DOM manipulation. Many developers use "attribute" and "property" interchangeably, but they're fundamentally different concepts. Understanding this distinction is crucial for interviews and bug-free code.

5.1 Attributes vs Properties â€” The Core Difference

HTML Attributes
  • Defined in HTML markup
  • Values are always strings
  • Accessed via getAttribute()
  • Represent the initial value
  • Visible in HTML source
vs
DOM Properties
  • Defined on JavaScript objects
  • Values can be any type
  • Accessed via element.property
  • Represent the current value
  • May differ from HTML

How Attributes Become Properties

// HTML: <input id="username" type="text" value="hello">

const input = document.getElementById('username');

// Initially, attribute and property are the same
console.log(input.getAttribute('value'));  // "hello" (attribute)
console.log(input.value);                  // "hello" (property)

// User types "world" in the input field...

// Now they differ!
console.log(input.getAttribute('value'));  // "hello" (still initial!)
console.log(input.value);                  // "world" (current value!)

// The attribute is the DEFAULT value
// The property is the CURRENT value
The Key Insight:
  • Attribute = What's written in HTML (the default/initial value)
  • Property = What's in the DOM object (the current/live value)
For most cases, use properties. Use attributes when you need the initial HTML value or for non-standard attributes.

5.3 Data Attributes (dataset)

Data attributes (data-*) let you store custom data on HTML elements. They're perfect for passing data to JavaScript without using hidden inputs or global variables.

// HTML:
// <div id="user" 
//      data-user-id="123"
//      data-role="admin"
//      data-is-active="true"
//      data-lastLoginDate="2024-01-15">
// </div>

const user = document.getElementById('user');

// â”â”â”â”â”â”â”â”â”â” Reading data attributes â”â”â”â”â”â”â”â”â”â”
// data-user-id â†’ dataset.userId (camelCase conversion!)
user.dataset.userId;         // "123"
user.dataset.role;           // "admin"
user.dataset.isActive;       // "true" (string, not boolean!)
user.dataset.lastlogindate;  // "2024-01-15"

// â”â”â”â”â”â”â”â”â”â” Setting data attributes â”â”â”â”â”â”â”â”â”â”
user.dataset.newProp = 'value';    // Creates data-new-prop
user.dataset.userId = '456';      // Updates data-user-id

// â”â”â”â”â”â”â”â”â”â” Deleting data attributes â”â”â”â”â”â”â”â”â”â”
delete user.dataset.role;           // Removes data-role

// â”â”â”â”â”â”â”â”â”â” Check existence â”â”â”â”â”â”â”â”â”â”
'userId' in user.dataset;          // true

// â”â”â”â”â”â”â”â”â”â” Alternative: getAttribute â”â”â”â”â”â”â”â”â”â”
user.getAttribute('data-user-id');  // "123"
Dataset values are ALWAYS strings! 
// HTML: <div data-count="42" data-active="true">

typeof element.dataset.count;   // "string" (not number!)
typeof element.dataset.active;  // "string" (not boolean!)

// You must parse them yourself:
const count = parseInt(element.dataset.count, 10);
const active = element.dataset.active === 'true';

// Or for JSON data:
// <div data-config='{"limit": 10, "enabled": true}'>
const config = JSON.parse(element.dataset.config);

5.4 Class Manipulation (classList)

The classList property provides a powerful API for working with CSS classes. It's much better than manipulating the className string directly.

// HTML: <div id="box" class="card active highlighted">

const box = document.getElementById('box');

// â”â”â”â”â”â”â”â”â”â” className (old way - avoid!) â”â”â”â”â”â”â”â”â”â”
box.className;                    // "card active highlighted"
box.className = 'new-class';      // REPLACES all classes!
box.className += ' another';     // Append (but awkward)

// â”â”â”â”â”â”â”â”â”â” classList (modern - recommended!) â”â”â”â”â”â”â”â”â”â”

// ADD classes
box.classList.add('new-class');
box.classList.add('class1', 'class2', 'class3');  // Multiple at once!

// REMOVE classes
box.classList.remove('active');
box.classList.remove('class1', 'class2');  // Multiple!

// TOGGLE classes
box.classList.toggle('active');           // Returns new state (boolean)
box.classList.toggle('active', true);     // Force ADD
box.classList.toggle('active', false);    // Force REMOVE
box.classList.toggle('dark', isDarkMode);  // Conditional

// CHECK if class exists
box.classList.contains('card');          // true
box.classList.contains('nonexistent');   // false

// REPLACE one class with another
box.classList.replace('old-class', 'new-class');

// Other useful properties/methods
box.classList.length;                    // Number of classes
box.classList.item(0);                   // First class name
box.classList.value;                     // Same as className
[...box.classList];                     // Convert to array

Practical classList Examples:

// Toggle menu open/closed
menuButton.addEventListener('click', () => {
    menu.classList.toggle('open');
});

// Conditional class based on state
element.classList.toggle('error', !isValid);
element.classList.toggle('loading', isLoading);

// Remove all state classes, then add the right one
button.classList.remove('pending', 'success', 'error');
button.classList.add(currentState);

// Check before action
if (!card.classList.contains('flipped')) {
    card.classList.add('flipped');
}

5.5 Style Manipulation

You can manipulate element styles directly via JavaScript. Remember: element.style only accesses inline styles, not styles from CSS files!

const box = document.getElementById('box');

// â”â”â”â”â”â”â”â”â”â” Setting inline styles â”â”â”â”â”â”â”â”â”â”
// CSS property names become camelCase in JS!
box.style.backgroundColor = 'red';      // background-color
box.style.fontSize = '20px';            // font-size
box.style.marginTop = '10px';           // margin-top
box.style.border = '1px solid black';   // shorthand works
box.style.display = 'none';             // hide element
box.style.display = '';                 // remove inline, use CSS

// â”â”â”â”â”â”â”â”â”â” Set multiple styles at once â”â”â”â”â”â”â”â”â”â”
box.style.cssText = 'color: red; font-size: 20px;';
// âš ï¸ This REPLACES all inline styles!

// Or use Object.assign
Object.assign(box.style, {
    color: 'blue',
    fontSize: '18px',
    padding: '10px'
});

// â”â”â”â”â”â”â”â”â”â” CSS Custom Properties (Variables) â”â”â”â”â”â”â”â”â”â”
box.style.setProperty('--main-color', 'blue');
box.style.setProperty('--spacing', '20px');

// â”â”â”â”â”â”â”â”â”â” Reading inline styles â”â”â”â”â”â”â”â”â”â”
box.style.backgroundColor;               // Only inline styles!
box.style.getPropertyValue('color');

// â”â”â”â”â”â”â”â”â”â” Removing inline styles â”â”â”â”â”â”â”â”â”â”
box.style.backgroundColor = '';          // Remove single
box.style.removeProperty('color');
box.style.cssText = '';                   // Remove all
box.removeAttribute('style');            // Remove style attribute

Getting Computed Styles:

// element.style only shows INLINE styles!
// To get the ACTUAL applied styles, use getComputedStyle:

const box = document.getElementById('box');

// Get all computed styles
const styles = getComputedStyle(box);

// Read computed values
styles.backgroundColor;              // "rgb(255, 0, 0)"
styles.fontSize;                     // "16px" (computed px value)
styles.display;                      // "block"
styles.getPropertyValue('margin-top');

// Get pseudo-element styles
const beforeStyles = getComputedStyle(box, '::before');
beforeStyles.content;

// âš ï¸ getComputedStyle is READ-ONLY!
styles.color = 'blue';  // Does nothing! Use element.style instead
element.style vs getComputedStyle:
  • element.style â€” Read/write inline styles only
  • getComputedStyle() â€” Read-only, shows final computed values from all sources (CSS files, inline, inherited)

5.6 Boolean Attributes

Some attributes are "boolean" â€” their presence alone means true, absence means false.

// HTML: <input type="checkbox" checked disabled readonly>

const checkbox = document.querySelector('input');

// â”â”â”â”â”â”â”â”â”â” PROPERTIES (preferred for boolean attrs) â”â”â”â”â”â”â”â”â”â”
checkbox.checked;     // true (boolean)
checkbox.disabled;    // true (boolean)
checkbox.readOnly;    // true (boolean)

// Setting via property
checkbox.checked = false;
checkbox.disabled = true;

// â”â”â”â”â”â”â”â”â”â” ATTRIBUTES (different behavior!) â”â”â”â”â”â”â”â”â”â”
checkbox.getAttribute('checked');   // "" (empty string, not true!)
checkbox.getAttribute('disabled');  // "" (empty string)

// For boolean attrs, presence = true, absence = false
checkbox.hasAttribute('checked');   // true
checkbox.hasAttribute('disabled');  // true

// Setting via attribute
checkbox.setAttribute('disabled', '');     // Enable disabled
checkbox.removeAttribute('disabled');      // Disable

// toggleAttribute is perfect for boolean attributes!
checkbox.toggleAttribute('disabled');      // Toggle on/off
checkbox.toggleAttribute('disabled', true); // Force enable
Don't use string 'false' for boolean attributes! 
// âŒ WRONG - This still DISABLES the element!
checkbox.setAttribute('disabled', 'false');
// The presence of the attribute (any value) = disabled

// âœ… CORRECT ways to enable:
checkbox.disabled = false;           // Use property
checkbox.removeAttribute('disabled'); // Remove attribute
