üéØ What is the Template Element?

The <template> element is a mechanism for holding HTML that is not to be rendered immediately when a page loads, but can be instantiated later using JavaScript. It's essentially a blueprint for creating DOM content.

üí° Key Characteristics

  • Inert: Content inside <template> is not rendered
  • No side effects: Scripts don't run, images don't load, styles don't apply
  • Clonable: Can be cloned multiple times efficiently
  • Valid HTML: Can contain any valid HTML, even elements that would normally require specific parents

üìù Basic Template Usage

<!-- Define the template in HTML -->
<template id="card-template">
    <div class="card">
        <img src="" alt="">
        <h3 class="title"></h3>
        <p class="description"></p>
    </div>
</template>

<!-- Container for cards -->
<div id="card-container"></div>
// Get the template
const template = document.getElementById('card-template');

// Clone the template content
const clone = template.content.cloneNode(true);

// Modify the clone
clone.querySelector('.title').textContent = 'My Card Title';
clone.querySelector('.description').textContent = 'Card description here';
clone.querySelector('img').src = 'image.jpg';

// Append to DOM
document.getElementById('card-container').appendChild(clone);

üìä Template vs innerHTML Performance

‚úÖ Using Template

  • Pre-parsed once by browser
  • Cloning is very fast
  • No HTML parsing on each use
  • Type-safe DOM manipulation

‚ùå Using innerHTML

  • HTML parsed every time
  • XSS vulnerability risk
  • Destroys existing event listeners
  • String concatenation mess
// ‚ùå Slow and unsafe
for (const item of items) {
    container.innerHTML += `
        <div class="card">
            <h3>${item.title}</h3>  <!-- XSS risk! -->
        </div>
    `;  // Re-parses ALL cards every iteration!
}

// ‚úÖ Fast and safe
const template = document.getElementById('card-template');
const fragment = document.createDocumentFragment();

for (const item of items) {
    const clone = template.content.cloneNode(true);
    clone.querySelector('h3').textContent = item.title;  // Safe!
    fragment.appendChild(clone);
}

container.appendChild(fragment);  // Single DOM update!

üîå Slot Element: Content Projection

The <slot> element is used within Shadow DOM to create placeholders where external content (from the "Light DOM") can be inserted.

üè† Host Element (Light DOM)
<my-card>
<span slot="title">Hello!</span>
<p>Default slot content</p>
</my-card>
‚Üì
üåë Shadow DOM
<header>
<slot name="title">
Named slot receives: "Hello!"
</header> <main>
<slot>
Default slot receives: <p> content
</main>

üìù Slot Types

Slot Type Syntax Description
Default Slot <slot></slot> Receives all content without a slot attribute
Named Slot <slot name="header"></slot> Receives content with slot="header"
Fallback Content <slot>Default</slot> Shows if no content is provided

class InfoCard extends HTMLElement {
    constructor() {
        super();
        this.attachShadow({ mode: 'open' });
        this.shadowRoot.innerHTML = `
            <style>
                :host {
                    display: block;
                    border: 2px solid #06b6d4;
                    border-radius: 12px;
                    overflow: hidden;
                }
                .header {
                    background: linear-gradient(135deg, #06b6d4, #8b5cf6);
                    padding: 15px;
                    color: white;
                }
                .body {
                    padding: 20px;
                }
                .footer {
                    background: #f1f5f9;
                    padding: 10px 15px;
                    font-size: 0.9em;
                    color: #64748b;
                }
                
                /* Style slotted content */
                ::slotted(h2) {
                    margin: 0;
                    font-size: 1.5rem;
                }
                ::slotted(p) {
                    margin: 10px 0 0;
                }
            </style>
            
            <div class="header">
                <slot name="header">
                    <h2>Default Header</h2>
                </slot>
            </div>
            
            <div class="body">
                <slot>
                    <p>No content provided</p>
                </slot>
            </div>
            
            <div class="footer">
                <slot name="footer">
                    Default footer
                </slot>
            </div>
        `;
    }
}

customElements.define('info-card', InfoCard);

üîç Slot API Methods

// Get assigned nodes for a slot
const slot = shadowRoot.querySelector('slot[name="header"]');
const assignedNodes = slot.assignedNodes();      // All nodes
const assignedElements = slot.assignedElements(); // Elements only

// Include fallback content
const withFallback = slot.assignedNodes({ flatten: true });

// Get the slot an element is assigned to
const element = document.querySelector('[slot="header"]');
element.assignedSlot;  // The slot element

// Listen for slot changes
slot.addEventListener('slotchange', (e) => {
    console.log('Slot content changed!');
    console.log('New nodes:', e.target.assignedNodes());
});

üéØ Template + Custom Element Pattern

// Define template in HTML or JavaScript
const template = document.createElement('template');
template.innerHTML = `

        .container { padding: 20px; border: 1px solid #ccc; }
        .title { color: #06b6d4; }

class MyWidget extends HTMLElement {
    constructor() {
        super();
        const shadow = this.attachShadow({ mode: 'open' });
        
        // Use the template - efficient for many instances!
        shadow.appendChild(template.content.cloneNode(true));
    }
}

customElements.define('my-widget', MyWidget);

üìã Best Practices

‚úÖ Template Best Practices

  • Use templates for repeated content: Cards, list items, table rows
  • Define templates in HTML when possible: Better for large templates
  • Use DocumentFragment for batch inserts: Collect clones, then append once
  • Clone with cloneNode(true): The true means deep clone
  • Modify clones before inserting: Fill in data, set attributes

üìå Slot Best Practices

  • Provide meaningful fallback content: Users see something if they forget to pass content
  • Use named slots for specific areas: header, footer, actions, icon
  • Use default slot for main content: Most flexible approach
  • Style slotted content with ::slotted(): Remember it only selects direct children
  • Listen for slotchange: React when slotted content changes
