5.1
Network Panel Overview

üåê What the Network Panel Shows

The Network panel records ALL HTTP requests made by the page, including:

  • Documents - HTML pages, iframes
  • Stylesheets - CSS files
  • Scripts - JavaScript files
  • XHR/Fetch - AJAX/API calls
  • Fonts - Web fonts
  • Images - All image formats
  • Media - Video, audio files
  • WebSocket - Real-time connections
  • Manifest - PWA manifest files
  • Other - Source maps, etc.
The Network panel only records while it's open! Open DevTools BEFORE loading the page to capture all requests.

Right-click column headers to show/hide columns. Add "Priority" to see resource loading priority!

Filtering Requests

üîç Filter Bar Basics

The filter bar at the top lets you narrow down requests. Simply type to filter by name/URL.

Type Filters

Click the buttons to filter by resource type:

All Fetch/XHR JS CSS Img Media Font Doc WS Manifest

üéØ Advanced Filter Syntax

Use special operators for powerful filtering:

Domain Filtering

domain:api.example.com

Shows only requests to specific domain.

Status Code Filtering

status-code:404
status-code:500

Filter by specific HTTP status.

Negative Filtering

-status-code:200
-domain:google.com

Exclude matching requests (show errors only).

Size Filtering

larger-than:100k
larger-than:1M

Find large resources causing slow loads.

Method Filtering

method:POST
method:PUT

Filter by HTTP method.

Mixed Content

mixed-content:displayed
mixed-content:blocked

Find HTTP resources on HTTPS pages.

Has Response Header

has-response-header:cache-control

Filter by presence of specific headers.

Regex Filtering

/api\/v[0-9]+\//

Use regex patterns for complex matching.

Combine filters: domain:api.example.com status-code:500 shows only API errors!

‚òëÔ∏è Filter Checkboxes

  • Invert - Show everything EXCEPT matching requests
  • Hide data URLs - Remove inline base64 data
  • Hide extension URLs - Remove browser extension requests
  • Blocked response cookies - Show requests with cookie issues
  • Blocked requests - Show ad-blocked/filtered requests
  • 3rd-party requests - Show only cross-origin requests

üìä Timing Phases Explained

Phase What It Measures Optimization Tips
Queueing Time waiting in browser queue Browser limits 6 connections per host. Use HTTP/2 or CDN.
Stalled/Blocking Time before request could be sent Reduce number of requests, prioritize critical resources.
DNS Lookup Resolving domain to IP address Use DNS prefetching: <link rel="dns-prefetch">
Initial Connection TCP handshake (SYN, SYN-ACK, ACK) Use preconnect: <link rel="preconnect">
SSL/TLS TLS negotiation for HTTPS Enable TLS 1.3, use session resumption.
Request Sent Uploading request data Reduce payload size, compress request body.
Waiting (TTFB) Time To First Byte - server processing Server optimization, caching, CDN edge caching.
Content Download Receiving response body Enable compression (gzip/brotli), reduce response size.
TTFB is a critical performance metric! High TTFB indicates server-side issues. Interviewers often ask about this!

What to Look For:

  • Long bars - Slow resources blocking others
  • Gaps - Browser waiting (connection limits or dependencies)
  • Cascade pattern - Resources loading in sequence (not parallel)
  • Green (TTFB) dominates - Server is slow
  • Blue (Download) dominates - Resource is too large

5.6
Request Blocking & Overriding

üö´ Blocking Requests

Block specific resources to test how your page behaves without them:

Method 1: Right-Click Block

  • Right-click any request
  • Select "Block request URL" or "Block request domain"
  • Reload page to see effect

Method 2: Request Blocking Panel

  • Open Command Menu (Ctrl+Shift+P)
  • Type "Show Request Blocking"
  • Add URL patterns to block
  • Supports wildcards: *.analytics.com
// Block patterns examples: *google-analytics* // Block analytics *.doubleclick.net // Block ads */api/tracking/* // Block tracking API *facebook* // Block Facebook widgets *.woff2 // Block all web fonts
Block third-party scripts to measure their performance impact on your page!

‚úèÔ∏è Request Overrides (Local Overrides)

Replace server responses with local files - perfect for testing changes on production!

Setup Steps:

  1. Go to Sources panel ‚Üí Overrides tab
  2. Click "Select folder for overrides"
  3. Choose/create a local folder
  4. Grant DevTools access
  5. Enable "Enable Local Overrides" checkbox

Creating an Override:

  1. Find request in Network panel
  2. Right-click ‚Üí "Override content"
  3. Edit the file in Sources panel
  4. Save (Ctrl+S)
  5. Reload - your version is served!
Overrides persist across browser sessions! Remember to disable when done debugging.
