Console API Methods

üìù What is the Console API?

The Console API provides methods for logging information to the browser's console. While most developers only use console.log(), the API offers many powerful methods for better debugging.

üìä Basic Logging Methods

console.log()

General purpose logging

console.log("Hello", variable)
console.info()

Informational messages (‚ÑπÔ∏è icon)

console.info("Process started")
console.warn()

Warning messages (‚ö†Ô∏è yellow)

console.warn("Deprecated!")
console.error()

Error messages (‚ùå red)

console.error("Failed!")
console.debug()

Debug level (hidden by default)

console.debug("Debug info")
console.clear()

Clear all console output

console.clear()
üìù This is console.log()
‚ÑπÔ∏è This is console.info()
‚ö†Ô∏è This is console.warn()
‚ùå This is console.error()
Basic Logging Examples
// Multiple arguments console.log("User:", userName, "Age:", userAge); // Object logging (maintains structure) console.log({ name: "John", age: 30 }); // Array logging console.log([1, 2, 3, 4, 5]); // Template literals (ES6) console.log(`User ${name} is ${age} years old`);

üî• Advanced Logging Methods

console.table() - Tabular Data Display
const users = [ { name: "Alice", age: 28, city: "NYC" }, { name: "Bob", age: 32, city: "LA" }, { name: "Carol", age: 25, city: "Chicago" } ]; console.table(users); // Displays as formatted table! // Show only specific columns console.table(users, ["name", "city"]);

0"Alice"28"NYC"
1"Bob"32"LA"
2"Carol"25"Chicago"
console.table() is a game-changer for debugging arrays of objects! It's much easier to read than standard log output.
console.dir() & console.dirxml()
// console.dir() - Display object with properties const element = document.querySelector('#app'); console.log(element); // Shows HTML representation console.dir(element); // Shows JavaScript object properties // console.dirxml() - Display XML/HTML tree console.dirxml(document.body);
console.group() - Organize Messages
console.group("User Authentication"); console.log("Checking credentials..."); console.log("Validating token..."); console.group("Permissions"); console.log("Admin: true"); console.log("Editor: false"); console.groupEnd(); console.log("Authentication complete!"); console.groupEnd(); // Use groupCollapsed() for collapsed groups by default console.groupCollapsed("Collapsed by default");
console.count() - Count Occurrences
function processItem(item) { console.count("processItem called"); // ... processing logic } processItem("a"); // processItem called: 1 processItem("b"); // processItem called: 2 processItem("c"); // processItem called: 3 // Reset counter console.countReset("processItem called"); processItem("d"); // processItem called: 1
console.table(), console.group(), and console.count() are frequently asked in interviews as "advanced console methods you know"!
3.2

Timing & Profiling

‚è±Ô∏è Measuring Execution Time

The Console API provides built-in timing methods to measure how long operations take - essential for performance optimization.

console.time() / timeEnd() / timeLog()
// Start a timer console.time("API Call"); await fetch("/api/data"); // Log intermediate time (without stopping) console.timeLog("API Call", "Data received"); // Output: API Call: 234.5ms Data received await processData(); // Stop timer and log final time console.timeEnd("API Call"); // Output: API Call: 456.7ms
console.trace() - Stack Trace
function firstFunction() { secondFunction(); } function secondFunction() { thirdFunction(); } function thirdFunction() { console.trace("Trace from thirdFunction"); } firstFunction(); /* Output: Trace from thirdFunction at thirdFunction (script.js:10) at secondFunction (script.js:6) at firstFunction (script.js:2) at script.js:13 */
Use console.trace() when you need to find "how did we get here?" - it shows the complete call stack leading to that point.
Performance Profiling
// Start CPU profile console.profile("MyProfile"); // ... code to profile ... for (let i = 0; i < 1000000; i++) { heavyComputation(); } // Stop profile - results appear in Performance tab console.profileEnd("MyProfile"); // Alternative: Use performance.mark() for more precision performance.mark("start"); // ... code ... performance.mark("end"); performance.measure("My Operation", "start", "end");
3.3

Console Utilities API

üõ†Ô∏è Built-in Console Utilities

DevTools provides special utility functions that only work in the Console. These are NOT part of JavaScript - they're DevTools features!

üîç Selection Helpers

Utility Equivalent To Description
$(selector) document.querySelector() Select first matching element
$$(selector) document.querySelectorAll() Select all matching elements (array)
$x(xpath) XPath query Select elements by XPath
$0 to $4 - Recently selected elements
$_ - Last evaluated expression result
Selection Utilities Examples
// Select single element $('.header') // First .header element $('#main-nav') // Element with id="main-nav" // Select all matching elements $$('.button') // All elements with class .button $$('div > p') // All p elements direct child of div // XPath selection $x('//div[@class="container"]') $x('//a[contains(@href, "example")]') // Use $_ to chain operations $('.price').textContent // Returns "$19.99" $_.replace('$', '') // Returns "19.99" parseFloat($_) // Returns 19.99
$(selector) is NOT jQuery! It's a DevTools shortcut. If jQuery is loaded on the page, it will override this utility.

üîß Inspection Helpers

Inspection Utilities
// inspect() - Jump to element or source inspect($('.header')) // Opens in Elements panel inspect(myFunction) // Opens function in Sources panel // getEventListeners() - Get all listeners getEventListeners($('#btn')) // Returns: { click: [...], mouseover: [...] } // keys() and values() - Object inspection const obj = { a: 1, b: 2, c: 3 }; keys(obj) // ["a", "b", "c"] values(obj) // [1, 2, 3] // copy() - Copy to clipboard copy($('.content').outerHTML) copy(JSON.stringify(data, null, 2)) // clear() - Clear console clear()
getEventListeners() is one of the most asked DevTools utilities in interviews! Know how to use it to debug event-related issues.

üëÄ Monitoring Functions

Function & Event Monitoring
// monitor() - Log function calls function calculateTotal(a, b) { return a + b; } monitor(calculateTotal); calculateTotal(5, 3); // Output: function calculateTotal called with arguments: 5, 3 unmonitor(calculateTotal); // Stop monitoring // monitorEvents() - Log DOM events monitorEvents($('#input'), 'focus'); monitorEvents($('#btn'), ['click', 'mouseover']); monitorEvents(window, 'resize'); // Event type shortcuts monitorEvents($('#input'), 'key'); // All keyboard events monitorEvents($('#div'), 'mouse'); // All mouse events unmonitorEvents($('#input')); // Stop monitoring
debug() - Add Breakpoints Programmatically
// Break when function is called debug(myFunction); // Now any call to myFunction() will pause execution myFunction(); // Debugger pauses here! // Remove breakpoint undebug(myFunction);
monitorEvents() is perfect for debugging "why isn't my click handler working?" - it shows exactly which events fire and when!

3.5

Live Expressions

üëÅÔ∏è What are Live Expressions?

Live Expressions let you watch the value of a JavaScript expression in real-time. The expression is evaluated continuously and the result updates automatically - perfect for monitoring changing values without cluttering the console with logs.

üìç How to Create a Live Expression

  1. Click the "eye" icon (üëÅÔ∏è) in the Console toolbar
  2. Type your JavaScript expression
  3. Press Enter - it will now update in real-time
  4. Click the X to remove the expression
Useful Live Expression Examples
// Watch which element is focused document.activeElement // Monitor scroll position window.scrollY // Watch viewport dimensions `${window.innerWidth} x ${window.innerHeight}` // Count elements in real-time document.querySelectorAll('.item').length // Watch performance performance.now().toFixed(0) // Watch localStorage localStorage.getItem('token') // Watch a specific variable (if global) myAppState.currentUser
Live Expressions are especially useful for debugging focus management, scroll behaviors, and dynamic content changes!

Console Assertions

‚úÖ console.assert()

Assertions only log a message when a condition is FALSE. This is useful for catching unexpected states without cluttering the console when everything is working correctly.

console.assert() Examples
// Basic assertion const age = 25; console.assert(age >= 18, "User must be adult"); // No output (condition is true) console.assert(age >= 30, "User must be 30+"); // Outputs: Assertion failed: User must be 30+ // Assert array not empty const items = []; console.assert(items.length > 0, "Items array is empty!", items); // Assert object has required property const user = { name: "John" }; console.assert(user.email, "User missing email!", user); // Use in development for sanity checks function processOrder(order) { console.assert(order.id, "Order missing ID"); console.assert(order.items.length, "Order has no items"); console.assert(order.total > 0, "Order total invalid"); // ... process order }

3.7

Console Settings & Filtering

‚öôÔ∏è Console Settings

Click the gear icon in the Console panel or access through DevTools settings:

  • Preserve log - Keep logs across page navigation
  • Show timestamps - Display time for each message
  • Autocomplete from history - Suggest previous commands
  • Group similar - Collapse duplicate messages
  • Eager evaluation - Preview results while typing
  • Log XMLHttpRequests - Show XHR/fetch requests
  • Selected context only - Filter by iframe context

üîç Filtering Console Messages

  • Use the filter buttons: All, Errors, Warnings, Info, Verbose
  • Type in filter box to search message content
  • Use regex: /error|fail/i
  • Negative filter: -verbose (exclude "verbose")
  • URL filter: url:script.js
Enable "Preserve log" when debugging page transitions or form submissions - your logs won't disappear on navigation!
