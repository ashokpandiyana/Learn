Navigation Techniques

Console Panel Mastery Intermediate

3.1 Console API Methods

Basic Logging

  • console.log() - General output
  • console.info() - Informational
  • console.warn() - Warning (yellow)
  • console.error() - Error (red)
  • console.debug() - Debug level (hidden by default)

Advanced Logging

  • console.table(array/object) - Tabular display
  • console.dir(object) - Object properties
  • console.dirxml(node) - XML/HTML view
  • console.group() / groupEnd() - Group messages
  • console.groupCollapsed() - Collapsed group
  • console.count(label) - Count calls
  • console.countReset(label) - Reset counter

Timing & Profiling

  • console.time(label) - Start timer
  • console.timeEnd(label) - End timer & log
  • console.timeLog(label) - Log intermediate time
  • console.profile() / profileEnd() - CPU profile
  • console.trace() - Print stack trace
console.table() is a game-changer for debugging arrays of objects!

3.2 Console Utilities API

Selection Helpers

  • $(selector) - Alias for document.querySelector()
  • $$(selector) - Alias for document.querySelectorAll()
  • $x(xpath) - Evaluate XPath expression
  • $0 - $4 - Recently selected elements
  • $_ - Last evaluated expression result

Inspection Helpers

  • inspect(object/function) - Jump to element/source
  • getEventListeners(element) - Get all listeners
  • keys(object) - Object keys
  • values(object) - Object values
  • copy(object) - Copy to clipboard
  • clear() - Clear console

Monitoring Functions

  • monitor(function) - Log function calls
  • unmonitor(function) - Stop monitoring
  • monitorEvents(element, [events]) - Log events
  • unmonitorEvents(element) - Stop event monitoring
  • debug(function) - Break on function call
  • undebug(function) - Remove breakpoint
monitorEvents() and getEventListeners() are frequently asked in interviews!

3.3 String Formatting

Format Specifiers

  • %s - String
  • %d or %i - Integer
  • %f - Float
  • %o - Object (expandable)
  • %O - Object (non-expandable)
  • %c - CSS styling
Style console output: console.log('%cStyled!', 'color: red; font-size: 20px;')

3.4 Console Settings

Options

  • Preserve log - Keep logs across navigation
  • Show timestamps
  • Autocomplete from history
  • Group similar messages
  • Eager evaluation
  • Log XMLHttpRequests
  • Selected context only (for iframes)

3.5 Live Expressions

Real-time Watching

  • Click eye icon to add live expression
  • Expression updates in real-time
  • Great for watching variables/DOM state
  • Example: document.activeElement
  • Example: performance.now()

3.6 Console Assertions

Testing in Console

  • console.assert(condition, message)
  • Only logs if condition is false
  • Useful for runtime assertions
  • Example: console.assert(arr.length > 0, 'Array is empty!')

Know network filter syntax! Common interview question: "How do you filter for failed requests?"

Memory Panel & Leak Detection Advanced

7.1 Memory Overview

JavaScript Memory Concepts

  • Heap - Where objects are stored
  • Stack - Primitive values & references
  • Garbage Collection - Automatic memory management
  • Memory Leak - Unreleased memory that grows

7.2 Heap Snapshots

Taking Snapshots

  • Click "Take heap snapshot"
  • Shows all objects in memory
  • Compare multiple snapshots
  • Force garbage collection first (trash icon)

Views

  • Summary - Objects grouped by constructor
  • Comparison - Diff between snapshots
  • Containment - Object hierarchy
  • Statistics - Memory breakdown pie chart

Key Columns

  • Shallow Size - Object's own memory
  • Retained Size - Memory freed if object deleted
  • Distance - Distance from GC root
Retained Size is what matters for memory leaks - large retained sizes indicate problem areas!

7.3 Allocation Timeline

Features

  • Record memory allocations over time
  • Blue bars = allocated, gray = freed
  • Remaining blue bars = potential leaks
  • Click bar to see allocation stack
  • Filter by constructor name

7.4 Allocation Sampling

Low-Overhead Profiling

  • Statistical sampling approach
  • Lower overhead than timeline
  • Good for production profiling
  • Shows allocation call stacks

7.5 Common Memory Leak Patterns

What to Look For

  • Detached DOM trees - DOM nodes referenced but not in document
  • Event listeners - Not removed on cleanup
  • Closures - Capturing large objects
  • Global variables - Accidental globals
  • Timers - setInterval not cleared
  • Observers - Not disconnected
Search for "Detached" in heap snapshots to find detached DOM trees - a classic leak!

7.6 Finding Leaks Workflow

Step-by-Step Process

  • 1. Take baseline heap snapshot
  • 2. Perform suspected leaking action
  • 3. Force garbage collection
  • 4. Take another snapshot
  • 5. Compare snapshots for growth
  • 6. Look at "Objects allocated between"
  • 7. Trace retaining paths

7.7 Retainers & Paths

Understanding Retention

  • Retainers panel shows what keeps object alive
  • Follow path from object to GC root
  • Look for unexpected retainers
  • Yellow = direct reference
  • Red = indirect (through closure)

7.8 Performance Monitor

Real-time Metrics

  • Open from Command Menu or More tools
  • CPU usage
  • JS heap size
  • DOM nodes count
  • JS event listeners
  • Documents / Frames / Layouts per second
Watch DOM node count while navigating SPA - should stay stable or you have a leak!
