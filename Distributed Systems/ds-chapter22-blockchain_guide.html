1Blockchain Fundamentals

A blockchain is a distributed, immutable ledger that records transactions in a chain of blocks. Each block contains a cryptographic hash of the previous block, transaction data, and a timestamp, creating an unbreakable chain of information.

üîê Key Properties

  • Immutability: Once data is written, it cannot be altered without changing all subsequent blocks
  • Decentralization: No single point of control or failure
  • Transparency: All participants can view the transaction history
  • Security: Cryptographic hashing ensures data integrity

2Distributed Consensus in Blockchain

Distributed consensus is the mechanism by which nodes in a blockchain network agree on the current state of the ledger without requiring a central authority. This is one of the most critical challenges in distributed systems.

The Byzantine Generals Problem

Consensus mechanisms solve the Byzantine Generals Problem: How can distributed parties agree on a strategy when some parties may be malicious or unreliable?

‚ö†Ô∏è Consensus Requirements:
  • Agreement: All honest nodes agree on the same value
  • Validity: The agreed value was proposed by at least one honest node
  • Termination: All honest nodes eventually decide on a value
  • Integrity: No honest node decides twice

Common Consensus Mechanisms

1. Proof of Work (PoW)

Miners solve complex mathematical puzzles to validate transactions and create new blocks.

2. Proof of Stake (PoS)

Validators are chosen based on the amount of cryptocurrency they hold and are willing to "stake."

3. Practical Byzantine Fault Tolerance (PBFT)

Nodes reach consensus through multiple rounds of voting, tolerating up to 1/3 malicious nodes.

4. Delegated Proof of Stake (DPoS)

Token holders vote for a limited number of delegates who validate transactions.

3Proof of Work vs Proof of Stake

Proof of Work (PoW)

PoW requires miners to solve computationally intensive cryptographic puzzles. The first miner to solve the puzzle gets to add the next block and receives a reward.

Proof of Stake (PoS)

PoS selects validators based on their stake in the network. Validators lock up their cryptocurrency as collateral and are rewarded for honest behavior.

class ProofOfStake { constructor() { this.validators = new Map(); // validator ‚Üí stake amount } addValidator(address, stake) { this.validators.set(address, stake); } selectValidator() { // Calculate total stake let totalStake = 0; for (let stake of this.validators.values()) { totalStake += stake; } // Random selection weighted by stake let randomNum = Math.random() * totalStake; let cumulativeStake = 0; for (let [address, stake] of this.validators) { cumulativeStake += stake; if (randomNum <= cumulativeStake) { return address; } } } validateBlock(validator, block) { // Validator signs the block if (this.validators.has(validator)) { block.validator = validator; return true; } return false; } }

4Smart Contracts

Smart contracts are self-executing programs stored on a blockchain that automatically enforce the terms of an agreement when predetermined conditions are met. They eliminate the need for intermediaries.

How Smart Contracts Work

Traditional Contract vs Smart Contract

Traditional: Alice wants to buy Bob's house ‚Üí Lawyers draft contract ‚Üí Both sign ‚Üí Money transferred via bank ‚Üí Title transferred via government office ‚Üí Multiple intermediaries, days/weeks

Smart Contract: Alice sends payment to contract ‚Üí Contract verifies payment ‚Üí Contract automatically transfers ownership ‚Üí Complete in minutes ‚Üí No intermediaries

5Distributed Ledger Technology (DLT)

DLT is a broader category that includes blockchain but also encompasses other distributed database technologies. Not all DLTs use blocks or chains.

Types of DLT

1. Blockchain

Linear chain of blocks, each containing multiple transactions. Examples: Bitcoin, Ethereum

  • Sequential block structure
  • High security through mining/staking
  • Lower throughput

2. Directed Acyclic Graph (DAG)

Transactions directly validate other transactions in a graph structure. Examples: IOTA, Hedera Hashgraph

  • No blocks or miners
  • Higher throughput and scalability
  • Each transaction validates 2+ previous transactions

3. Holochain

Agent-centric DLT where each user has their own chain

  • No global consensus required
  • Extremely scalable
  • Lower energy consumption

6Cryptocurrency Systems

Cryptocurrencies are digital currencies that use cryptography for security and operate on blockchain networks. They enable peer-to-peer transactions without intermediaries.

How Cryptocurrency Transactions Work

Transaction Lifecycle

  1. Initiation: Alice wants to send 1 BTC to Bob
  2. Broadcasting: Transaction is broadcast to the network
  3. Validation: Nodes verify Alice has sufficient balance
  4. Mining/Validation: Transaction included in a block
  5. Confirmation: Block added to chain (typically 6 confirmations for Bitcoin)
  6. Completion: Bob receives 1 BTC

Cryptographic Components

const crypto = require('crypto'); class Wallet { constructor() { // Generate key pair const { privateKey, publicKey } = crypto.generateKeyPairSync('rsa', { modulusLength: 2048, publicKeyEncoding: { type: 'spki', format: 'pem' }, privateKeyEncoding: { type: 'pkcs8', format: 'pem' } }); this.privateKey = privateKey; this.publicKey = publicKey; // Address is hash of public key this.address = this.generateAddress(); } generateAddress() { return crypto.createHash('sha256') .update(this.publicKey) .digest('hex') .substring(0, 40); } signTransaction(transaction) { // Sign transaction with private key const sign = crypto.createSign('SHA256'); sign.update(JSON.stringify(transaction)).end(); return sign.sign(this.privateKey, 'hex'); } static verifySignature(publicKey, signature, transaction) { // Verify signature with public key const verify = crypto.createVerify('SHA256'); verify.update(JSON.stringify(transaction)); return verify.verify(publicKey, signature, 'hex'); } } class Transaction { constructor(fromAddress, toAddress, amount) { this.fromAddress = fromAddress; this.toAddress = toAddress; this.amount = amount; this.timestamp = Date.now(); } calculateHash() { return crypto.createHash('sha256') .update(this.fromAddress + this.toAddress + this.amount + this.timestamp) .digest('hex'); } signTransaction(signingKey) { if (signingKey.address !== this.fromAddress) { throw new Error('Cannot sign transactions for other wallets'); } this.signature = signingKey.signTransaction(this); } isValid() { // Mining rewards have no fromAddress if (this.fromAddress === null) return true; if (!this.signature || this.signature.length === 0) { throw new Error('No signature in this transaction'); } return Wallet.verifySignature( this.fromAddress, this.signature, this ); } }

Key Concepts

üîë UTXO Model (Bitcoin)

Unspent Transaction Output: Like cash denominations. If you have a $50 bill and buy something for $30, you give the $50 and receive $20 change.

  • Each transaction consumes previous UTXOs and creates new ones
  • No accounts, just transaction inputs and outputs
  • Better privacy (addresses can be one-time use)

üìä Account Model (Ethereum)

Like a bank account with a balance that increases/decreases with transactions.

  • Each address has a balance
  • Simpler for smart contracts
  • Easier to track account state

‚õèÔ∏è Mining Rewards & Halving

Bitcoin miners receive block rewards that halve every 210,000 blocks (~4 years):

  • 2009-2012: 50 BTC per block
  • 2012-2016: 25 BTC per block
  • 2016-2020: 12.5 BTC per block
  • 2020-2024: 6.25 BTC per block
  • 2024-2028: 3.125 BTC per block
üí° Scalability Challenges:
  • Bitcoin: ~7 transactions per second
  • Ethereum: ~15-30 transactions per second
  • Visa: ~24,000 transactions per second
  • Solutions: Layer 2 (Lightning Network, Rollups), Sharding, Sidechains

üéì Key Takeaways

  • Blockchain is an immutable, distributed ledger using cryptographic hashing and consensus mechanisms
  • Distributed Consensus solves the Byzantine Generals Problem, allowing trustless agreement
  • PoW vs PoS: Trade-offs between energy consumption, security, and decentralization
  • Smart Contracts automate agreements without intermediaries, enabling DeFi and dApps
  • DLT encompasses blockchain and other distributed database architectures (DAG, Holochain)
  • Cryptocurrencies enable peer-to-peer digital transactions with strong cryptographic security
