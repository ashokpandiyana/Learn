## 20.6 Common Pitfalls & How to Avoid

### Mistake 1: Rushing to Code

**‚ùå Wrong Approach:**
```
Read problem ‚Üí Start coding immediately
```

**‚úÖ Right Approach:**
```
1. Read carefully (2 min)
2. Work through examples (3 min)
3. Identify pattern (2 min)
4. Explain approach (3 min)
5. Code solution (20-40 min)
6. Test & optimize (5 min)
```

---

### Mistake 2: Not Practicing Edge Cases

**Critical Edge Cases Checklist:**
```
‚ñ° Empty input
‚ñ° Single element
‚ñ° All same elements
‚ñ° Sorted/reverse sorted
‚ñ° Duplicates
‚ñ° Negative numbers
‚ñ° Integer overflow
‚ñ° Null/None values
```

---

### Mistake 3: Neglecting Complexity Analysis

**Always State:**
```
Time Complexity: O(?)
- Best case: O(?)
- Average case: O(?)
- Worst case: O(?)

Space Complexity: O(?)
- Auxiliary space: O(?)
- Recursion stack: O(?)
- Total: O(?)
```

---

### Mistake 4: Not Reviewing Solutions

**After Each Problem:**
```
1. Review optimal solution
2. Compare with your approach
3. Learn new techniques
4. Add to problem journal
5. Mark for future review
```

## Quick Reference: Problem Pattern Finder

```python
def identify_pattern(problem_description):
    """Quick pattern identifier"""
    keywords = problem_description.lower()
    
    if "all" in keywords and ("combination" in keywords or "permutation" in keywords):
        return "Backtracking"
    
    if "minimum" in keywords or "maximum" in keywords:
        return "DP / Memoization"
    
    if "tree" in keywords or "graph" in keywords:
        return "DFS / BFS"
    
    if "sorted" in keywords and "search" in keywords:
        return "Binary Search / Divide & Conquer"
    
    if "substring" in keywords or "subsequence" in keywords:
        return "Pick/Not-Pick Pattern"
    
    return "Analyze further"
```
